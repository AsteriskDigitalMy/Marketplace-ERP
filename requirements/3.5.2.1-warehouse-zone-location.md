# 3.5.2.1 Warehouse / Zone / Location Management — Frontend Requirements

**Source:** Section 3.5.2.1  
**Purpose:** Multi-level digital warehouse map — Group → Factory → Warehouse → Zone → Location.

---

## Screens

### 1. Warehouse hierarchy browser (`/wms/locations`)

**Split layout:** tree navigator (left) + detail panel (right).

#### Tree levels (expandable)

1. Group
2. Factory
3. Warehouse
4. Zone
5. Location

**Tree node labels:** code + name + type badge.

**Actions per level:** Add child, Edit, Deactivate (confirm if has stock).

---

### 2. Warehouse form (`/wms/locations/warehouses/new`, `.../:id/edit`)

| Field | Type | Required |
|-------|------|----------|
| Warehouse code | Text | Yes, unique |
| Warehouse name | Text | Yes |
| Warehouse type | Multi-select | Raw material, semi-finished, finished, auxiliary, defective, hazardous |
| Affiliated factory | Select | Yes |
| Supervisor | User picker | Yes |
| Status | Toggle | Active |

---

### 3. Zone form

| Field | Type | Required |
|-------|------|----------|
| Zone code | Text | Yes |
| Zone name | Text | Yes |
| Parent warehouse | Read-only context | |
| Zone purpose | Select | Inspection-pending, qualified, unqualified, cold storage, constant-temp, general |
| Storage conditions | Text / structured | Temperature range, humidity, etc. |
| Responsible personnel | User picker | Yes |
| Status | Toggle | Active |

---

### 4. Location form

| Field | Type | Required |
|-------|------|----------|
| Location code | Text | Yes, unique — auto-generate option |
| Parent zone | Read-only | |
| Dimensions (L×W×H) | Number inputs | Optional |
| Weight capacity (kg) | Number | Optional |
| Storage type | Select | Pallet, shelf, floor, bin |
| Location status | Select | Idle, occupied, locked, disabled |

---

### 5. Visual location map (`/wms/locations/map/:zoneId`)

**Grid or rack visualization** of locations in selected zone.

#### Color coding

| Status | Color token |
|--------|-------------|
| Idle | `muted` |
| Occupied | `primary` |
| Locked | `warning` |
| Disabled | `destructive` muted |

**Interactions:**

- Click location → `Sheet` with stock breakdown table
- Hover tooltip: code, status, utilization %
- Filter: status, material type
- Search: jump to location code

#### Stock breakdown panel (on click)

| Column | |
|--------|--|
| Material code | |
| Material name | |
| Batch | |
| Quantity | Tabular nums |
| Reserved | |
| Available | |

## Mock Data Shape

```ts
type WarehouseNodeType = 'group' | 'factory' | 'warehouse' | 'zone' | 'location'
type LocationStatus = 'idle' | 'occupied' | 'locked' | 'disabled'

interface WarehouseTreeNodeDto {
  Id: string
  ParentId: string | null
  Type: WarehouseNodeType
  Code: string
  Name: string
  Children: WarehouseTreeNodeDto[]
  // warehouse-specific
  WarehouseTypes?: string[]
  FactoryId?: string
  Supervisor?: { Id: string; DisplayName: string }
  // zone-specific
  ZonePurpose?: string
  StorageConditions?: string
  // location-specific
  Dimensions?: { Length: number; Width: number; Height: number }
  WeightCapacityKg?: number
  StorageType?: string
  Status?: LocationStatus
  StockSummary?: { MaterialCount: number; TotalQty: number }
}

interface LocationStockDetailDto {
  LocationCode: string
  Lines: {
    MaterialCode: string
    MaterialName: string
    Batch: string
    Quantity: number
    Reserved: number
    Available: number
  }[]
}
```
