# 3.1.3.4 Multi-Mode Project Progress Updates — Frontend Requirements

**Source:** F-PROJ-004 — Section 3.1.3 Project Management  
**Routes:** `/pms/tasks/my`, `/pms/projects/:id/progress`, `/pms/projects/:id/progress/import`

---

## Functional Summary

Three update modes: personal manual, leader batch, Excel import. Progress 0–100 integers; 100% requires completion description and attachments. Missed update cycle → overdue alert.

**Prerequisites:** Executor for personal mode; task `In Progress`.

---

## Personal Manual (`/pms/tasks/my`)

Per-task update form:

| Field | Required |
|-------|----------|
| Progress % | 0–100 integer |
| Actual execution date | Yes — ≥ planned start |
| Work description | Yes |
| On-site issues | Optional ([3.1.3.6](./3.1.3.6-issue-reporting.md)) |
| Attachments | Required at 100% |

Overdue badge if past update cycle without report.

---

## Leader Batch (`/pms/projects/:id/progress`)

Multi-select tasks → unified progress % and description → apply.

---

## Excel Import

Download template → upload → row validation results (same fields).
