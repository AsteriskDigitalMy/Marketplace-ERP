# 3.1.3.4 Multi-Mode Project Progress Updates ‚Äî Frontend Requirements

**Source:** F-PROJ-004 ‚Äî Section 3.1.3 Project Management  
**Routes:** `/pms/tasks/my`, `/pms/projects/:id/progress`, `/pms/projects/:id/progress/import`  
**Related:** [3.1.3.3](./3.1.3.3-sub-task-creation.md), [3.1.3.6](./3.1.3.6-issue-reporting.md), [3.1.3.8](./3.1.3.8-kpi-data-sync.md), [3.1.3.9](./3.1.3.9-gantt-chart.md)

## Functional Summary

Three update modes: personal manual, leader batch, Excel import. Progress 0‚Äì100 integers; 100% requires completion description and attachments. Missed update cycle ‚Üí overdue alert.

**Prerequisites:** Executor for personal mode; task `In Progress`.

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Role | Goals | Pain Points |
|---------|------|-------|-------------|
| **Task Executor** | Updates own tasks | Quick mobile-friendly progress entry | Forgetting weekly update cadence |
| **Project Leader** | Batch oversight | Update multiple tasks at once | Inconsistent descriptions across team |
| **PMO Analyst** | Bulk data load | Import spreadsheet updates | Row validation failures unclear |

### Core Capabilities (FR-xxx)

| ID | Capability | Priority |
|----|------------|----------|
| FR-001 | Personal task queue at `/pms/tasks/my` | Must |
| FR-002 | Per-task progress form with 0‚Äì100 integer validation | Must |
| FR-003 | Leader batch update at `/pms/projects/:id/progress` | Must |
| FR-004 | Excel template download + upload validation at `.../progress/import` | Must |
| FR-005 | 100% completion requires narrative + attachments | Must |
| FR-006 | Overdue `Badge` when update cycle missed | Must |
| FR-007 | Embedded issue reporting per [3.1.3.6](./3.1.3.6-issue-reporting.md) | Must |
| FR-008 | Progress log history (mock timeline) | Should |
| FR-009 | Updates trigger KPI sync mock [3.1.3.8](./3.1.3.8-kpi-data-sync.md) | Should |

### Edge Cases & Alternative Paths

- **Progress decrease:** Warning confirm (audit trail preserved).
- **Actual date < planned start:** Validation error.
- **100% without attachments:** Submit blocked.
- **Import partial failures:** Results table pass/fail per row; no partial commit mock.
- **Overdue task:** Red `Badge` "Overdue" on card; links to update form.
- **Batch with mixed statuses:** Only `in_progress` tasks selectable.

## Phase 2: Screen-by-Screen Functional Specifications

### Screen: My Tasks ‚Äî `/pms/tasks/my`

* **Objective:** Executor personal queue for progress updates.

* **UI Elements & Field-Level Specs:**
  - `Card` list or `DataTable`: Task Name, Project, Due cycle, Progress `Progress` bar, Overdue `Badge`, Actions ("Update").
  - Filter `Tabs`: All | Overdue | Due this week.
  - Empty state: "No assigned tasks" with link to projects.

* **Functional Logic:**
  - Load tasks where `OwnerId === currentUser` and status `in_progress`.
  - Click Update ‚Üí expand inline `Card` form or navigate to task update panel.

---

### Screen / Panel: Personal Progress Update Form

* **Objective:** Submit single-task progress with optional issue report.

* **UI Elements & Field-Level Specs:**
  - `Form`:
    - **Progress %** ‚Äî `Slider` or `Input` integer 0‚Äì100.
    - **Actual execution date** ‚Äî `Calendar`; ‚â• planned start.
    - **Work description** ‚Äî `Textarea`, required.
    - **On-site issues** ‚Äî optional section per [3.1.3.6](./3.1.3.6-issue-reporting.md).
    - **Attachments** ‚Äî `Input` file (mock); required when progress = 100.
  - Submit `Button`; Cancel collapses panel.

* **Functional Logic:**
  1. Validate fields; at 100% enforce attachments + description.
  2. Save ‚Üí append `TaskProgressUpdateDto` to mock history; update task `ProgressPct`.
  3. Clear overdue flag if within cycle; trigger KPI sync badge [3.1.3.8](./3.1.3.8-kpi-data-sync.md).

---

### Screen: Leader Batch Progress ‚Äî `/pms/projects/:id/progress`

* **Objective:** Apply unified progress update to multiple tasks.

* **UI Elements & Field-Level Specs:**
  - `DataTable` with row `Checkbox` multi-select (in_progress only).
  - Batch fields: Progress %, shared Description `Textarea`.
  - `Button` "Apply to N tasks"; confirmation `AlertDialog`.
  - Link to import mode.

* **Functional Logic:**
  - Leader permission gate; creates individual log entries per task mock.

---

### Screen: Excel Import ‚Äî `/pms/projects/:id/progress/import`

* **Objective:** Bulk upload progress from template.

* **UI Elements & Field-Level Specs:**
  - `Button` "Download template" (mock `.xlsx`).
  - Drag-drop `Card` or file `Input` for upload.
  - Results `DataTable`: Row #, Task ID, Status (`Badge` pass/fail), Errors.
  - `Alert` summary: X passed, Y failed.

* **Functional Logic:**
  - Row validation mirrors personal form rules; failed rows skipped.

---

### Panel: Progress History ‚Äî `Sheet` or tab

* **Objective:** Audit trail of updates per task.

* **UI Elements & Field-Level Specs:** `ScrollArea` timeline: date, user, %, excerpt.

* **Functional Logic:** Read-only mock log.
