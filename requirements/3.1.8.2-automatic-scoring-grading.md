# 3.1.8.2 Automatic Employee Performance Scoring & Grading — Frontend Requirements

**Source:** F-PERF-002 — Section 3.1.8 Performance Appraisal & Scoring  
**Routes:** `/pms/appraisal/cycles`, `/pms/appraisal/cycles/:id/generate`  
**Workflow:** Requires schemes from [3.1.8.1](./3.1.8.1-appraisal-scheme-configuration.md) → outputs drafts to [3.1.8.3](./3.1.8.3-preliminary-review-list.md) → [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md) → [3.1.8.5](./3.1.8.5-hr-rectification.md) / [3.1.8.7](./3.1.8.7-executive-final-review.md)

---

## Functional Summary

At the appraisal node, the system retrieves scheme rules and individual KPI data, computes weighted total scores, assigns grades, and generates preliminary appraisal drafts for all staff.

**Input:** Appraisal statistical cycle, employee corporate ID (batch: all employees in cycle).  
**Prerequisites:** Appraisal scheme configured; period KPI data calculated.

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Goals | Pain Points |
|---------|-------|-------------|
| **Auditor / Performance Admin** | Batch-generate fair, consistent preliminary scores for a cycle | Manual calculation errors; unclear KPI readiness |
| **HR Administrator** | Confirm cycle is ready before auditors run generation | No single view of data completeness |
| **Department Manager** (indirect) | Trust automated grades match published scheme | Opaque weighting in legacy process |

### Core Capabilities (FR-001…)

| ID | Capability | Priority |
|----|------------|----------|
| FR-001 | List appraisal cycles with scheme, KPI status, and generation status | Must |
| FR-002 | Display pre-check panel: active scheme summary, employee count, KPI completeness | Must |
| FR-003 | Block generation when KPI data incomplete or no active scheme | Must |
| FR-004 | Batch generate all staff appraisals with progress UI | Must |
| FR-005 | Compute weighted total: Σ (KPI score × weight) per [3.1.8.1](./3.1.8.1-appraisal-scheme-configuration.md) | Must |
| FR-006 | Auto-assign grade from scheme grade rules | Must |
| FR-007 | Optional individual employee preview before batch commit | Should |
| FR-008 | On success, link to preliminary review queue [3.1.8.3](./3.1.8.3-preliminary-review-list.md) | Must |

### Edge Cases & Alternative Paths

- **KPI completeness < 100%:** Generate button disabled; tooltip explains missing data sources.
- **No active scheme for cycle scope:** Pre-check fails; link to [3.1.8.1](./3.1.8.1-appraisal-scheme-configuration.md).
- **Cycle already Generated:** Generate disabled; CTA to preliminary review list.
- **Cycle Published:** Read-only; no re-generate (mock policy).
- **Partial generation failure (mock):** Progress modal shows failed employees; retry failed only.
- **Zero employees in scope:** Empty state on generate page; block generate.
- **Individual preview — employee not in scope:** Inline error "Employee not in cycle scope."
- **User navigates away during progress:** Confirm dialog; mock continues in background.

## Phase 2: Screen-by-Screen Functional Specifications

### Screen: Cycle Management (`/pms/appraisal/cycles`)

* **Objective:** Monitor appraisal cycles and navigate to generation or downstream review.
* **UI Elements & Field-Level Specs:**
  * Page title: "Appraisal Cycles"
  * Table columns: Cycle label (e.g. "2026 Q1"), Scheme name (link to [3.1.8.1](./3.1.8.1-appraisal-scheme-configuration.md) read-only), Period dates, KPI data status (badge: Incomplete / Complete), Appraisal status (Not started / Ready to generate / Generated / Published), Actions
  * Appraisal status meanings documented in column header tooltip
  * Row action: **Generate appraisals** (enabled when Ready) → `/pms/appraisal/cycles/:id/generate`; **View preliminary** (when Generated) → [3.1.8.3](./3.1.8.3-preliminary-review-list.md)
  * Filters: Year, Appraisal status
  * Four async states on table
* **Functional Logic:**
  * `Ready to generate` when KPI complete and status Not started
  * Published rows: no generate action; link to reports if applicable

### Screen: Generate Flow (`/pms/appraisal/cycles/:id/generate`)

* **Objective:** Validate prerequisites and batch-generate preliminary appraisal drafts.
* **UI Elements & Field-Level Specs:**
  * Breadcrumb: Cycles → {Cycle label} → Generate
  * **Pre-check panel** card:
    * Active scheme summary: indicator names + weights (read-only from [3.1.8.1](./3.1.8.1-appraisal-scheme-configuration.md))
    * Employee count in scope
    * KPI data completeness progress bar with % label
    * Pass/fail checklist icons per pre-check
  * **Individual preview** collapsible section: Employee ID search, **Preview** button; results panel — total score, auto grade badge, expandable indicator breakdown table
  * Primary CTA: **Generate all staff appraisals** (disabled until all pre-checks pass)
  * Secondary: **Back to cycles**
* **Functional Logic:**
  * Load cycle + bound scheme; 404 if invalid id
  * Preview calls mock calc without persisting
  * Generate opens progress modal (see below)

### Modal: Batch Generation Progress

* **Objective:** Show per-employee scoring progress during batch run.
* **UI Elements & Field-Level Specs:**
  * Progress bar (overall %)
  * Scrollable log: employee name, status (pending / scoring / done / failed)
  * Expandable sample row: formula display `Σ (KPI score × weight)` with line items
  * On complete: success summary — "N drafts created" + **Go to preliminary review** link
  * **Close** disabled until complete or user confirms cancel
* **Functional Logic:**
  * Mock staggered updates every 200ms
  * Apply grade rules from scheme after score computed
  * Set cycle status `Generated`; create rows for [3.1.8.3](./3.1.8.3-preliminary-review-list.md)

### Modal: Generation Success Summary

* **Objective:** Confirm completion and route auditor to next workflow step.
* **UI Elements:** Count by grade (A/B/C/D chips), **Open preliminary review**, **Stay on page**
* **Functional Logic:** Primary CTA navigates to `/pms/appraisal/preliminary?cycleId=:id`

| # | Trigger | Source | Destination | Conditional paths |
|---|---------|--------|-------------|-------------------|
| 1 | Nav to Appraisal Cycles | Sidebar | Cycle Management | Permission gate |

| 9 | **Back to cycles** | Generate Flow | Cycle Management | — |
| 10 | Post-generation | Cycle Management | Row status = Generated | — |
