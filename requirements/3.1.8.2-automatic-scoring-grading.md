# 3.1.8.2 Automatic Employee Performance Scoring & Grading — Frontend Requirements

**Source:** F-PERF-002 — Section 3.1.8 Performance Appraisal & Scoring  
**Routes:** `/pms/appraisal/cycles`, `/pms/appraisal/cycles/:id/generate`

---

## Functional Summary

At the appraisal node, the system retrieves scheme rules and individual KPI data, computes weighted total scores, assigns grades, and generates preliminary appraisal drafts for all staff.

**Input:** Appraisal statistical cycle, employee corporate ID (batch: all employees in cycle).  
**Prerequisites:** Appraisal scheme configured; period KPI data calculated.

---

## Cycle Management (`/pms/appraisal/cycles`)

Table: Cycle label (e.g. "2026 Q1"), Scheme name, Period dates, KPI data status, Appraisal status, Actions.

| Appraisal status | Meaning |
|------------------|---------|
| Not started | KPI data incomplete or cycle not open |
| Ready to generate | KPI complete; "Generate appraisals" enabled |
| Generated | Drafts created; in preliminary review |
| Published | Final approval complete |

---

## Generate Flow (`/pms/appraisal/cycles/:id/generate`)

### Pre-check panel

- Active scheme summary (indicators + weights)
- Employee count in scope
- KPI data completeness bar (mock %)

### Generate action

Button **"Generate all staff appraisals"** (disabled until pre-checks pass).

On click:

1. Mock progress modal — per-employee scoring steps
2. Weighted calculation display (sample row expandable): `Σ (KPI score × weight)`
3. Auto grade assignment per scheme rules ([3.1.8.1](./3.1.8.1-appraisal-scheme-configuration.md))
4. Success summary: N drafts created → link to [preliminary review list](./3.1.8.3-preliminary-review-list.md)

### Individual preview (optional expand)

Search by employee ID → show calculated score, grade, indicator breakdown before batch commit.
