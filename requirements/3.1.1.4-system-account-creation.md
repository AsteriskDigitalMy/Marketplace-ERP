# 3.1.1.4 System Account Creation — Frontend Requirements

**Source:** F-BASE-004 — Section 3.1.1 System Basic Management  
**Routes:** `/pms/admin/accounts`, `/pms/admin/accounts/import`

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Goals |
|---------|-------|
| **System Administrator (primary)** | Onboard employees quickly via single create or bulk Excel import with clear per-row error reporting. |
| **HR Coordinator** | Import large batches after hiring events; download template and fix failed rows offline. |
| **IT Security Officer** | Ensure initial passwords meet strength rules and employee IDs are globally unique. |
| **Department Manager (indirect)** | Expects new accounts to appear in account list with correct department and roles bound. |

### Core Capabilities

1. **FR-001 — Account list hub:** Paginated **Table** of accounts at `/pms/admin/accounts` with navigation to create and import flows.
2. **FR-002 — Single account create:** Form at `/pms/admin/accounts/new` aligned with `UserAccountCreateRequestSchema` in `src/models/pms/identity.ts`.
3. **FR-003 — Employee ID uniqueness:** Client + mock server check; block submit on duplicate.
4. **FR-004 — Department org picker:** Select active department from org tree (`OrgUnit`); disabled units excluded.
5. **FR-005 — Role binding:** Multi-select at least one valid preset `SystemRole`; validate against `RoleIds` min(1).
6. **FR-006 — Password strength:** Initial password min 8 chars (Zod); mock strength meter (weak/fair/strong).
7. **FR-007 — Post-create notification:** Display generated `LoginAccount` and mock password notification **Toast**.
8. **FR-008 — Bulk import:** Template download, file upload, row-level validation, results **Table** at `/pms/admin/accounts/import`.
9. **FR-009 — Import summary:** Chips for success/fail counts; downloadable failed-rows report (mock).
10. **FR-010 — Async states:** Loading, empty, error, success across list, create, and import flows.

### Edge Cases & Alternative Paths

- Duplicate employee ID on single create → inline field error.
- No roles configured in system → **Alert** on create form; submit disabled.
- Import file wrong format → **Toast** error before parsing.
- Import partial success → results table shows mixed statuses; user downloads failures only.
- User uploads empty spreadsheet → validation message "No data rows found."
- Double-click submit on create or import confirm → disabled until complete.
- User navigates away mid-import → confirm dialog if upload in progress.
- Department picker: user expands tree to disabled node → node not selectable.

## Phase 2: Screen-by-Screen Functional Specifications

### Screen Name: Account List (`/pms/admin/accounts`)

* **Objective:** Entry point for account management; launch single create and bulk import.

* **UI Elements & Field-Level Specs:**
  * **Page header:** "User Accounts"; count badge.
  * **Toolbar — New Account (`Button`):** Navigates to `/pms/admin/accounts/new`.
  * **Toolbar — Bulk Import (`Button` outline):** Navigates to `/pms/admin/accounts/import`.
  * **Search (`Input`):** Filter by name, employee ID, login account; debounced.
  * **Accounts `Table`:** Columns Employee Name, Employee ID, Login Account, Department, Status **Badge**, Actions (View → `/pms/admin/accounts/:id`).
  * **Pagination:** shadcn pagination; page size 20.
  * **Empty state:** "No accounts yet" + **New Account** CTA.
  * **Loading:** Table skeleton rows.

* **Functional Logic:**
  * Fetch from `src/mocks/pms/user-accounts.ts`.
  * Row click or View navigates to account detail ([3.1.1.5](./3.1.1.5-account-lifecycle-management.md)).

### Screen Name: Single Account Create (`/pms/admin/accounts/new`)

* **Objective:** Create one user account with HR fields, password, and role binding.

* **UI Elements & Field-Level Specs:**
  * **Employee name (`Input`):** Required; max 200.
  * **Employee ID (`Input`):** Required; max 50; uniqueness check on blur (async).
  * **Department (`Combobox` + org tree popover):** Required; active units only from `OrgUnit`.
  * **Position/title (`Input`):** Required; max 100.
  * **Initial password (`Input` type="password"):** Required; min 8; show/hide toggle; strength meter.
  * **Bound role(s) (`Multi-select` or checkbox group):** Required ≥ 1; options from `SystemRole` mocks.
  * **Cancel (`Button` outline):** → `/pms/admin/accounts`.
  * **Create Account (`Button`):** Disabled when invalid or submitting.

* **Functional Logic:**
  * Validate with `UserAccountCreateRequestSchema`.
  * Submit mock `POST`; generate `GeneratedLoginAccount`.
  * Success **Toast:** "Account created — Login: {account}. Notification sent (mock)."
  * Navigate to `/pms/admin/accounts/:id` or back to list (config: prefer detail page).
  * Double-click prevention on submit.

### Screen Name: Bulk Import (`/pms/admin/accounts/import`)

* **Objective:** Import many accounts from Excel with per-row validation feedback.

* **UI Elements & Field-Level Specs:**
  * **Step indicator:** 1 Download template → 2 Upload → 3 Results.
  * **Download template (`Button` link):** Mock `.xlsx` from static asset.
  * **File upload (`Input` type="file" or dropzone):** Accept `.xlsx`, `.xls`; max size mock 5MB.
  * **Upload & Validate (`Button`):** Disabled until file selected.
  * **Progress (`Progress` bar):** Shown during mock parse/validate.
  * **Results `Table`:** Columns Row #, Employee ID, Status **Badge** (success/failed), Error message.
  * **Summary chips:** "{N} succeeded", "{N} failed".
  * **Download failed rows (`Button` outline):** Mock CSV of failures only.
  * **Done (`Button`):** → `/pms/admin/accounts`.

* **Functional Logic:**
  * Parse file (mock delay); validate each row against same rules as single create.
  * Populate `BulkImportResultDto[]`.
  * No partial commit UI toggle — mock commits successes immediately on validate complete.
  * Errors per row in **Table**; scroll to first failure on complete.

## Phase 3: Screen Transition & User Flow Mapping

| # | Trigger Event | Source Screen | Destination Screen | Conditional Paths |
|---|---------------|---------------|--------------------|-------------------|
| 1 | Page load | — | Account List | Permission denied → 403 |
| 2 | Click **New Account** | Account List | Single Account Create | — |
| 3 | Click **Bulk Import** | Account List | Bulk Import | — |
| 4 | Click **Create Account** (success) | Single Account Create | Account Detail or Account List | Prefer detail `/pms/admin/accounts/:id` |
| 5 | Click **Cancel** | Single Account Create | Account List | Dirty form → confirm |
| 6 | Duplicate employee ID | Single Account Create | Single Account Create | Inline error; no navigation |
| 7 | Upload + validate complete | Bulk Import | Bulk Import (results step) | All failed → no Done until acknowledged |
| 8 | Click **Done** | Bulk Import | Account List | — |
| 9 | Click row **View** | Account List | Account Detail (`/pms/admin/accounts/:id`) | — |
