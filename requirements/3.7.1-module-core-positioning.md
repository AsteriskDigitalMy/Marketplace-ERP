# 3.7.1 Core Module Positioning — Frontend Requirements

**Source:** Section 3.7.1 — Business Intelligence (BI) Visualization Dashboard  
**Module role:** Decision brain — aggregate SCM, MES, WMS, SAP data into multidimensional, role-based visual analytics.

---

## Overview

BI is the **read-only analytics layer** of the ERP frontend. It does not mutate operational data; it visualizes mocked aggregates derived from the same DTO shapes as source modules.

Three-tier matrix:

| Tier | Audience | Primary routes |
|------|----------|----------------|
| Strategic | Group / executives | `/bi/executive` |
| Operational | Department heads | `/bi/operations/*` |
| Execution | Workshop / warehouse site | `/bi/kanban/*` |

| Component | Purpose |
|-----------|---------|
| `BiKpiCard` | Metric + trend sparkline |

## Data Layer (mock)

`src/mocks/bi/` — aggregate endpoints shaped as:

```ts
interface BiDashboardPayloadDto {
  DashboardId: string
  LastRefreshedAt: string
  RefreshIntervalSeconds: number
  Filters: Record<string, string>
  Kpis: { Label: string; Value: number | string; Unit?: string; TrendPercent?: number }[]
  Charts: BiChartSeriesDto[]
}
```

Hooks: `useBiDashboard(id, filters)` — simulates RT-OLAP latency (mock delay 200–800ms).

## Permission Model (UI)

Dashboard access gated by role scope:

| Role (mock) | Visible data scope |
|-------------|-------------------|
| Executive | Group-wide |
| Regional manager | Region filter locked |
| Plant manager | Single plant |
| Workshop lead | Workshop only |

Unauthorized dashboard → 403 empty state with scope explanation.
