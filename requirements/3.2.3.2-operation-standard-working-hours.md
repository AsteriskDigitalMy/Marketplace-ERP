# 3.2.3.2 Operation Standard Working Hours Management — Frontend Requirements

**Source:** Section 3.2.3.2  
**Purpose:** Maintain standard working hours per operation for scheduling, piece-rate wages, and efficiency analysis benchmarks.

---

## Screens

### 1. Working hours list (`/pdm/working-hours`)

**Table grouped by operation or flat list toggle.**

| Column | Notes |
|--------|-------|
| Operation | Link — from process library |
| Product category | Badge |
| Setup time (min) | Tabular nums |
| Processing time (min) | Tabular nums |
| Auxiliary time (min) | Tabular nums |
| Total standard hours | Calculated, bold |
| Version | `v{n}` |
| Approval status | Draft / In review / Approved |
| Effective date | |

**Filters:** operation, category, approval status.

**Actions:** `New entry`, `Merge operations` (opens merge wizard).

---

### 2. Working hours form (`/pdm/working-hours/new`, `/pdm/working-hours/:id/edit`)

| Field | Type | Notes |
|-------|------|-------|
| Operation | Process picker | Required |
| Product category | Select | Required |
| Setup time | Duration input (minutes) | ≥ 0 |
| Processing time | Duration input | ≥ 0 |
| Auxiliary time | Duration input | ≥ 0 |
| Total | Read-only calculated | setup + processing + auxiliary |
| Notes | Textarea | Optional |

**Live total** updates as user types (tabular numerals).

**Submit:** requires approval — `Submit for review` (not immediately effective).

---

### 3. Operation merge wizard (`/pdm/working-hours/merge`)

**Multi-step dialog or page:**

**Step 1 — Select operations**

- Multi-select from process library (min 2).
- Show selected list with remove.

**Step 2 — Name composite operation**

| Field | |
|-------|--|
| Composite name | Text |
| Composite code | Auto-suggested, editable |

**Step 3 — Review calculated total**

- Table of source operations with individual hours.
- **Auto-calculated merged total** (sum of all selected).
- Allow manual override with required justification textarea.

**Step 4 — Confirm**

- Summary + submit for approval.

---

### 4. Approval & version history

**Detail page tabs:**

#### Current version

Approved hours display (read-only when approved).

#### Version history

| Version | Effective | Setup | Processing | Auxiliary | Total | Changed by | Actions |
|---------|-----------|-------|------------|-----------|-------|------------|---------|
| v3 | 2026-06-01 | 5 | 12 | 3 | 20 | Jane | Compare |
| v2 | 2026-04-15 | 5 | 10 | 3 | 18 | John | Compare |

**Compare view (`Sheet`):**

- Side-by-side v2 vs v3.
- **Variance highlight:** cells where values differ shown with warning background and delta column (`+2 min processing`).

#### Pending approval

If modification in review, banner on detail: "Pending approval — v4 awaiting sign-off." Show draft values grayed alongside current effective.

---

## Business Rule UI Messaging

- Info alert on edit of approved record: "Changes require approval and will create a new version."
- Tooltip on total field: "Used by MES scheduling and piece-rate calculations."
