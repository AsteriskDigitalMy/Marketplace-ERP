# 3.1.2.1 New KPI Indicator Addition — Frontend Requirements

**Source:** F-KPI-001 — Section 3.1.2 KPI Indicator Library Management  
**Route:** `/pms/kpi/indicators/new`

---

## Functional Summary

Create performance evaluation indicators with formula, targets, cycle, data source, weight, and alert threshold. Saving creates version V1.0 after formula simulation validation.

**Prerequisites:** KPI indicator library management permission.

---

## Indicator Form

| Field | Required | Validation |
|-------|----------|------------|
| Indicator code | Yes | Globally unique |
| Indicator name | Yes | |
| Business category | Yes | Dictionary picker |
| Statistical scope/method | Yes | |
| Calculation formula | Yes | Formula editor ([3.1.2.3](./3.1.2.3-kpi-formula-editor.md)) |
| Target value | Yes | > 0 |
| Challenge value | Optional | |
| Baseline value | Optional | |
| Statistical cycle | Yes | Daily/weekly/monthly/quarterly/annual |
| Evaluation object | Yes | Dept/individual/project |
| Data source | Yes | Auto/manual/mixed |
| Evaluation weight | Optional | % |
| Anomaly alert threshold | Optional | |

**Pre-submit:** "Simulate & validate" runs mock trial calculation; block save on syntax errors.

On save: toast with version `V1.0` created.
