# 3.1.3.11 Project Completion Acceptance Approval — Frontend Requirements

**Source:** F-PROJ-011 — Section 3.1.3 Project Management  
**Route:** `/pms/projects/acceptance-reviews`

---

## Functional Summary

Auditors verify overall project outcomes. Pass → archived/sealed permanently; fail → return with rectification requirements.

**Prerequisites:** Final project acceptance permission; status `Pending acceptance`.

---

## Review Inbox

Table: Project code, Name, Leader, Target achievement, Submitted at, Actions.

---

## Review Detail

Tabs: Execution materials, Task summary, Outcome attachments, KPI achievement ([3.1.3.8](./3.1.3.8-kpi-data-sync.md)).

| Field | Required |
|-------|----------|
| Comprehensive opinion | Yes |
| Overall evaluation grade | A/B/C/D |
| Decision | Pass / Fail |

**Pass:** status `Archived`; read-only seal badge; data retained.  
**Fail:** rectification requirements + deadline; return to leader.

Archived projects: searchable in archive list for later business review (mock).
