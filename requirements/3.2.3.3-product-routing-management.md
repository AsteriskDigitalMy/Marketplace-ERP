# 3.2.3.3 Product Routing Management — Frontend Requirements

**Source:** Section 3.2.3.3  
**Purpose:** Configure complete production routing per finalized product — operation sequence, dependencies, equipment, hours, and inspection requirements.

---

---

### 2. Visual routing builder (`/pdm/routing/:id/edit`)

**Primary UI — drag-and-drop flow canvas.**

#### Canvas features

| Feature | Implementation |
|---------|----------------|
| Operation nodes | Cards: sequence #, process name, workshop, std hours |
| Drag reorder | Change sequence order |
| Parallel group | User can place operations side-by-side in parallel lane |
| Dependency arrows | Connect nodes; validation prevents cycles |
| Add operation | Opens `ProcessPicker` from standard library |
| Remove operation | Confirm if node has BOM links |

#### Node detail panel (`Sheet` on node click)

| Field | Source |
|-------|--------|
| Process | From library (read-only ref) |
| Standard hours | Auto from working hours table; override with justification |
| Required equipment | Multi-select (mock equipment master) |
| Inspection requirement | Checkbox + link to inspection standard |
| BOM material input node | Select materials from product BOM consumed at this step |

#### BOM linkage panel (side drawer)

- Load product BOM tree (mock).
- Drag material lines onto operation nodes OR select via dropdown on node.
- Visual indicator on node: material count badge.

---

### 3. Routing validation

**Pre-save checks** (inline error list):

| Rule | Message |
|------|---------|
| Empty routing | At least one operation required |
| Cycle detected | Dependency cycle error with highlighted nodes |
| Unlinked BOM materials | Warning list of materials not assigned to any operation |
| Missing hours | Error on nodes without standard hours |

---

### 4. Routing approval & MES sync

**Submit for approval** → `ApprovalTimeline` (Technical → Production).

On approval:

- Version locked
- **MES sync banner** (mock): "Routing v{n} synchronized to MES"
- Sync status badge on list → `Synced`

---

### 5. Routing version management

**Version selector** in header (same pattern as design management).

| Action | Behavior |
|--------|----------|
| View history | Table of versions with change summary |
| Compare | Side-by-side canvas or table diff of operation sequences |
| Create new version | Editing approved routing forks to new draft v{n+1} |

**Change summary** required textarea when saving new version.

---

### 6. Read-only routing view (`/pdm/routing/:id`)

For archived / approved routings:

- Canvas read-only
- Export PNG/PDF of flow (mock download)
- Pad-friendly simplified list view: ordered operations with SOP links

## Mock Data Shape

```ts
interface ProductRoutingDto {
  Id: string
  ProductId: string
  StyleNumber: string
  ProductName: string
  Version: number
  ApprovalStatus: 'draft' | 'in_review' | 'approved'
  MesSyncStatus: 'pending' | 'synced' | 'failed'
  Operations: RoutingOperationDto[]
  Dependencies: { fromOperationId: string; toOperationId: string }[]
  ParallelGroups: string[][]  // operation ids per parallel lane
  BomLinks: { operationId: string; bomLineIds: string[] }[]
  Versions: RoutingVersionSummaryDto[]
}

interface RoutingOperationDto {
  Id: string
  Sequence: number
  ProcessId: string
  ProcessCode: string
  ProcessName: string
  Workshop: string
  StandardHoursMinutes: number
  EquipmentIds: string[]
  InspectionRequired: boolean
  InspectionStandardId: string | null
}
```

Mocks: `src/mocks/pdm/routing.ts`.

**Note:** Canvas can use a library (e.g. React Flow) or simplified vertical sortable list for MVP — document both; MVP may ship list-based reorder with parallel group tags.

---

## Acceptance Criteria

- [ ] Visual (or list-based) routing builder supports reorder, parallel groups, dependencies
- [ ] Operations added from standard process library without re-entry
- [ ] BOM materials linkable to operation nodes with validation warnings
- [ ] Approval workflow and MES sync status displayed (mocked)
- [ ] Version history with compare and new version on change
- [ ] Validation prevents cycles and surfaces unlinked materials
- [ ] Four async states throughout
