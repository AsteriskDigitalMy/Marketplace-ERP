# 3.1.3.9 Automatic Project Gantt Chart Generation — Frontend Requirements

**Source:** F-PROJ-009 — Section 3.1.3 Project Management  
**Route:** `/pms/projects/:id/gantt`  
**Related:** [3.1.3.3](./3.1.3.3-sub-task-creation.md), [3.1.3.2](./3.1.3.2-project-initiation-approval.md) (Gantt init), [3.1.3.4](./3.1.3.4-project-progress-updates.md), [3.1.3.5](./3.1.3.5-task-duration-change.md)

## Functional Summary

Auto-generated Gantt from planned and actual task durations. Timeline zoom, filter by executor, click drill-down, export HD image (mock).

**Prerequisites:** Project has sub-tasks ([3.1.3.3](./3.1.3.3-sub-task-creation.md)).

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Role | Goals | Pain Points |
|---------|------|-------|-------------|
| **Project Leader** | Schedule oversight | See planned vs actual at a glance | Cluttered chart with many executors |
| **PMO Executive** | Portfolio review | Export chart for meetings | Non-responsive desktop-only charts |
| **Task Executor** | Focus on own work | Filter to personal bars | Cannot see dependency impact |

### Core Capabilities (FR-xxx)

| ID | Capability | Priority |
|----|------------|----------|
| FR-001 | Gantt tab at `/pms/projects/:id/gantt` auto-built from tasks | Must |
| FR-002 | Timeline scale toggle: day / week / month | Must |
| FR-003 | Filter by executor multi-select | Must |
| FR-004 | Legend: planned (blue), actual (green), overdue (red) | Must |
| FR-005 | Click task bar → detail `Sheet` drawer | Must |
| FR-006 | Hover tooltip: name, %, dates | Must |
| FR-007 | Export image mock PNG download | Should |
| FR-008 | Horizontal scroll on narrow viewports | Must |
| FR-009 | Refresh on duration change approve [3.1.3.5](./3.1.3.5-task-duration-change.md) | Should |

### Edge Cases & Alternative Paths

- **No sub-tasks:** Empty state CTA "Create tasks" → [3.1.3.3](./3.1.3.3-sub-task-creation.md).
- **All tasks filtered out:** Empty chart message "Adjust filters".
- **Missing actual dates:** Show planned bar only; actual segment hidden.
- **Dependency lines:** Optional mock arrows between prerequisite tasks.
- **Export failure:** Toast error with retry.

## Phase 2: Screen-by-Screen Functional Specifications

### Screen: Gantt View — `/pms/projects/:id/gantt`

* **Objective:** Visualize project schedule from task planned/actual data.

* **UI Elements & Field-Level Specs:**
  - Project tab: **Gantt** on detail layout.
  - **Toolbar** `Card`:
    - Timeline scale — `ToggleGroup` or `Select`: Day | Week | Month.
    - Executor filter — multi-select `Combobox` with `Badge` chips.
    - `Button` "Export image" (outline) with `Download` icon.
    - `Button` "Refresh" icon.
  - **Legend** — inline labels with color swatches: Planned (`primary`), Actual (success semantic), Overdue (destructive).
  - **Chart area** — custom Gantt or shadcn Chart/Recharts timeline mock:
    - Left column: task names (sticky).
    - Right scrollable timeline with dual bars per row.
    - `ScrollArea` horizontal on mobile.
  - **Tooltip** — `Tooltip` on bar hover: task name, progress %, planned/actual date range.

* **Functional Logic:**
  1. Load `GanttTaskDto[]` from `src/mocks/pms/project-gantt.ts` derived from [3.1.3.3](./3.1.3.3-sub-task-creation.md) tasks.
  2. Apply executor filter client-side.
  3. Scale changes recompute column widths mock.
  4. Overdue tasks: actual bar or planned end past today flagged red.
  5. Export triggers mock blob download `gantt-{projectCode}.png`.

---

### Drawer: Task Detail — `Sheet`

* **Objective:** Drill-down from Gantt bar to task context.

* **UI Elements & Field-Level Specs:**
  - Task name, owner, status `Badge`, progress `Progress` bar.
  - Planned vs actual dates comparison.
  - Links: "View task" → `/pms/projects/:id/tasks`; "Update progress" → [3.1.3.4](./3.1.3.4-project-progress-updates.md).

* **Functional Logic:**
  - Opened on bar click; loads task by `TaskId`.

- [ ] Legend and overdue red styling for late tasks
- [ ] Click bar opens task detail `Sheet` with progress and date summary
- [ ] Hover `Tooltip` with name, %, and dates
- [ ] Export image mock PNG download
- [ ] Responsive horizontal `ScrollArea` on narrow viewports
- [ ] Empty state when no sub-tasks ([3.1.3.3](./3.1.3.3-sub-task-creation.md))
