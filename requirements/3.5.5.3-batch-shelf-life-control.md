Search by serial — single result detail.

**One item, one code** — display QR/barcode preview (mock).

---

### 4. Shelf-life control panel

**Expiring soon queue** — sortable by days remaining.

**Expired / locked stock:**

- Read-only table
- Banner: "Expired — outbound blocked"
- CTA: quarantine transfer (link to misc outbound stub)

---

### 5. Picking enforcement UI (embedded in outbound PDA)

When allocation strategy = FIFO/FEFO:

- Recommended batch shown prominently
- Attempt to scan wrong batch → **block** with error toast
- Attempt to pick expired batch → **structural block** — no override in standard flow

Admin override log (supervisor only, mock role): rare exception with mandatory reason.

## Mock Data Shape

```ts
interface BatchDto {
  BatchNumber: string
  MaterialCode: string
  MaterialName: string
  SupplierName: string
  ArrivalDate: string
  ProductionDate: string | null
  ShelfLifeEnd: string
  DaysRemaining: number
  Status: 'active' | 'near_expiry' | 'expired' | 'locked'
  InspectionReportNumber: string
  OnHandByLocation: { LocationCode: string; Quantity: number }[]
  LifecycleEvents: { Event: string; Timestamp: string }[]
}

interface SerialNumberDto {
  SerialNumber: string
  ProductCode: string
  BatchNumber: string
  Status: 'in_stock' | 'shipped' | 'scrapped'
  LocationCode: string | null
  CustomerShipReference: string | null
}
```
