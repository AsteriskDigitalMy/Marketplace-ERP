# 3.5.6 Inventory Counting (Stock-Taking) — Frontend Requirements

**Source:** Section 3.5.6  
**Flow:** Plan → task generation → freeze scope → PDA/RFID count → variance review → adjustment approval → archive

---

## Admin Screens

| Column | |
|--------|--|

---

### 2. Create plan (`/wms/stock-taking/new`)

| Field | Type |
|-------|------|
| Count type | Select |
| Scope | Multi-select tree (warehouse → zone → location) OR material filter |
| Scheduled date | Date |
| Assigned counters | Multi user picker |
| Notes | Textarea |

**On activate:** mock generates tasks + freezes transactions in scope (banner on WMS home).

---

### 3. Plan detail (`/wms/stock-taking/:id`)

**Tabs:** Overview, Tasks, Count entries, Variance, Approval, Archive

#### Tasks tab

Per counter PDA task list with completion %.

#### Variance tab

| Material | Location | Book qty | Physical qty | Variance | Variance % |
|----------|----------|----------|--------------|----------|------------|

Highlight non-zero variances. Sort by absolute variance.

**Approval workflow** for adjustments → on approve, update ledger + surplus/shortage logs (mock SAP sync).

---

## PDA Counting (`/wms/pda/stock-taking/:taskId`)

### Barcode mode (default)

1. Scan location barcode
2. Per material: scan material barcode
3. Enter physical count
4. **Real-time variance indicator** — green if match, amber/red if diff vs book (mock book qty)

### RFID bulk mode (toggle)

- `Simulate RFID scan` button — mock reads all tags in location
- Table auto-populated with detected materials/qty
- Variance calculated automatically

---

## Archive

Closed plans → read-only archive with:

- Original plan
- Raw count entries (who, when, what)
- Approved variance adjustments
- Permanent — no delete in UI

## Mock Data Shape

```ts
type CountPlanType = 'periodic' | 'cycle' | 'movement_triggered' | 'ad_hoc'

interface StockTakingPlanDto {
  Id: string
  PlanNumber: string
  CountType: CountPlanType
  Scope: { WarehouseCodes: string[]; ZoneCodes: string[]; LocationCodes: string[] }
  ScheduledDate: string
  AssignedCounters: { Id: string; DisplayName: string }[]
  Status: 'draft' | 'active' | 'counting' | 'review' | 'closed'
  FreezeActive: boolean
  Tasks: StockTakingTaskDto[]
  VarianceLines: {
    MaterialCode: string
    LocationCode: string
    BookQty: number
    PhysicalQty: number
    Variance: number
  }[]
  ApprovalStatus: string | null
}

interface StockTakingTaskDto {
  Id: string
  CounterId: string
  LocationCode: string
  Status: 'pending' | 'in_progress' | 'complete'
  Entries: { MaterialCode: string; PhysicalQty: number; ScannedAt: string }[]
}
```
