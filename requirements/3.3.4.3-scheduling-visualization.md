# 3.3.4.3 Scheduling Visualization and Adjustment — Frontend Requirements

**Source:** Section 3.3.4.3

---

## Gantt board (`/scm/scheduling/board`)

**Views:** by order | workshop | line | equipment.

**Color status:** Pending schedule, Scheduled, Dispatched, In production, Completed, Delayed.

---

### Drag-and-drop adjustment

- Move WO bar horizontally (time shift)
- Reassign line/equipment via drop target
- Auto-recalculate subsequent ops (mock ripple indicator)

**Priority bump** — context menu raises WO priority.

**Adjustment log** — sidebar lists all manual changes with user and timestamp.

---

## Mock interactions

On drag end: toast "Schedule recalculated" + log entry.

Conflict detection: overlap alert on same line.

---

## Acceptance Criteria

- [ ] Gantt with multi-view and status colors
- [ ] Drag time and line reassignment with conflict warnings
- [ ] Priority adjustment and automatic change log
- [ ] Four async states on board load
