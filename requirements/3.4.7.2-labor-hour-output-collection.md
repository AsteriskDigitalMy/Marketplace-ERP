# 3.4.7.2 Labor Hour and Output Data Collection — Frontend Requirements

**Source:** Section 3.4.7.2  
**Purpose:** Auto-collect from Bao-Gong — hours, output, multi-person split, hour types, anomaly flagging.

---

## Screens

### Labor log (`/mes/personnel/labor`)

Auto-populated from Bao-Gong reports.

| Operator | WO | Process | Start | End | Hours | Qualified | Unqualified | Type |
|----------|-----|---------|-------|-----|-------|-----------|-------------|------|

**Hour types:** Regular, Overtime, Rework, Auxiliary — badge per row.

---

### Multi-person allocation (`/mes/personnel/labor/:reportId/allocate`)

When multiple operators on one step:

| Operator | Allocation % | Hours | Output |
|----------|--------------|-------|--------|

% must sum to 100 — validation.

---

### Anomaly review queue

Flagged rows: unusual hours/output vs norms.

Reviewer: confirm or adjust with note — read-only after lock.

**Tamper protection UI:** locked records show lock icon; edit disabled.

---

### Pad — operator labor summary

Today's hours and output totals for logged-in user.

---

interface LaborHourRecordDto {

  QualifiedOutput: number
  UnqualifiedOutput: number
  IsAnomaly: boolean
  AnomalyReviewStatus: 'pending' | 'confirmed' | 'adjusted' | null
  Allocations: { OperatorId: string; Percent: number }[] | null

}
