# 3.1.3.8 Automatic Synchronization of Project Data to KPIs — Frontend Requirements

**Source:** F-PROJ-008 — Section 3.1.3 Project Management  
**Route:** `/pms/projects/:id/kpi-sync` (read-only monitor)

---

## Functional Summary

Project completion progress, on-time rate, and delay counts auto-sync to data collection for project-type KPI calculations. Data carries project sync source flag; traceable back to tasks.

**Prerequisites:** KPI mapping rules configured at initiation ([3.1.3.1](./3.1.3.1-project-initiation.md)).

---

## Sync Monitor Panel

On project detail → **KPI Sync** tab:

| Metric | Source | Last synced |
|--------|--------|-------------|
| Overall completion % | Task rollup | timestamp |
| On-time completion rate | Task dates vs actual | timestamp |
| Delay count | Overdue tasks | timestamp |

### Sync log table

Task event, captured value, target KPI indicator, sync status, trace link to task.

**Trace:** click KPI row → jump to originating sub-task detail.

Mock: status changes on task update trigger sync badge animation.
