**Source:** F-PROJ-008 — Section 3.1.3 Project Management

**Related:** [3.1.3.1](./3.1.3.1-project-initiation.md) (KPI binding), [3.1.3.4](./3.1.3.4-project-progress-updates.md) (sync triggers), [3.1.2.1](./3.1.2.1-kpi-indicator-addition.md), [3.1.4.2](./3.1.4.2-manual-data-entry.md), [3.1.3.11](./3.1.3.11-project-acceptance-approval.md)

## Functional Summary

Project completion progress, on-time rate, and delay counts auto-sync to data collection for project-type KPI calculations. Data carries project sync source flag; traceable back to tasks.

**Prerequisites:** KPI mapping rules configured at initiation ([3.1.3.1](./3.1.3.1-project-initiation.md)).

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Role | Goals | Pain Points |
|---------|------|-------|-------------|
| **Project Leader** | Monitors KPI contribution | Confirm project data feeds indicators | Opaque sync failures |
| **KPI Administrator** | Validates data pipeline | Trace values to source tasks | Manual reconciliation with [3.1.4.2](./3.1.4.2-manual-data-entry.md) |
| **Auditor** | Reviews acceptance | KPI achievement evidence ([3.1.3.11](./3.1.3.11-project-acceptance-approval.md)) | Stale sync timestamps |

### Core Capabilities (FR-xxx)

| ID | Capability | Priority |
|----|------------|----------|
| FR-001 | KPI Sync tab on project detail `/pms/projects/:id/kpi-sync` | Must |
| FR-002 | Metric cards: completion %, on-time rate, delay count | Must |
| FR-003 | Sync log table with task events, values, KPI targets, status | Must |
| FR-004 | One-click trace from KPI row to source sub-task | Must |
| FR-005 | `project_auto_sync` source flag displayed | Must |
| FR-006 | Auto-sync animation on task update from [3.1.3.4](./3.1.3.4-project-progress-updates.md) | Must |
| FR-007 | Sync error state in log with retry mock | Should |

### Edge Cases & Alternative Paths

- **No bound KPIs:** Empty state link to project edit [3.1.3.1](./3.1.3.1-project-initiation.md).
- **Sync error row:** `Badge` destructive; expand error detail.
- **Disabled KPI after binding:** Warning `Alert` on metric card.
- **Task deleted after sync:** Log retains snapshot; trace shows "task removed".
- **Read-only:** No edit controls on monitor.

## Phase 2: Screen-by-Screen Functional Specifications

### Screen: KPI Sync Monitor — `/pms/projects/:id/kpi-sync`

* **Objective:** Read-only visibility into project → KPI data pipeline.

* **UI Elements & Field-Level Specs:**
  - Project tab navigation includes **KPI Sync** alongside Tasks, Gantt, etc.
  - **Metric summary row** — three `Card`s:
    - Overall completion % (from task rollup)
    - On-time completion rate (planned vs actual dates)
    - Delay count (overdue tasks)
    - Each shows Last synced timestamp + `Badge` `project_auto_sync`.
  - **Mappings table** — `DataTable`: KPI Name (link [3.1.2.2](./3.1.2.2-kpi-indicator-lifecycle.md)), Synced Value, Source Flag, Last Synced, Trace action.
  - **Sync log** — `DataTable`: At, Task (link), Event, Target KPI, Value, Status (`Badge` success/error), Trace link.
  - Refresh `Button` with `Loader2`; manual refresh mock.
  - Sync pulse animation `Badge` when triggered from progress update [3.1.3.4](./3.1.3.4-project-progress-updates.md).

* **Functional Logic:**
  1. Load `ProjectKpiSyncDto` from mock by project id.
  2. Trace link → `/pms/projects/:id/tasks?highlight={taskId}` ([3.1.3.3](./3.1.3.3-sub-task-creation.md)).
  3. Error rows expandable via `Collapsible` showing error message mock.
  4. Poll or event mock updates Last synced on task changes.

---

### Drawer: Sync Trace Detail — `Sheet`

* **Objective:** Show contributing tasks and events for one KPI metric.

* **UI Elements & Field-Level Specs:**
  - KPI header name/code.
  - List of `SourceTaskIds` with progress contribution breakdown (mock).
  - Link each task to task detail.

* **Functional Logic:**
  - Opened from mapping table Trace action.

## Mock Data Shape

interface ProjectKpiSyncDto {
  ProjectId: string
  Mappings: {
    KpiId: string

}
