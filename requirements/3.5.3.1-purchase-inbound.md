## PDA Screens (`/wms/pda/inbound/purchase`)

### Step 1: Task list

Cards: delivery notice #, supplier, line count, due. Real-time mock refresh badge.

### Step 2: Arrival verification

1. Scan delivery note barcode (`ScanField`)
2. Per line: scan material barcode
3. Enter received qty (large numeric keypad style)
4. **Variance alert** if qty ≠ expected — `Alert` destructive, requires supervisor override note

### Step 3: Inspection request (auto)

On successful verification — success screen: "IQC request generated" + materials moved to inspection-pending zone (mock).

### Step 4: Receive inspection result

| IQC result | Next action UI |
|------------|----------------|
| Pass | Put-away tasks unlocked |
| Fail | Return / isolation task cards |

### Step 5: Put-away

1. Show recommended location chip (from put-away strategy)
2. Scan material barcode
3. Scan location barcode
4. Confirm qty → complete line

### Step 6: Complete

Summary + inbound ticket # (mock). Sync badges animate to Synced.
