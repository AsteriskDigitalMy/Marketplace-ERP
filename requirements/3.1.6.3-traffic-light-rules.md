# 3.1.6.3 Global Traffic Light Status Color Rule Configuration — Frontend Requirements

**Source:** F-DASH-003 — Section 3.1.6 KPI Visualization Monitoring Dashboard  
**Route:** `/pms/settings/traffic-lights`  
**Prerequisites:** User has system global parameter configuration permission (mock role gate)

---

## Functional Summary

Uniformly customize color threshold standards for progress, achievement rates, and performance levels across the system. Saved rules sync to all dashboards and lists.

**Input:** Rule classification, numerical threshold ranges, display colors.

---

## Rule Categories

| Category | Example metrics | Default bands |
|----------|-----------------|---------------|
| Progress achievement | KPI completion %, project progress | Green ≥ 90%, Yellow 70–89%, Red < 70% |
| Project delay | Days behind schedule | Green on time, Yellow 1–7 days, Red > 7 days |
| Performance level | Appraisal score | Green A/B, Yellow C, Red D |

---

## Admin UI (`/pms/settings/traffic-lights`)

### Rule list

Table columns: Category, Metric scope, Green range, Yellow range, Red range, Last updated, Actions (Edit).

Filter by category. Empty state with "Add rule" CTA.

### Rule editor (dialog or side panel)

| Field | Validation |
|-------|------------|
| Category | Required select |
| Metric scope | Optional multi-select (KPI names, modules) |
| Green — min / max | Required numbers; min ≤ max |
| Yellow — min / max | Required; no overlap with green/red |
| Red — min / max | Required |
| Preview swatches | Live sample badges using entered ranges |

**Overlap validation:** inline errors if ranges intersect.

### Save behavior

- Save → toast "Rules saved — dashboards will update"
- Mock: update in-memory rule store; all cockpit cards, BI widgets, and list status badges re-read rules on next render
- Unsaved changes warning on navigate away
