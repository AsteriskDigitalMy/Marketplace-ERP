# 3.1.9.2 Implementation & Execution Tracking of Improvement Proposals — Frontend Requirements

**Source:** F-PDCA-002 — Section 3.1.9 PDCA Improvement Closed-Loop Management  
**Routes:** `/pms/pdca/proposals/:id/execution`, `/pms/pdca/tasks`

---

## Functional Summary

Approved proposals decompose into execution tasks with owners and deadlines. Assignees update progress in real time; management monitors the full rectification lifecycle.

**Input:** Proposal ID, execution steps, responsible persons, planned completion deadlines.  
**Prerequisites:** Proposal reviewed and approved.

---

## Management — Task Decomposition (`/pms/pdca/proposals/:id/execution`)

Visible when proposal status is `Approved` (management permission).

### Execution steps editor

Dynamic rows:

| Field | Required |
|-------|----------|
| Step description | Yes |
| Responsible person | Yes (employee picker) |
| Planned deadline | Yes (date) |
| Order | Auto / drag reorder |

**"Create execution tasks"** → generates individual to-do per assignee; status → `In execution`.

---

## Assignee To-Do (`/pms/pdca/tasks`)

Personal task list: Proposal title, step description, deadline, progress %, Status.

| Status | Meaning |
|--------|---------|
| Not started | Assigned, no updates |
| In progress | Progress < 100% |
| Completed | Marked done |
| Overdue | Past deadline |

### Progress update (assignee)

| Field | Required |
|-------|----------|
| Progress % | 0–100 slider or input |
| Progress notes | Yes if < 100% on save |
| Completion evidence | Optional mock attachment |

Submit → updates proposal aggregate progress bar.

---

## Management Monitor View

On proposal execution page:

- Gantt-style or stepper timeline (mock)
- Per-step: owner, deadline, progress, overdue flag
- Overall proposal progress = average or weighted of steps

Alert badge when any step overdue (link to [alert module](./3.1.7.1-alert-rule-configuration.md) optional mock).
