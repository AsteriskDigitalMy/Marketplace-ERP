# 3.1.4.1 Automatic Push of Periodic Data-Filling Tasks — Frontend Requirements

**Source:** F-DATA-001 — Section 3.1.4 Data Collection and Filling  
**Route:** `/pms/data-collection/tasks` (admin), `/pms/data-collection/my-tasks` (assignee)

---

## Functional Summary

System issues periodic filling tasks per indicator rules and positions. Dual reminder (pop-up + message). Overdue triggers alert and log.

**Prerequisites:** KPI set to manual filling; preset filling time reached (mock scheduler).

---

## Admin — Filling Rules (`/pms/data-collection/rules`)

Configure: indicator, cycle, target positions, template binding, due offset.

---

## Employee To-Do (`/pms/data-collection/my-tasks`)

| Column | Notes |
|--------|-------|
| Task name | Indicator + period |
| Due date | Countdown badge |
| Status | Pending / Submitted / Overdue |

**On login:** in-station pop-up for due tasks (dismissible; respect session once-per-day mock).

Overdue → red badge + alert record link ([3.1.7.1](./3.1.7.1-alert-rule-configuration.md) optional).
