# 3.5.2.3 Warehouse Strategy Configuration — Frontend Requirements

**Source:** Section 3.5.2.3  
**Purpose:** Automated rules for put-away, allocation, counting, and alerts.

---

## Screens

### Strategy hub (`/wms/strategies`)

**Card grid** linking to four strategy editors:

| Strategy | Route | Description |
|----------|-------|-------------|
| Put-away | `/wms/strategies/putaway` | Location recommendation on inbound |
| Allocation | `/wms/strategies/allocation` | Pick location/batch on outbound |
| Counting | `/wms/strategies/counting` | Cycle, movement-triggered, periodic |
| Alerts | `/wms/strategies/alerts` | Threshold, expiry, stagnant, location |

Each card shows active rule count + last modified.

---

### 1. Put-away strategy (`/wms/strategies/putaway`)

**Priority-ordered rule list** (drag to reorder).

| Rule | Parameters |
|------|------------|
| Centralized storage (same material) | Enabled toggle |
| ABC classification zones | Map A→zone, B→zone, C→zone |
| Turnover frequency | High-turnover → near dispatch zone |

**Add rule** dropdown from template library.

**Test panel:** input material code → mock recommended location output.

---

### 2. Allocation strategy (`/wms/strategies/allocation`)

**Radio default + ordered fallbacks:**

| Strategy | Config |
|----------|--------|
| FIFO | Enabled |
| FEFO | Enabled, expiry window days |
| Batch priority | Batch attribute ranking table |
| Location priority | Location order list |

**Simulation:** input material + qty → mock pick list (location, batch, qty).

---

### 3. Counting strategy (`/wms/strategies/counting`)

| Strategy type | Config UI |
|---------------|-----------|
| Cycle counting | ABC frequency: A weekly, B monthly, C quarterly |
| Movement-triggered | Trigger after N transactions |
| Periodic | Cron-style schedule builder |

**Preview:** next 30 days counting schedule calendar (mock events).

---

### 4. Alert strategy (`/wms/strategies/alerts`)

**Toggle per alert type + thresholds:**

| Alert | Config fields | Notify roles |
|-------|---------------|--------------|
| Inventory upper/lower | % or absolute vs min/max | Procurement, planning, warehouse |
| Expiration | Days before expiry | QC, warehouse |
| Stagnant stock | Days without movement | Warehouse manager |
| Location exception | Over-capacity %, vacant timeout, lock timeout | Warehouse |

**Notification preview:** who receives email/in-app (mock recipient list).
