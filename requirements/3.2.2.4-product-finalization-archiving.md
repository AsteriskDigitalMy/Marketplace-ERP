### 5. Archived product detail (`/pdm/finalization/archived/:id`)

**Read-only by default** with prominent banner: "Archived product — modifications require change application."

**Tabs:**

1. **Summary** — all linked version references
2. **BOM snapshot** — read-only tree
3. **Routing snapshot** — read-only flow diagram
4. **Cost snapshot** — read-only breakdown
5. **Sync history** — mocked log of downstream pushes
6. **Change history** — links to change requests

**CTA:** `Request data change` → navigates to change management with entity pre-filled.

---

## Change Protection UX

When user attempts inline edit on archived data (any linked view):

- Block action with `Dialog`: "Archived data cannot be edited directly. Submit a change application."
- Primary button → change request form.

## Acceptance Criteria

- [ ] Queue shows only sampling-approved styles with artifact completeness indicators
- [ ] Finalization form blocks submit until all linked artifacts validated
- [ ] Approval transitions product to archived with sync status panel
- [ ] Archived registry is searchable and read-only
- [ ] Edit attempts on archived data redirect to change application flow
- [ ] All list/detail views implement four async states
