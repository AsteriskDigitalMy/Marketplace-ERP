# 3.4.6.3 Equipment Maintenance Management — Frontend Requirements

**Source:** Section 3.4.6.3  
**Purpose:** Preventive maintenance plans, execution, fault tickets, repair closure.

---

## Screens

### Maintenance plans (`/mes/equipment/maintenance/plans`)

Auto-generated from cycles — items, responsible, due date.

**Due soon** filter + reminder badges.

---

### Maintenance execution form

| Field | |
|-------|--|
| Maintenance content | Textarea |
| Spare parts replaced | Line editor |
| Hours spent | Number |
| Result | Select |
| Photos | Upload mock |

Updates equipment status on save.

---

### Fault report (Pad `/mes/pad/equipment/fault`)

| Field | Required |
|-------|----------|
| Equipment # | Scan |
| Phenomenon | Yes |
| Reporter | Auto |
| Time | Auto |

Pushes to equipment admin queue.

---

### Repair ticket workflow

| Status | Actions |
|--------|---------|
| Open | Assign technician |
| In repair | Log repair content, cause, parts, duration |
| Pending confirm | Reporter confirms closure |
| Closed | Archive |

---

### Maintenance archive

Search by equipment, time, type — permanent read-only records.

---
