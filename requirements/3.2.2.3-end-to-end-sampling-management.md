# 3.2.2.3 End-to-End Sampling Management (ePDM Core) — Frontend Requirements

**Source:** Section 3.2.2.3  
**Purpose:** Closed-loop UI for sampling — notification → material verification → requisition → production → review → cost pricing handoff.

---

## Pipeline Overview Screen (`/pdm/sampling`)

**Kanban or stepper dashboard** showing sampling jobs by stage:

1. Notification issued
2. Technical material verification
3. Material requisition
4. Sample production
5. Garment / sample review
6. Cost pricing handoff

Each card: style number, quantity, delivery date, lead, overdue flag.

**Filters:** stage, lead, delivery date range, overdue.

---

## Stage 1: Sampling Notification

### Screen: Notification list + detail

**Auto-generation UI:** when design review passes, show toast + new row in sampling queue (mock event).

**Notification form (read-only after generation, editable only in draft):**

| Field | Source |
|-------|--------|
| Sample style number | From approved design |
| Quantity | Number input |
| Delivery date | Date picker |
| Process requirements | Textarea |
| Sampling lead | User picker |

**Action:** `Issue notification` → pushes to sample production department (mock status change + toast).

---

## Stage 2: Technical Material Verification

### Screen: Sample BOM editor (`/pdm/sampling/:id/bom`)

**Table editor** for bill of materials:

| Column | Editable |
|--------|----------|
| Material code | Yes (combobox from material master, mocked) |
| Material name | Auto-filled |
| Specification | Auto-filled |
| Unit | Auto-filled |
| Unit consumption | Number |
| Wastage rate (%) | Number |
| Required qty | Calculated (consumption × (1 + wastage) × sample qty) |

**Actions:**

- `Add line`, `Remove line`, `Import from template` (mock)
- `Verify inventory` — triggers mock check; shows per-line stock status:

| Stock status | Badge |
|--------------|-------|
| Sufficient | Success |
| Partial | Warning |
| Shortage | Destructive |

**Inventory panel (mock):** on-hand, in-transit, shortage qty per material.

## Stage 3: Sample Material Requisition

### Screen: Requisition detail (`/pdm/sampling/:id/requisition`)

**Auto-generated requisition** from sample BOM (read-only header + editable notes).

| Field | Display |
|-------|---------|

- Button `Push to WMS` → status `Sent`

## Stage 4: Sample Production Allocation

### Screen: Production tasks (`/pdm/sampling/:id/production`)

**Task assignment form:**

| Field | Type |
|-------|------|
| Sample group | Select |
| Assigned personnel | Multi-select |
| Requirements | Textarea |
| Completion deadline | Date picker |

**Task list for assignees:**

- Card per task with requirements, deadline, status.
- **Pad-optimized read-only view** (responsive breakpoint ≤ 1024px): large text, full-width cards — mock route `/pdm/sampling/:id/production/pad`.

**Statuses:** Assigned → In progress → Complete.

---

## Stage 5: Garment / Sample Review

### Screen: Multi-department review (`/pdm/sampling/:id/review`)

**Review panel per department:**

| Department | Reviewer (mock) | Status | Comments |
|------------|-----------------|--------|----------|
| Technical | | Pending / Pass / Fail | Textarea |
| Production | | | |
| Quality | | | |
| Sales | | | |
| Customer | | | |

**Overall outcome logic (frontend):**

- Any `Fail` → overall `Failed` → show `Send back for modification` CTA.
- All `Pass` → overall `Passed` → unlock cost pricing handoff.

**Fail path:**

- `Request re-sampling` dialog with modification requirements summary.
- Returns job to production stage (mock status rollback).

**Pass path:**

- Approval banner + navigate CTA to cost pricing section.

---

## Stage 6: Review Comments Archiving

### Screen: Archive tab on sampling detail

**Permanent record** bound to style number:

- All review opinions (timestamped, by department)
- Modification records (linked to re-sample iterations)
- Sampling version history (`Sample v1`, `Sample v2`, …)

**UI:** chronological `Timeline` component with expandable entries and attachment links.

---

## Sampling Detail Shell (`/pdm/sampling/:id`)

**Persistent header:** style number, current stage stepper, delivery date, overdue alert.

**Tabs:** Overview, BOM, Requisition, Production, Review, Archive.

Stage tabs locked until prior stage complete (visual lock icon + tooltip).
