# 3.1.8.1 Multi-Scenario Appraisal Scheme Custom Configuration — Frontend Requirements

**Source:** F-PERF-001 — Section 3.1.8 Performance Appraisal & Scoring  
**Route:** `/pms/appraisal/schemes`

---

## Functional Summary

Configure differentiated performance appraisal schemes by department, role, and project type. Schemes bind KPI indicators, weights, scoring criteria, and grade division rules.

**Prerequisites:** Performance appraisal scheme configuration permission.

---

## Scheme List

Columns: Scheme name, Applicable scope (dept/role/project type), Cycle, Indicator count, Status (Draft / Active / Archived), Actions.

Actions: Edit, Copy, Archive. "New scheme" primary button.

---

## Scheme Editor (`/pms/appraisal/schemes/new`, `/pms/appraisal/schemes/:id`)

### Basic info

| Field | Required |
|-------|----------|
| Scheme name | Yes |
| Applicable departments | Multi-select |
| Applicable roles | Multi-select |
| Project types | Optional multi-select |
| Appraisal cycle | Monthly / Quarterly / Semi-annual / Annual |

### Indicator & weights

- Picker from KPI indicator library (mock searchable list)
- Selected indicators table: name, weight %, remove
- **Validation:** weights must sum to 100% — inline error if not

### Scoring & grade rules

| Grade | Score range (min–max) | Color (from traffic-light rules) |
|-------|----------------------|----------------------------------|
| A | e.g. 90–100 | Green |
| B | e.g. 80–89 | Green |
| C | e.g. 70–79 | Yellow |
| D | < 70 | Red |

Editable ranges; no overlap validation.

### Actions

- Save as draft
- Activate (takes effect for next cycle — mock toast)
- Copy scheme → opens new draft with duplicated fields
