# 3.1.8.1 Multi-Scenario Appraisal Scheme Custom Configuration — Frontend Requirements

**Source:** F-PERF-001 — Section 3.1.8 Performance Appraisal & Scoring  
**Route:** `/pms/appraisal/schemes`  
**Workflow:** Entry point for appraisal workflow — feeds [3.1.8.2](./3.1.8.2-automatic-scoring-grading.md) → [3.1.8.3](./3.1.8.3-preliminary-review-list.md) → [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md) → [3.1.8.5](./3.1.8.5-hr-rectification.md) / [3.1.8.7](./3.1.8.7-executive-final-review.md) → [3.1.8.6](./3.1.8.6-secondary-confirmation.md)

---

## Functional Summary

Configure differentiated performance appraisal schemes by department, role, and project type. Schemes bind KPI indicators, weights, scoring criteria, and grade division rules.

**Prerequisites:** Performance appraisal scheme configuration permission.

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Goals | Pain Points |
|---------|-------|-------------|
| **HR / Performance Administrator** | Define schemes per dept/role; activate before each cycle | Manual spreadsheets; inconsistent weights across teams |
| **Auditor** | Verify scheme rules match policy before generation | No visibility into which scheme applies to which scope |
| **Department Manager** (read-only) | Understand how team members will be scored | Unclear indicator weights and grade cutoffs |

### Core Capabilities (FR-001…)

| ID | Capability | Priority |
|----|------------|----------|
| FR-001 | List appraisal schemes with scope, cycle, indicator count, and status | Must |
| FR-002 | Create and edit scheme basic info (name, departments, roles, project types, cycle) | Must |
| FR-003 | Pick KPI indicators from library and assign weights summing to 100% | Must |
| FR-004 | Configure grade score ranges (A/B/C/D) with traffic-light colors | Must |
| FR-005 | Save as draft or activate scheme for next appraisal cycle | Must |
| FR-006 | Copy existing scheme to new draft | Should |
| FR-007 | Archive inactive schemes (read-only thereafter) | Should |
| FR-008 | Validate weight sum and grade range overlap before save/activate | Must |

### Edge Cases & Alternative Paths

- **Weight sum ≠ 100%:** Inline error on indicators table; Save and Activate disabled until resolved.
- **Overlapping grade ranges:** Warning banner; block Activate; allow Save as draft with acknowledgment.
- **No indicators selected:** Block save; empty-state CTA to open KPI picker.
- **Activate while another active scheme overlaps scope:** Mock conflict modal — list conflicting schemes; user must archive or narrow scope.
- **Edit active scheme:** Fields read-only except Copy; prompt to copy-to-draft for changes.
- **Archived scheme:** No edit; Copy still available.
- **Empty list:** Empty state with "New scheme" primary CTA and link to KPI indicator library docs.
- **Permission denied:** Full-page 403 with contact-admin message.

## Phase 2: Screen-by-Screen Functional Specifications

### Screen: Scheme List (`/pms/appraisal/schemes`)

* **Objective:** Browse, filter, and manage all appraisal schemes; launch create/edit/copy/archive actions.
* **UI Elements & Field-Level Specs:**
  * Page title: "Appraisal Schemes"
  * Primary button: **New scheme** → `/pms/appraisal/schemes/new`
  * Filters: Status (Draft / Active / Archived), Cycle, Department (multi), Search by name
  * Data table columns: Scheme name (link to edit if draft), Applicable scope (dept/role/project chips), Cycle, Indicator count, Status badge, Actions (Edit | Copy | Archive)
  * Status badges: Draft (neutral), Active (green), Archived (gray)
  * Row actions: Edit (draft only), Copy (all), Archive (active/draft; confirm modal)
  * Four async states: skeleton table → empty → error with retry → data
* **Functional Logic:**
  * Fetch schemes on mount; refetch after filter change (debounced search 300ms)
  * Archive sets `Status: archived`; toast confirmation
  * Copy navigates to `/pms/appraisal/schemes/new?copyFrom=:id` with duplicated fields

### Screen: Scheme Editor — Create (`/pms/appraisal/schemes/new`)

* **Objective:** Define a new appraisal scheme from scratch or from copy source.
* **UI Elements & Field-Level Specs:**
  * Breadcrumb: Schemes → New scheme
  * **Basic info** card: Scheme name (required, max 100), Applicable departments (multi-select), Applicable roles (multi-select), Project types (optional multi-select), Appraisal cycle (radio: Monthly / Quarterly / Semi-annual / Annual)
  * **Indicators & weights** card: "Add indicators" opens KPI library picker modal; selected table — KPI name, Weight % (numeric 0–100, step 1), Remove row; footer shows running total with green (100%) or red (≠100%)
  * **Scoring & grade rules** card: editable table — Grade (A/B/C/D fixed), Min score, Max score, Color preview (from [traffic-light rules](../3.1.6.3-traffic-light-rules.md))
  * Sticky footer: **Save as draft**, **Activate** (primary, disabled until validations pass), **Cancel** → list with unsaved-changes confirm
* **Functional Logic:**
  * On `?copyFrom=`, pre-fill all sections from source scheme; append " (Copy)" to name
  * Weight validation real-time; grade overlap check on blur
  * Activate: success toast "Effective next cycle" + redirect to list with Active badge
  * Save draft: allow incomplete indicators; persist `Status: draft`

### Screen: Scheme Editor — Edit (`/pms/appraisal/schemes/:id`)

* **Objective:** Modify draft scheme or view active/archived scheme in read-only mode.
* **UI Elements & Field-Level Specs:** Same layout as Create; active/archived shows read-only fields + banner "Active schemes cannot be edited — use Copy"
* **Functional Logic:**
  * Load scheme by `:id`; 404 if not found
  * Draft: full edit + Save/Activate
  * Active/Archived: disable inputs; show Copy and Back only

### Modal: KPI Indicator Library Picker

* **Objective:** Search and multi-select KPI indicators from mock library.
* **UI Elements & Field-Level Specs:**
  * Search input (name/code), category filter
  * Checkbox list: KPI name, code, category
  * Selected count chip; **Add selected** / **Cancel**
* **Functional Logic:**
  * Exclude already-added KPIs from list
  * Append selected rows with default weight 0; focus first new weight field

### Modal: Scope Conflict Warning (Activate)

* **Objective:** Inform admin of overlapping active schemes before activation.
* **UI Elements & Field-Level Specs:** List of conflicting scheme names and overlapping scope; **Review conflicts** / **Cancel activate**
* **Functional Logic:** Block activation until conflicts resolved (mock: user must archive conflicting scheme first)

### Modal: Archive Confirmation

* **Objective:** Confirm archival of scheme.
* **UI Elements:** Warning text; scheme name; **Archive** / **Cancel**
* **Functional Logic:** Set status archived; disable for [3.1.8.2](./3.1.8.2-automatic-scoring-grading.md) cycle binding if was active

## Phase 3: Screen Transition & User Flow Mapping

| # | Trigger | Source | Destination | Conditional paths |
|---|---------|--------|-------------|-------------------|
| 1 | Page load | App nav / deep link | Scheme List | Permission gate → 403 |
| 2 | Click **New scheme** | Scheme List | Scheme Editor (new) | — |
| 3 | Click scheme name (draft) | Scheme List | Scheme Editor (`:id`) | Active/archived → read-only view |
| 4 | Click **Edit** | Scheme List row | Scheme Editor (`:id`) | Draft only |
| 5 | Click **Copy** | Scheme List row | Scheme Editor (new, `?copyFrom=`) | — |
| 6 | Click **Add indicators** | Scheme Editor | KPI Picker modal | Close → return to editor |
| 7 | **Save as draft** success | Scheme Editor | Scheme List | Stay on editor optional (toast) |
| 8 | **Activate** success | Scheme Editor | Scheme List | Conflict → Scope Conflict modal |
| 9 | **Cancel** with dirty form | Scheme Editor | Scheme List | Unsaved changes confirm |
| 10 | Archive confirm | Scheme List | Scheme List (refetch) | — |
| 11 | Post-activate | Scheme List | [3.1.8.2](./3.1.8.2-automatic-scoring-grading.md) cycles | Admin may navigate manually |
