# 3.3.2.5 End-to-End Order Tracking — Frontend Requirements

**Source:** Section 3.3.2.5

---

## Order status model

`Pending review` → `Effective` → `In production` → `In warehouse` → `Partially shipped` → `Shipped` → `Closed` | `Cancelled`

---

### Tracking detail (`/scm/orders/:id/tracking`)

**Progress panels (mock links):**

| Domain | Metrics |
|--------|---------|
| Production plan | Plan #, status |
| Work orders | MES progress % |
| Procurement | PO lines received % |
| FG inventory | On-hand qty |
| Shipment | Shipped / unshipped |
| Invoicing | Invoiced / paid |

---

### Gantt visualization (`/scm/orders/:id/tracking/gantt`)

Order workflow nodes with completion % and dates — color by status (on-time / delayed).

---

### Alerts

Delay, material shortage, production anomaly → alert chips on order header + SCM dashboard.

---

### Customer portal (`/portal/orders/:id`)

Read-only subset: status, production %, estimated ship date (no internal costs).

---
