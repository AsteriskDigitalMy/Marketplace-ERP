# 3.5.4.3 Miscellaneous Outbound Management — Frontend Requirements

**Source:** Section 3.5.4.3  
**Types:** Outsourced material issuance, warehouse transfers (outbound leg), purchase returns, inventory shortages, scrap disposal.

---

### Create notice (`/wms/outbound/misc/new`)

Same pattern as miscellaneous inbound:

- Type, reference, reason, lines
- Formal approval workflow required
- Allocation strategy preview per line (mock FIFO/FEFO picks)

---

## PDA Execution (`/wms/pda/outbound/misc/:id`)

Approved notices only.

Per line:

1. Scan location (per allocation recommendation)
2. Scan material + batch
3. Enter qty
4. Log operator + timestamp

**Complete** → outbound document by type + ledger update + module sync (mock).
