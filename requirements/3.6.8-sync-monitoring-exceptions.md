## Screens

### Sync monitoring dashboard (`/sap` — primary widgets)

Real-time (mock polling):

| Widget | Content |
|--------|---------|
| Success rate gauge | 24h % — alert if < 99% |
| Synced / failed counts | By hour chart |
| Interface health | Online/offline per protocol |
| Open exceptions | Count with link |

---

### Sync log (`/sap/logs`)

**Permanent, searchable log:**

| Timestamp | Doc type | Doc # | Direction | Status | SAP message | Operator |

Filters: time range, doc type, status, doc #.

Expand row → full payload JSON (formatted `Sheet`).

Export CSV mock.

---

### Exception queue (`/sap/exceptions`)

Failed after 3 auto-retries:

| Doc # | Type | Error | Owner | Locked | Actions |

**Locked** — financial processing blocked for document (banner on source ops screen).

Actions:

- View error detail
- Edit document (link to source module if applicable)
- **Manual re-sync** — progress → unlock on success

Retry history sub-table: attempt 1–3 timestamps and messages.

---

### Month-end close guard (`/sap/month-end`)

**Pre-close validation:**

Scan month for unsynced or failed documents.

| Check | Result |
|-------|--------|
| All P2P synced | ✓/✗ |
| All O2C synced | ✓/✗ |
| All CO settlements | ✓/✗ |
| All inventory postings | ✓/✗ |

If any ✗ → **block close** with list of blocking documents + link to exceptions.

`Proceed with month-end close` only when all pass (mock confirm).

---

### Consistency check (`/sap/reconciliation/consistency`)

Periodic ops vs SAP comparison across modules — generates variance report (links to P2P/O2C/inventory reconciliations).

Schedule + last run result.

## Mock Data Shape

```ts
interface SyncLogEntryDto {
  Id: string
  Timestamp: string
  DocumentType: string
  DocumentNumber: string
  Direction: 'ops_to_sap' | 'sap_to_ops'
  Status: 'success' | 'failed' | 'retrying'
  Payload: Record<string, unknown>
  ReturnMessage: string
  Operator: string | null
  RetryAttempts: number
}

interface SyncExceptionDto {
  Id: string
  DocumentType: string
  DocumentNumber: string
  ErrorMessage: string
  DetailedError: string
  Owner: string
  IsLocked: boolean
  RetryAttempts: number
  RetryHistory: { Attempt: number; Timestamp: string; Message: string }[]
  CanManualResync: boolean
}
```

Mocks: `src/mocks/sap/sync-monitoring.ts`.

## Acceptance Criteria

- [ ] Dashboard shows success rate with < 99% alert
- [ ] Sync log searchable with expandable payload
- [ ] Exception queue with lock status and manual re-sync
- [ ] Retry history visible per failed document
- [ ] Month-end close blocked when unsynced docs exist
- [ ] Consistency check report links to module reconciliations
- [ ] Four async states on all monitoring views
