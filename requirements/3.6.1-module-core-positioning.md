# 3.6.1 Core Module Positioning — Frontend Requirements

**Source:** Section 3.6.1 — SAP Financial System Integration  
**Module role:** Deep integration between SCM/MES/WMS and SAP — AP, AR, production costing, inventory valuation. Auto financial vouchers; business-financial alignment; ledger matches physical stock.

---

## Overview

This module is primarily an **integration control plane** in the frontend — not a replacement for SAP FI/CO screens. Users monitor sync health, trigger re-syncs, view reconciliation, and see SAP voucher status on operational documents across SCM, MES, and WMS.

## Module Dashboard (`/sap`)

### KPI cards

| Metric | |
|--------|--|
| Sync success rate (24h) | Target ≥ 99% |
| Failed documents (open) | Link to exceptions |
| Pending vouchers | By process P2P/O2C/CO/MM |
| Last consistency check | Pass / variances |
| Interfaces online | REST / OData / RFC status (mock) |

### Process health strip

Four tiles: **AP** | **AR** | **Production costing** | **Inventory valuation** — each with synced/failed counts.

### Recent sync activity

Feed of last 20 sync events with status chips.

## Cross-Module SAP Status Component

Reusable `SapSyncBadge` embedded on operational screens:

| Status | UI |
|--------|-----|
| Pending | muted |
| Syncing | spinner |
| Synced | success |
| Failed | destructive + retry link |
| Locked | warning (exception) |

Used on: PO, goods receipt, invoice, payment, sales order, delivery, WO closure, inventory docs (mock placement in SCM/WMS/MES UIs).
