# 3.1.3.6 One-Click Reporting of Project Execution Issues — Frontend Requirements

**Source:** F-PROJ-006 — Section 3.1.3 Project Management  
**Route:** Embedded in `/pms/tasks/my` progress form ([3.1.3.4](./3.1.3.4-project-progress-updates.md))  
**Related:** [3.1.3.4](./3.1.3.4-project-progress-updates.md), [3.1.3.7](./3.1.3.7-issue-closed-loop.md), [3.1.3.3](./3.1.3.3-sub-task-creation.md)

## Functional Summary

Report on-site difficulties and resource needs during progress updates. Task marked at risk; project leader notified in real time.

**Prerequisites:** Sub-task `In Progress`.

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Role | Goals | Pain Points |
|---------|------|-------|-------------|
| **Task Executor** | Reports blockers during progress update | One-click issue capture without separate form | Issues lost if not filed during update |
| **Project Leader** | Monitors risk | Immediate visibility of at-risk tasks | Noise from minor issues |
| **Resource Manager** | Allocates personnel/equipment | Understand resource type needed | Vague issue descriptions |

### Core Capabilities (FR-xxx)

| ID | Capability | Priority |
|----|------------|----------|
| FR-001 | Issue section embedded in progress update form [3.1.3.4](./3.1.3.4-project-progress-updates.md) | Must |
| FR-002 | Required issue description and resource type picker | Must |
| FR-003 | "Flag task at risk" checkbox default on when issue filled | Must |
| FR-004 | Create `ProjectIssueDto` on submit with progress | Must |
| FR-005 | Task risk `Badge` on lists and project views | Must |
| FR-006 | Real-time leader notification toast (mock) | Must |
| FR-007 | Leader risk feed banner on project detail → [3.1.3.7](./3.1.3.7-issue-closed-loop.md) | Must |

### Edge Cases & Alternative Paths

- **Issue fields partially filled:** All issue fields required together or none (toggle expand).
- **Risk flag off with issue text:** Warn "Issue will still be recorded".
- **Progress submit without issue:** Normal flow; no issue record.
- **Multiple issues same task:** Allow; each progress update can add new issue.
- **Leader is also executor:** Notification still fires (mock self-notify).

## Phase 2: Screen-by-Screen Functional Specifications

### Section: Issue Report (embedded) — `/pms/tasks/my` progress form

* **Objective:** Capture execution issues alongside progress in a single submit.

* **UI Elements & Field-Level Specs:**
  - Collapsible `Card` or `Accordion` "Report execution issue" below work description.
  - **Issue description** — `Textarea`, required when section expanded, max 2000 chars.
  - **Required resource type** — `Select`: personnel | equipment | material | budget | other.
  - **Flag task at risk** — `Checkbox`, default checked when description non-empty.
  - Optional `Badge` preview "At risk" when checkbox on.
  - Helper `Alert`: issues route to leader for assignment ([3.1.3.7](./3.1.3.7-issue-closed-loop.md)).

* **Functional Logic:**
  1. Expanding section enables validation on issue fields.
  2. On progress submit with issue: create `ProjectIssueDto` status `open`; link `TaskId`.
  3. Set task risk flag; show `Badge` on `/pms/tasks/my` and project task table [3.1.3.3](./3.1.3.3-sub-task-creation.md).
  4. Mock toast to project leader: "New issue on {task}".

---

### Banner: Leader Risk Feed — `/pms/projects/:id` (project detail)

* **Objective:** Surface open issues to project leader at a glance.

* **UI Elements & Field-Level Specs:**
  - `Alert` variant destructive/warning when open issues > 0.
  - Text: "{N} open issues require attention".
  - `Button` link "View issues" → `/pms/projects/:id/issues` ([3.1.3.7](./3.1.3.7-issue-closed-loop.md)).
  - Mini list (max 3): task name, resource type, reported at.

* **Functional Logic:**
  - Visible to project leader role only.
  - Dismissible per session (local state); reappears on new issue.

---

### Indicator: Task Risk Badge — task lists

* **Objective:** Visual at-risk marker across surfaces.

* **UI Elements & Field-Level Specs:** `Badge` variant destructive "At risk" with `AlertTriangle` icon.

* **Functional Logic:** Shown on task row when risk flag true and issue not `closed`.
