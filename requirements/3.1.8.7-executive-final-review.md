# 3.1.8.7 Executive Final Performance Review — Frontend Requirements

**Source:** F-PERF-007 — Section 3.1.8 Performance Appraisal & Scoring  
**Route:** `/pms/appraisal/final`  
**Workflow:** Terminal approval for A/B from [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md) and post-HR path from [3.1.8.6](./3.1.8.6-secondary-confirmation.md); full history spans [3.1.8.1](./3.1.8.1-appraisal-scheme-configuration.md) → [3.1.8.2](./3.1.8.2-automatic-scoring-grading.md) → [3.1.8.3](./3.1.8.3-preliminary-review-list.md) → [3.1.8.5](./3.1.8.5-hr-rectification.md)

---

## Functional Summary

Corporate executive (CFO) completes final approval of employee performance. On approval, results publish network-wide; rejection returns to Auditor for re-verification.

**Input:** Employee ID, final approval opinions.  
**Prerequisites:** Status "Pending final review"; highest final review permission.

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Goals | Pain Points |
|---------|-------|-------------|
| **CFO / Executive** | Final sign-off on performance; publish cycle results | Too much detail without process context |
| **Auditor** | Receive rejections with clear rationale for rework | Rejected items lost in queue |
| **Employee** (downstream) | See published result in personal center | Delayed visibility after approval |

### Core Capabilities (FR-001…)

| ID | Capability | Priority |
|----|------------|----------|
| FR-001 | Final review queue with routing source (direct A/B vs post-HR) | Must |
| FR-002 | Non-actionable C/D HR-path rows per [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md) policy | Must |
| FR-003 | Full-process appraisal history on detail page | Must |
| FR-004 | Approve / Reject with opinion (required on reject) | Must |
| FR-005 | Approve → `Published`; mock personal center sync | Must |
| FR-006 | Reject → `Returned to auditor`; flag [3.1.8.3](./3.1.8.3-preliminary-review-list.md) for adjustment | Must |
| FR-007 | Bulk approve with confirmation (actionable rows only) | Should |
| FR-008 | Cycle-level publish when all employees approved | Must |

### Edge Cases & Alternative Paths

- **C/D HR-path row on CFO list:** Grayed, tooltip "HR queue — not actionable"; no approve/reject.
- **Bulk approve includes non-actionable rows:** Excluded automatically; count in confirm modal.
- **Reject without opinion:** Block submit.
- **Approve with optional opinion:** Allowed.
- **Last employee approved in cycle:** Trigger cycle publish flow + modal.
- **Employee with open dispute (mock flag):** Excluded from bulk approve.
- **Partial cycle — some still pending HR [3.1.8.5](./3.1.8.5-hr-rectification.md):** Cycle publish blocked; show remaining counts.
- **Rejection loop:** Auditor adjusts via preliminary flow [3.1.8.3](./3.1.8.3-preliminary-review-list.md) / [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md).

## Phase 2: Screen-by-Screen Functional Specifications

### Screen: Final Review Queue (`/pms/appraisal/final`)

* **Objective:** CFO reviews and approves/rejects pending final performance results.
* **UI Elements & Field-Level Specs:**
  * Page title: "Executive Final Review"
  * Filters: Department, Grade, Routing source (Direct A/B / Post-HR secondary)
  * Table columns: Checkbox (actionable only), Employee, Department, Confirmed grade, Score, Routed from (badge), Status, Actions (**Review**)
  * Non-actionable rows: disabled checkbox, muted row, tooltip per [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md)
  * Toolbar: **Bulk approve** (enabled when ≥1 actionable selected)
  * Cycle selector (when multiple open cycles)
  * Four async states
* **Functional Logic:**
  * Bulk approve opens confirmation with count; mock sequential approve
  * Dispute-flagged rows excluded from bulk

### Screen: Review Detail (`/pms/appraisal/final/:employeeId`)

* **Objective:** Executive decision with full appraisal lifecycle context.
* **UI Elements & Field-Level Specs:**
  * Breadcrumb: Final Queue → {Employee name}
  * **Full-process record** (accordion/tabs):
    1. Auto scoring & indicators — [3.1.8.2](./3.1.8.2-automatic-scoring-grading.md)
    2. Preliminary review & shunting — [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md)
    3. HR assistance (if applicable) — [3.1.8.5](./3.1.8.5-hr-rectification.md)
    4. Secondary confirmation (if applicable) — [3.1.8.6](./3.1.8.6-secondary-confirmation.md)
  * **Executive decision** card:
    * Decision (radio: Approve / Reject)
    * Final approval opinion (textarea; required if Reject)
    * **Submit decision** (primary), **Back to queue**
  * Read-only mode when `IsActionableForCfo === false`
* **Functional Logic:**
  * Approve → `published`; mock employee personal center banner
  * Reject → `returned_to_auditor`; preliminary queue flag mock
  * Check if cycle complete → trigger publish summary

### Modal: Bulk Approve Confirmation

* **Objective:** Confirm mass approval of selected actionable employees.
* **UI Elements:** Count, list of names (scrollable), exclusion note for non-actionable/dispute, **Approve all** / **Cancel**
* **Functional Logic:** On confirm, batch mock approve; refresh queue

### Modal: Cycle Publish — Network Wide

* **Objective:** Celebrate and confirm cycle-level publication when all approvals done.
* **UI Elements:** Cycle label, employee count, **Publish network-wide** (mock), link to employee self-service view, link to [3.1.10.1](./3.1.10.1-standard-fixed-reports.md) quarterly appraisal report
* **Functional Logic:** Set cycle status `Published` from [3.1.8.2](./3.1.8.2-automatic-scoring-grading.md) context

### Modal: Reject Confirmation

* **Objective:** Confirm rejection returns case to Auditor.
* **UI Elements:** Opinion preview, **Confirm reject** / **Cancel**
* **Functional Logic:** Navigate to queue; toast notifies auditor (mock)
