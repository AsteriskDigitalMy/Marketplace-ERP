# 3.7.2.7 Execution Site Electronic Kanban — Frontend Requirements

**Source:** Section 3.7.2 — Execution Monitoring Layer  
**Routes:** `/bi/kanban/workshop`, `/bi/kanban/warehouse`

---

## Design constraints

- **Full-screen** layout — no app sidebar (kiosk mode)
- **Dark theme** default, high contrast
- **Auto-refresh** 3–10 seconds (configurable)
- Minimal decorative animation; respect `prefers-reduced-motion`
- Red highlight for lagging lines / errors (use `destructive` token, not hardcoded hex)

---

## Workshop assembly line kanban (`/bi/kanban/workshop`)

### Per-line cards (grid)

| Field | Display |
|-------|---------|
| Work order # | Large monospace |
| Product model | |
| Target output | Tabular nums |
| Actual output | Tabular nums |
| Progress | Bar % |
| Direct pass rate (FPY) | % |
| Equipment fault code | Red if active |

**Alert styling:** lagging line (actual < target pace) → red text on progress; high failure frequency → pulsing border (unless reduced motion).

Plant/workshop selector in corner (admin only).

---

## Warehouse inbound/outbound kanban (`/bi/kanban/warehouse`)

| Panel | Content |
|-------|---------|
| Today inbound | Planned / completed |
| Today outbound | Planned / completed |
| Picking queue | Queue depth per zone |
| Stock-taking progress | % complete |
| Picking errors | Warning list, red count |

Guides forklift/PDA operators — large fonts, 360px+ readable at distance.

---

## Carousel mode (`?carousel=1`)

Rotate between workshop and warehouse views every N seconds for shared TV wall.
