# 3.1.8.6 Secondary Confirmation of Performance Rectification Effectiveness — Frontend Requirements

**Source:** F-PERF-006 — Section 3.1.8 Performance Appraisal & Scoring  
**Route:** `/pms/appraisal/secondary`  
**Workflow:** Reviews output from [3.1.8.5](./3.1.8.5-hr-rectification.md) (C/D path from [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md)) → Pass may route to [3.1.8.7](./3.1.8.7-executive-final-review.md); Fail returns to [3.1.8.5](./3.1.8.5-hr-rectification.md)

---

## Functional Summary

Auditor reviews HR assistance and rectification effectiveness. Pass → route to CFO final review; fail → return to HR for re-rectification.

**Input:** Employee ID, final opinion of secondary review.  
**Prerequisites:** Employee status is "HR Processed".

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Goals | Pain Points |
|---------|-------|-------------|
| **Auditor** | Verify HR assistance quality before closing C/D path or escalating | No single view of HR + appraisal history |
| **HR Specialist** | Understand why case returned for re-rectification | Opaque rejection reasons |
| **CFO** (downstream) | Trust HR-path employees were vetted | — |

### Core Capabilities (FR-001…)

| ID | Capability | Priority |
|----|------------|----------|
| FR-001 | Secondary review queue of HR-processed employees | Must |
| FR-002 | Nav badge count of items awaiting confirmation | Should |
| FR-003 | Review detail with read-only appraisal, HR records, PDCA link | Must |
| FR-004 | Pass/Fail decision with required final opinion | Must |
| FR-005 | Pass → `pending_final_review` (CFO queue per policy) | Must |
| FR-006 | Fail → return to HR with opinion; repopulate [3.1.8.5](./3.1.8.5-hr-rectification.md) queue | Must |
| FR-007 | Confirmation dialog before submit | Must |
| FR-008 | Inline workflow diagram for HR path context | Should |

### Edge Cases & Alternative Paths

- **Pass without linked PDCA:** Valid when HR did not generate proposal.
- **Fail with empty opinion:** Block submit; opinion required on both pass and fail.
- **Linked PDCA still in execution [3.1.9.2](./3.1.9.2-pdca-execution-tracking.md):** Show status badge; auditor may still pass/fail HR assistance independently.
- **Employee already reviewed:** Queue excludes; direct URL shows read-only outcome.
- **Concurrent auditor access (mock):** Optimistic lock toast on stale submit.
- **Pass routing for grade C/D:** Mock routes to [3.1.8.7](./3.1.8.7-executive-final-review.md) as read-only or complete HR path per grade policy — document in UI help.

### Workflow Diagram

```
Preliminary (C/D) → [3.1.8.4] → HR assistance [3.1.8.5] → Secondary confirm ─┬─ Pass → [3.1.8.7] (policy-dependent)
                                                                              └─ Fail → HR re-rectification [3.1.8.5]
```

## Phase 2: Screen-by-Screen Functional Specifications

### Screen: Secondary Review Queue (`/pms/appraisal/secondary`)

* **Objective:** List employees awaiting auditor secondary confirmation after HR assistance.
* **UI Elements & Field-Level Specs:**
  * Page title: "Secondary Rectification Review"
  * Badge: awaiting count in page header and nav
  * Table: Employee, Department, Grade, HR submitted at (relative + absolute tooltip), Linked PDCA (icon link if present), Status, Actions (**Review**)
  * Inline help: collapsible workflow diagram linking [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md) → [3.1.8.5](./3.1.8.5-hr-rectification.md) → this screen
  * Empty state: "No items awaiting secondary review"
  * Four async states
* **Functional Logic:**
  * Filter `pending_secondary_review` only
  * PDCA column links to `/pms/pdca/proposals/:id` when `LinkedPdcaProposalId` set

### Screen: Review Detail (`/pms/appraisal/secondary/:employeeId`)

* **Objective:** Auditor evaluates HR rectification effectiveness and decides pass/fail.
* **UI Elements & Field-Level Specs:**
  * Breadcrumb: Secondary Queue → {Employee name}
  * **Read-only panels** (accordion or tabs):
    1. Original appraisal summary — score, grade, indicators (from [3.1.8.2](./3.1.8.2-automatic-scoring-grading.md) / [3.1.8.3](./3.1.8.3-preliminary-review-list.md))
    2. HR assistance records — full timeline from [3.1.8.5](./3.1.8.5-hr-rectification.md)
    3. Linked PDCA — status chip + link to [3.1.9.1](./3.1.9.1-pdca-proposal-submission.md) / execution [3.1.9.2](./3.1.9.2-pdca-execution-tracking.md) if in progress
  * **Auditor decision** card:
    * Secondary review result (radio: Pass / Fail, required)
    * Final opinion (textarea, required)
    * **Submit review** (primary), **Back to queue**
* **Functional Logic:**
  * Submit opens confirmation modal with decision summary
  * Pass: status → `pending_final_review`; add to [3.1.8.7](./3.1.8.7-executive-final-review.md) per policy
  * Fail: status → `returned_to_hr` / `pending_hr_processing`; HR queue repopulated with auditor opinion visible to HR

### Modal: Submit Review Confirmation

* **Objective:** Prevent accidental pass/fail submission.
* **UI Elements:** Decision label (Pass/Fail), opinion preview (truncated), **Confirm** / **Cancel**
* **Functional Logic:** On confirm, persist and navigate to queue with toast
