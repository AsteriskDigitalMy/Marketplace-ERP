# 3.5.2.2 Material Master Data Management — Frontend Requirements

**Source:** Section 3.5.2.2  
**Purpose:** Warehouse attributes for materials — barcoding, storage, stock thresholds, batch/serial/FIFO rules.

---

## Screens

### 1. Material warehouse profile list (`/wms/materials`)

| Column | Sortable |
|--------|----------|
| Material code | Yes |
| Material name | Yes |
| Specifications | |

---

### 2. Material warehouse profile form (`/wms/materials/:materialCode`)

**Tabbed form** — material selected from master list (combobox) or linked from PDM sync.

#### Tab: Basic

| Field | Type |
|-------|------|
| Material code | Read-only |
| Material name | Read-only (from master) |
| Specifications | Read-only |
| Unit of measure | Read-only |

#### Tab: Warehouse attributes

| Field | Type | Required |
|-------|------|----------|
| Barcode rule | Select | Material only, Material+batch, Material+serial |
| Packaging spec | Text | No |
| Storage conditions | Text | No |
| Recommended zone | Location tree picker | No |
| Recommended location | Location picker (filtered) | No |
| Shelf life (days) | Number | If perishable |
| Safety stock | Number | Yes |
| Minimum stock | Number | Yes |
| Maximum stock | Number | Yes |
| ABC classification | Select A/B/C | Yes |
| Batch management required | Checkbox | |
| Serial number management required | Checkbox | |
| FIFO required | Checkbox | |

**Validation:** min ≤ safety ≤ max; serial and batch mutual exclusivity warning if both checked without business justification.

#### Tab: Barcode / RFID rules

**Rule builder UI:**

| Rule type | Encoded fields |
|-----------|----------------|
| Material + batch | code, batch, spec, inbound date, shelf life |
| Material + serial | code, serial, spec, inbound date |

- Preview panel: sample barcode label (mock SVG) + RFID tag payload JSON.
- `Generate sample tag` button — shows mock encoded string.

#### Tab: Sync

Read-only sync log:

| Source module | Last synced | Status | Fields changed |
|---------------|-------------|--------|----------------|
| PDM | timestamp | Synced | — |
| SCM | timestamp | Synced | — |

`Sync now` button (mock refresh).
