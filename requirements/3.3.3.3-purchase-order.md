# 3.3.3.3 Purchase Order Management — Frontend Requirements

**Source:** Section 3.3.3.3

---

## Screens

### PO list (`/scm/purchase-orders`)

| PO # | Supplier | Amount | Delivery | Status | Ship / Arrive / IQC / Pay |

Overdue arrival highlighted.

---

### Create PO

From approved requisition (auto-fill) or manual.

**Mandatory:** supplier, PO #, material lines (code, spec, qty, price, amount), delivery date/location, payment, quality standards, acceptance requirements.

Approval → dispatch to supplier + generate contract PDF (mock).

---

### PO change request

Same pattern as order changes — re-approval, update plans on pass.

---

### Tracking timeline

Shipping → arrival → inspection → warehousing → payment milestones.

---

## Mock Data Shape

```ts
interface PurchaseOrderDto {
  Id: string
  PoNumber: string
  SupplierCode: string
  Status: string
  Lines: { MaterialCode: string; Quantity: number; UnitPrice: number; Amount: number; DeliveryDate: string }[]
  ApprovalStatus: string
  Tracking: { Milestone: string; Status: string; Timestamp: string | null }[]
  IsOverdue: boolean
}
```
