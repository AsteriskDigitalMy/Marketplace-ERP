# 3.1.4.2 Manual Form Entry for Business Data — Frontend Requirements

**Source:** F-DATA-002 — Section 3.1.4 Data Collection and Filling  
**Route:** `/pms/data-collection/fill/:taskId`

---

## Functional Summary

Employees complete standardized forms for production/business data. Real-time validation; submit → multi-level approval queue, status `Pending Review`.

**Prerequisites:** Employee has received filling task ([3.1.4.1](./3.1.4.1-periodic-filling-tasks.md)).

---

## Dynamic Filling Form

Rendered from template (mock schema):

- Field types: number, text, date, select (from dictionaries)
- Real-time validation against KPI thresholds (min/max, required)
- Inline errors; block submit until valid

Header: indicator name, period, read-only metadata.

Submit → success → status `Pending Review`; supervisor queue +1 (mock).
