# 3.4.2.4 Work Order Production Reporting (Bao-Gong) — Frontend Requirements

**Source:** Section 3.4.2.4  
**Purpose:** Process-level start/complete reporting via Pad — qualified/unqualified qty, hours, WIP update.

---

## Pad Flow (`/mes/pad/reporting`)

### Login / operator context

Select operator (mock session) before reporting.

### Step 1 — Scan

Scan work order barcode or product RFID → load process, requirements, SOP.

### Step 2 — Process start

**Pre-check panel:** materials ✓, equipment ✓, personnel ✓ (mock toggles).

`Start` button → records start time, operator, equipment #.

### Step 3 — Process complete

| Field | Validation |
|-------|------------|
| Qualified qty | Required, ≥ 0 |
| Unqualified qty | Required if any defect |
| Non-conformance reason | Required if unqualified > 0 |
| Complete qty total | ≤ WO qty − already reported |

**Block submit** if validation fails — inline errors.

### Step 4 — Post-submit

Success screen: updated WO progress %, next process notification (mock).

---

## Admin — Reporting log (`/mes/work-orders/:id/reports`)

Table: process, operator, equipment, start, complete, qualified, unqualified, reason.

---

## Process routing status

On WO detail: stepper updates per Bao-Gong — `Not started` → `In progress` → `Complete`.

WIP qty badge updates (mock).
