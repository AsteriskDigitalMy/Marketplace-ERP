# 3.1.1.5 Account Full Lifecycle Management — Frontend Requirements

**Source:** F-BASE-005 — Section 3.1.1 System Basic Management  
**Route:** `/pms/admin/accounts/:id`

---

## Functional Summary

Full lifecycle: activate, disable, password reset, and HR info edit. All operations logged.

**Prerequisites:** User management permission.

---

## Account Detail

Header: name, employee ID, login account, status badge (Active / Disabled).

### Actions menu

| Operation | UI |
|-----------|-----|
| Disable | Confirm → "Session cleared" toast; status → Disabled |
| Activate | Confirm → status → Active |
| Reset password | Confirm → show mock generated password + "notification sent" |
| Edit employee info | Inline or dialog form |

### Editable HR fields

Name, department, position, contact info (mock). Employee ID read-only.

Operation log panel: timestamp, operator, action, detail (mock immutable list).

---

interface UserAccountDto {
