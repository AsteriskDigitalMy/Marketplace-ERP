# 3.1.6.2 Cockpit Data Hierarchical Drill-Down Interaction — Frontend Requirements

**Source:** F-DASH-002 — Section 3.1.6 KPI Visualization Monitoring Dashboard  
**Route:** `/pms/cockpit` (interaction overlay — `Sheet` on desktop, full-screen `Sheet` on mobile)  
**Prerequisites:** Role cockpit page loaded ([3.1.6.1](./3.1.6.1-role-cockpit-auto-loading.md))  
**UI:** shadcn/ui — `Sheet`, `Breadcrumb`, `Table`, `Badge`, `Button`, `Skeleton`, `ScrollArea`, `Popover`, `Avatar`, `Separator`

---

## Functional Summary

Click dashboard charts, data cards, or status color blocks to drill down layer by layer into detailed business data. Macro summary → departmental breakdown → team/group/individual performance.

**Input:** Unique identification ID of clicked data element.

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Role | Goals | Pain points |
|---------|------|-------|-------------|
| **Executive** | Strategic viewer | From group metric to dept breakdown in two clicks | Dead-end charts with no detail |
| **Department Manager** | Operational lead | Identify underperforming teams from summary card | Export-heavy workflows |
| **HR / Auditor** | Individual accountability | Reach employee-level KPI scores | Privacy concerns — read-only only |
| **Keyboard user** | Accessibility | Navigate drill-down without mouse | Non-focusable chart segments |

### Core Capabilities

| ID | Capability | Description |
|----|------------|-------------|
| FR-001 | Multi-target drill triggers | Cards, chart segments/bars/points, status blocks, table rows |
| FR-002 | Three-level hierarchy | Level 1 dept/category → Level 2 team/group → Level 3 individual |
| FR-003 | Sheet overlay UI | Wide `Sheet` desktop; full-screen mobile; cockpit remains mounted underneath |
| FR-004 | Breadcrumb navigation | Chips jump to ancestor levels without closing sheet |
| FR-005 | Per-level async fetch | Skeleton on level change; independent error/empty per level |
| FR-006 | Status color on rows | Traffic-light from [3.1.6.3](./3.1.6.3-traffic-light-rules.md) on metric cells |
| FR-007 | Employee profile popover | Level 3 name click → read-only popover (mock profile) |
| FR-008 | State preservation | Close sheet → cockpit scroll and refresh state unchanged |
| FR-009 | Keyboard support | Focus ring on triggers; Enter opens; Escape closes sheet |

### Edge Cases & Alternative Paths

- **Click element with no children:** Level 1 shows empty "No detail data for this selection."
- **Fetch error on level 2:** Inline error in sheet with retry; breadcrumb still navigable.
- **Rapid breadcrumb clicks:** Cancel in-flight fetch; load latest requested level.
- **Chart segment too small on mobile:** Minimum 44px touch target via invisible hit area.
- **Individual level privacy:** Only show employees user is permitted to see (mock filter).
- **Sheet open + cockpit auto-refresh:** Underlying cards update; sheet content stale until user closes and re-opens.
- **Deep link mock:** Optional `?drill=widgetId:pointId` opens sheet on load (dev only).

## Phase 2: Screen-by-Screen Functional Specifications

### Screen Name: Drill-Down Sheet — Level 1 Summary (`/pms/cockpit` overlay)

* **Objective:** Show first breakdown of clicked parent metric (e.g. by department).
* **UI Elements & Field-Level Specs:**
  * `Sheet` side right, `w-full sm:max-w-2xl lg:max-w-4xl`.
  * Title: parent metric name.
  * `Breadcrumb`: Cockpit > `{MetricName}`.
  * `Table` columns: Name (dept/category), primary metric value, unit, achievement %, status color dot, Actions (chevron).
  * Rows: `cursor-pointer`, hover `bg-muted/50`, focus visible ring.
  * Footer: `Button` "Close" outline.
* **Functional Logic:**
  * Open on click with `DrillDownRequest { SourceWidgetId, DataPointId, Level: 1 }`.
  * Fetch `DrillDownNodeDto[]` children of root.
  * Row click → Level 2 with `ParentId = row.Id`.

### Screen Name: Drill-Down Sheet — Level 2 Team / Group

* **Objective:** Break down one department/category into teams or groups.
* **UI Elements & Field-Level Specs:**
  * Breadcrumb: Cockpit > `{Dept}` > Teams.
  * `Table`: Team name, KPI values (multi-metric columns from `Metrics[]`), achievement %, lead name, status badges.
  * Sortable columns (client-side mock).
* **Functional Logic:**
  * Request `Level: 2` with parent node id.
  * Row click → Level 3 if children exist; else popover "No further breakdown."

### Screen Name: Drill-Down Sheet — Level 3 Individual

* **Objective:** List employee-level performance for selected team.
* **UI Elements & Field-Level Specs:**
  * Breadcrumb: Cockpit > `{Dept}` > `{Team}` > Individuals.
  * `Table`: Rank, Employee name (link style), Employee ID, individual KPI scores, composite status.
  * Name click → `Popover` with read-only mock profile: photo placeholder, title, dept, email.
* **Functional Logic:**
  * Request `Level: 3`; render flat individual rows.
  * No further drill; name popover only.

### Screen Name: Drill-Down Loading / Empty / Error (sheet body states)

* **Objective:** Communicate fetch state inside sheet without closing.
* **UI Elements & Field-Level Specs:**
  * Loading: 5-row `Skeleton` table.
  * Empty: centered message + illustration icon.
  * Error: `Alert` + Retry `Button`.
* **Functional Logic:**
  * Retry re-issues fetch for current level and parent id.

### Click Targets on Cockpit (parent page)

| Widget type | Click behavior |
|-------------|----------------|
| Data card | Open detail sheet with breakdown table |
| Chart segment / bar / point | Open detail sheet filtered to segment |
| Status color block | Open detail sheet for items in that status band |
| Table row | Open detail sheet for that entity |

All clickable elements: `cursor-pointer`, `tabIndex={0}`, `aria-label` describing drill action.

## Phase 3: Screen Transition & User Flow Mapping

| # | Trigger | Source | Destination | Conditional paths |
|---|---------|--------|-------------|-------------------|
| 1 | Click card | Cockpit card | Sheet Level 1 | No data → empty state |
| 2 | Click chart segment | Cockpit chart | Sheet Level 1 filtered | — |
| 3 | Click status block | Status legend/block | Sheet Level 1 filtered by band | — |
| 4 | Click table row | Cockpit table | Sheet Level 1 for entity | — |
| 5 | Click dept row | Sheet L1 | Sheet L2 | — |
| 6 | Click team row | Sheet L2 | Sheet L3 | No children → toast |
| 7 | Click breadcrumb chip | Sheet any level | Sheet ancestor level | Reload that level data |
| 8 | Escape / Close | Sheet | Cockpit (unchanged scroll) | — |
| 9 | Enter on focused trigger | Cockpit keyboard | Sheet Level 1 | Same as click |
| 10 | Click employee name | Sheet L3 | Popover profile | — |

## Mock Data Shape

```ts
interface DrillDownNodeDto {
  Id: string
  Label: string
  Level: 'department' | 'team' | 'individual'
  ParentId: string | null
  Metrics: { Name: string; Value: number; Unit: string; StatusColor: 'green' | 'yellow' | 'red' }[]
  Children?: DrillDownNodeDto[]
}

interface DrillDownRequest {
  SourceWidgetId: string
  DataPointId: string
  Level: number
}
```
