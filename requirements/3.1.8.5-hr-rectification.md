# 3.1.8.5 HR Performance Rectification Processing — Frontend Requirements

**Source:** F-PERF-005 — Section 3.1.8 Performance Appraisal & Scoring  
**Route:** `/pms/appraisal/hr`

---

## Functional Summary

HR assists low-performance employees through interviews, communication, skills training, and job guidance. Can optionally convert issues into a formal PDCA improvement proposal.

**Input:** Employee ID, rectification assistance records, whether to generate PDCA proposal.  
**Prerequisites:** Employee status is "Pending HR Processing" (routed from [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md)).

---

## HR Queue (`/pms/appraisal/hr`)

Table: Employee, Department, Grade, Preliminary score, Days in queue, Status, Actions.

Filter: department, overdue assistance (> SLA days mock).

---

## Rectification Workspace (`/pms/appraisal/hr/:employeeId`)

### Performance shortcoming summary (read-only)

- Grade, total score, low-scoring indicators highlighted
- Link to full appraisal detail

### Assistance record form

| Field | Required |
|-------|----------|
| Interview date | Yes |
| Communication summary | Yes |
| Assistance type | Multi: interview / training / job guidance / other |
| Training or guidance details | Yes if type selected |
| Follow-up plan | Yes |
| Attachments | Optional (mock) |

### PDCA one-click conversion

Checkbox: **"Generate PDCA improvement proposal"**

When checked, on submit:

- Pre-fill PDCA form fields from assistance record ([3.1.9.1](./3.1.9.1-pdca-proposal-submission.md))
- Create linked proposal (mock ID); show link to proposal

### Submit

**"Complete HR assistance"** → status `HR Processed` → pushes secondary confirmation task to Auditor ([3.1.8.6](./3.1.8.6-secondary-confirmation.md)).
