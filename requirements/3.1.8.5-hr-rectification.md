# 3.1.8.5 HR Performance Rectification Processing — Frontend Requirements

**Source:** F-PERF-005 — Section 3.1.8 Performance Appraisal & Scoring  
**Route:** `/pms/appraisal/hr`  
**Workflow:** Receives C/D from [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md) after [3.1.8.3](./3.1.8.3-preliminary-review-list.md) / [3.1.8.2](./3.1.8.2-automatic-scoring-grading.md) → submits to [3.1.8.6](./3.1.8.6-secondary-confirmation.md) → optional PDCA via [3.1.9.1](./3.1.9.1-pdca-proposal-submission.md) → may reach [3.1.8.7](./3.1.8.7-executive-final-review.md) after secondary pass

---

## Functional Summary

HR assists low-performance employees through interviews, communication, skills training, and job guidance. Can optionally convert issues into a formal PDCA improvement proposal.

**Input:** Employee ID, rectification assistance records, whether to generate PDCA proposal.  
**Prerequisites:** Employee status is "Pending HR Processing" (routed from [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md)).

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Goals | Pain Points |
|---------|-------|-------------|
| **HR Specialist** | Document assistance for C/D employees; close HR queue items | Scattered notes across email and paper |
| **Employee** (indirect) | Receive structured improvement support | Unclear follow-up expectations |
| **Auditor** | Review HR work in [3.1.8.6](./3.1.8.6-secondary-confirmation.md) | Incomplete assistance records |

### Core Capabilities (FR-001…)

| ID | Capability | Priority |
|----|------------|----------|
| FR-001 | HR queue listing grade C/D employees with days-in-queue | Must |
| FR-002 | Filter by department and overdue SLA (> mock days) | Should |
| FR-003 | Rectification workspace with read-only performance shortcoming summary | Must |
| FR-004 | Assistance record form with typed interventions | Must |
| FR-005 | One-click PDCA proposal generation pre-filled from record [3.1.9.1](./3.1.9.1-pdca-proposal-submission.md) | Must |
| FR-006 | Submit completes HR assistance → status `HR Processed` → notifies Auditor [3.1.8.6](./3.1.8.6-secondary-confirmation.md) | Must |
| FR-007 | HR role permission gate | Must |
| FR-008 | Link to full appraisal detail from [3.1.8.3](./3.1.8.3-preliminary-review-list.md) context | Should |

### Edge Cases & Alternative Paths

- **Employee returned from failed secondary review [3.1.8.6](./3.1.8.6-secondary-confirmation.md):** Queue shows "Re-rectification" badge; prior records read-only in timeline; new record required.
- **Multiple assistance sessions:** Allow append records before final submit (mock: single form per visit or add timeline entries).
- **PDCA checked but proposal creation fails:** Toast error; HR can retry; assistance still savable without PDCA.
- **Missing required fields for selected assistance types:** Inline validation on training/guidance details.
- **Overdue SLA:** Row highlighted red; sort by days in queue default.
- **Employee not in HR queue:** 404 on workspace deep link.
- **Submit without PDCA:** Valid path; `LinkedPdcaProposalId` null.

## Phase 2: Screen-by-Screen Functional Specifications

### Screen: HR Queue (`/pms/appraisal/hr`)

* **Objective:** List employees pending HR rectification assistance.
* **UI Elements & Field-Level Specs:**
  * Page title: "HR Performance Rectification"
  * Nav badge: count of pending items (synced from [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md))
  * Filters: Department, **Overdue only** toggle (> SLA days mock)
  * Table columns: Employee, Department, Grade (C/D badge), Preliminary score, Days in queue (red if overdue), Status (Pending HR / Re-rectification), Actions (**Open workspace**)
  * Empty state: "No employees awaiting HR assistance"
  * Four async states
* **Functional Logic:**
  * Only grade C/D employees with `pending_hr_processing` or returned status
  * Row click or action → `/pms/appraisal/hr/:employeeId`

### Screen: Rectification Workspace (`/pms/appraisal/hr/:employeeId`)

* **Objective:** Capture HR assistance and optionally spawn PDCA proposal.
* **UI Elements & Field-Level Specs:**
  * Breadcrumb: HR Queue → {Employee name}
  * **Performance shortcoming summary** (read-only card): Grade, total score, low-scoring indicators highlighted (name, score, gap); **View full appraisal** link (mock detail drawer from preliminary data)
  * **Assistance record form:**
    * Interview date (date picker, required)
    * Communication summary (textarea, required)
    * Assistance type (multi-checkbox: interview / training / job guidance / other)
    * Training or guidance details (textarea, required if training or job guidance selected)
    * Follow-up plan (textarea, required)
    * Attachments (mock file names, optional)
  * **PDCA conversion** card: Checkbox **Generate PDCA improvement proposal**; when checked, show preview of fields to pre-fill per [3.1.9.1](./3.1.9.1-pdca-proposal-submission.md)
  * **Prior assistance timeline** (if re-rectification): read-only entries from previous visits
  * Sticky footer: **Complete HR assistance** (primary), **Save draft** (secondary), **Back to queue**
* **Functional Logic:**
  * Validate conditional fields on assistance types
  * On submit with PDCA: create mock proposal; set `LinkedPdcaProposalId`; show link to `/pms/pdca/proposals/:id`
  * On submit: status → `hr_processed`; push task to [3.1.8.6](./3.1.8.6-secondary-confirmation.md); decrement HR badge
  * Save draft: persist form without status change

### Modal: PDCA Created Success

* **Objective:** Confirm linked proposal after one-click conversion.
* **UI Elements:** Proposal ID, **Open proposal** → `/pms/pdca/proposals/:id`, **Continue in HR workspace**
* **Functional Logic:** Proposal pre-filled via `?fromHr=employeeId` pattern from [3.1.9.1](./3.1.9.1-pdca-proposal-submission.md)

### Modal: Complete Assistance Confirmation

* **Objective:** Confirm submission sends employee to secondary review.
* **UI Elements:** Summary of assistance types; PDCA yes/no; **Submit** / **Cancel**
* **Functional Logic:** Required before first-time complete (not draft save)
