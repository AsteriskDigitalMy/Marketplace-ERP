# 3.1.7.2 Exception Alert Closed-Loop Disposal Process — Frontend Requirements

**Source:** F-ALERT-002 — Section 3.1.7 Exception Alert & Management  
**Routes:** `/pms/alerts`, `/pms/alerts/:id/disposal`

---

## Functional Summary

After an exception alert fires, the business owner completes root-cause investigation, rectification, and optimization within a deadline — forming a full management closed loop.

**Input:** Alert record ID, cause, rectification measures, completion results.  
**Prerequisites:** Corresponding business exception alert has been triggered.

---

## Alert Inbox (`/pms/alerts`)

| Column | Notes |
|--------|-------|
| Alert ID | Link to disposal |
| Type / object | From rule ([3.1.7.1](./3.1.7.1-alert-rule-configuration.md)) |
| Level | General / Important / Urgent badge |
| Triggered at | Timestamp |
| Deadline | Countdown badge; overdue → escalated level (visual) |
| Status | Open / In progress / Pending verification / Closed |
| Owner | Assigned responsible person |

Filters: status, level, type, overdue only.

---

## Disposal Workflow (`/pms/alerts/:id/disposal`)

### Timeline header

- Alert summary, level, escalation history (auto-escalate if past deadline unprocessed)
- Status stepper: `Triggered` → `Investigating` → `Rectifying` → `Pending verification` → `Closed`

### Step 1 — Root cause (owner)

| Field | Required |
|-------|----------|
| Problem cause | Yes, textarea |
| Impact scope | Optional |

Save draft or submit → moves to rectification.

### Step 2 — Rectification plan (owner)

| Field | Required |
|-------|----------|
| Disposal / rectification measures | Yes |
| Planned completion date | Yes |
| Attachments | Optional (mock file names) |

Submit for confirmation → notifies manager (mock banner).

### Step 3 — Management verification

| Field | Required |
|-------|----------|
| Verification result | Pass / Fail |
| Verification opinion | Yes on fail |

- **Pass** → status `Closed`; log retained
- **Fail** → return to owner with comments; deadline may reset (mock)

### Escalation display

If unprocessed past deadline: show escalated level badge + audit log entry ("Auto-escalated to Urgent at …").

---

## Disposal Log (read-only panel)

Chronological entries: trigger, assignments, submissions, escalations, verifications, closure. Immutable audit trail (mock).
