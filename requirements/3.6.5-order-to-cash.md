# 3.6.5 Order-to-Cash (O2C) Integration — Frontend Requirements

**Source:** Section 3.6.5

---

## Process timeline (`/sap/o2c`)

| Step | Operational doc | SAP result |
|------|-----------------|------------|
| Sales order approved | SO # | SAP sales order + stock reserve |
| Goods issue | Delivery # | GI voucher + COGS |
| Sales invoice | Invoice # | Revenue + AR |
| Collection | Receipt # | AR clear + bank |
| Return | Return # | Reversal GI + revenue |

Same drawer detail pattern as P2P.

---

### Document list (`/sap/o2c/documents`)

Filter by customer, step, sync status.

---

### Embedded on SCM order screens

Sales order, shipment, invoice, collection — `SapSyncBadge` + voucher link.

OQC ship gate (MES 3.4.5.2) may show AR sync prerequisite banner (optional cross-link).

---

### O2C reconciliation (`/sap/o2c/reconciliation`)

Daily: delivery + invoice + collection vs SAP AR.

Month-end: customer statement vs SAP AR balance.

Variance alerts to finance/sales managers.
