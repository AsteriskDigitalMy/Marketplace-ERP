# 3.4.2.5 Work Order Status Management and Closure — Frontend Requirements

**Source:** Section 3.4.2.5

## Status Model (UI)

| Status | Color token |
|--------|-------------|
| Pending dispatch | muted |
| Dispatched | info |
| Pending material picking | warning |
| In production | primary |
| Completed | success |
| Under quality inspection | warning |
| Stocked into warehouse | success |
| Closed | muted |
| Paused | warning |
| Cancelled | destructive |

**Status filter** on all WO lists; badge on cards.

---

## Pause / Resume

**Pause dialog:** reason (material shortage, anomaly, order change), expected resume date optional.

While paused:

- Banner on WO detail and Pad views
- Pause duration tracked; excluded from production hours (note in closure report)

**Resume** — confirm dialog; clears pause banner.

---

## Closure workflow (`/mes/work-orders/:id/close`)

### Preconditions checklist

| Check | Status |
|-------|--------|
| All processes complete | ✓/✗ |
| Finished goods fully stocked (WMS) | ✓/✗ |
| Materials fully returned | ✓/✗ |
| Quality inspections finalized | ✓/✗ |

Submit disabled until all ✓ (mock validation).
