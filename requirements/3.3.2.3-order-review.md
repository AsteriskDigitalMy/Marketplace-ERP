# 3.3.2.3 Multi-Dimensional Order Review Management — Frontend Requirements

**Source:** Section 3.3.2.3

---

## Review pipeline UI

Four stages (configurable by order type / customer tier):

| Stage | Department | Output |
|-------|------------|--------|
| Delivery | Planning | Achievable? Suggested date |
| Technical | Technical | BOM/process complete? Risks |
| Material | Procurement | Shortage risks, lead times |
| Financial | Finance | Margin, credit, pricing |

---

### Review inbox (`/scm/orders/reviews`)

Filter by stage, department, urgent fast-track flag.

---

### Review form (per stage)

| Field | Required on fail |
|-------|------------------|
| Decision | Pass / Fail |
| Opinion | Yes |
| Risk notes | If risks |
| Suggested adjustments | Optional |

**Fail paths:** return to sales for edit OR reject order.

**Fast-track** toggle for urgent orders — shortened timeline badge.

On all pass → status `Effective` → auto-enter planning queue (mock banner).

## Acceptance Criteria

- [ ] Four-stage review timeline with department actions
- [ ] Fail returns to sales or rejects with required comments
- [ ] Fast-track urgent channel visually distinct
- [ ] All-pass transitions order to effective with planning CTA
