# 3.6.4 Procure-to-Pay (P2P) Integration — Frontend Requirements

**Source:** Section 3.6.4

---

## Process timeline UI (`/sap/p2p`)

Visual pipeline for each PO lifecycle:

| Step | Operational doc | SAP result | Sync status |
|------|-----------------|------------|-------------|
| PO approved | PO # | SAP PO + budget reserve | badge |
| Goods receipt | Receipt # | Material doc + GR/IR | badge |
| Invoice | Invoice # | Invoice voucher + AP | badge |
| Payment | Payment req # | Payment voucher | badge |
| Return | Return # | Reversal docs | badge |

Click step → detail drawer with payload summary and SAP voucher # (mock).

---

### Document list (`/sap/p2p/documents`)

Filter by step, supplier, date, sync status.

Columns: doc #, type, supplier, amount, trigger node, SAP voucher, status, actions (retry).

---

### Embedded on SCM screens

On PO detail, goods receipt, invoice, payment (SCM 3.3.3.x):

- `SapSyncBadge` + voucher number when synced
- Link "View in SAP integration" → P2P timeline

---

### P2P reconciliation (`/sap/p2p/reconciliation`)

Daily auto-match: operational receipt + invoice + payment vs SAP AP.

| Supplier | Ops amount | SAP amount | Variance | Status |

Discrepancies flagged → push finance/procurement (mock alert).

**Month-end:** supplier statement generator vs SAP AP balance — PDF mock export.
