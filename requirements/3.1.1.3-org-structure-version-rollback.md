# 3.1.1.3 Organizational Structure Version Rollback and Recovery — Frontend Requirements

**Source:** F-BASE-003 — Section 3.1.1 System Basic Management  
**Route:** `/pms/admin/org/:id/history`

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Goals |
|---------|-------|
| **System Administrator** | Recover from mistaken org restructuring by previewing historical snapshots and rolling back safely. |
| **Auditor / Compliance Officer** | Trace who changed the org structure, when, and what the tree looked like at any point in time. |
| **HR / Org Architect** | Compare current vs historical layout before deciding to restore a version. |
| **Supreme Admin** | Perform rollback with confidence that a new audit entry is created (non-destructive history). |

### Core Capabilities

1. **FR-001 — Version timeline:** Display chronological list of org structure changes for unit `:id` (and subtree context) using `OrgVersion` from `src/models/pms/organization.ts`.
2. **FR-002 — Historical preview:** On version select, render read-only tree snapshot from `TreeSnapshot` with diff highlights vs current (added/removed/renamed — mock).
3. **FR-003 — One-click rollback:** Restore selected snapshot after confirmation; creates new version record and operation log (mock).
4. **FR-004 — Permission gate:** Restrict access to architecture management or system audit permission; others see 403.
5. **FR-005 — Rollback progress feedback:** Show in-dialog progress during mock async restore; prevent duplicate rollback requests.
6. **FR-006 — Async states:** Loading, empty (no versions), error, and success for timeline fetch, preview load, and rollback.

### Edge Cases & Alternative Paths

- No version history → empty state "No changes recorded yet" with link back to edit page.
- User selects version then rapidly clicks another → cancel in-flight preview fetch; show latest only.
- Rollback while preview still loading → **Rollback** button disabled until preview ready.
- Rollback fails (mock) → **Toast** error; remain on history page; no partial tree update.
- Current version selected → **Rollback** hidden or disabled with tooltip "Already on this version."
- User lacks audit permission → entire page 403 with link to `/pms/admin/org`.
- Very large snapshot → tree preview uses collapsed default with expand-all control.
- Double-click **Restore this version** → disabled after first click.

## Phase 2: Screen-by-Screen Functional Specifications

### Screen Name: Version History (`/pms/admin/org/:id/history`)

* **Objective:** Browse org change timeline, preview any snapshot, and initiate rollback.

* **UI Elements & Field-Level Specs:**
  * **Page header:** "Version History — {Unit Name}"; breadcrumb: Admin → Org Structure → Edit → History.
  * **Unit context card:** `Code`, current version #, last change summary.
  * **Timeline panel (left `ScrollArea`, ~35% width):** Vertical timeline items — version #, `SnapshotAt`, `Actor`, `Summary` truncated; selected item highlighted.
  * **Preview panel (right, ~65%):** Read-only org tree; diff legend (**Badge** green = added, red = removed, amber = renamed).
  * **Preview toolbar:** Expand all / Collapse all; toggle "Show diff only" (mock filter).
  * **Restore this version (`Button` destructive):** Enabled when selected version ≠ current; opens Rollback Confirmation Dialog.
  * **Back to edit (`Button` outline):** Navigates to `/pms/admin/org/:id/edit`.
  * **Loading:** Timeline skeleton + tree skeleton.
  * **Empty:** Illustration + message.
  * **Error:** **Alert** with retry.

* **Functional Logic:**
  * Fetch version list from `src/mocks/pms/org-versions.ts` mapped to `OrgVersionDto`.
  * Timeline click: load `TreeSnapshot`; compute mock diff vs current tree; render preview.
  * Permission check on mount; redirect or 403 if denied.
  * Keyboard: up/down arrows move timeline selection (UX enhancement).

### Screen Name: Rollback Confirmation Dialog (modal on `/pms/admin/org/:id/history`)

* **Objective:** Confirm destructive-in-effect restore of a historical org snapshot.

* **UI Elements & Field-Level Specs:**
  * **Dialog:** Title "Restore version {N}?"; description warns current structure will be replaced by snapshot.
  * **Summary block:** Version #, `SnapshotAt`, `Actor`, `Summary`.
  * **Impact note (`Alert`):** "A new version record will be created. This action is audited."
  * **Cancel (`Button` outline):** Closes dialog.
  * **Confirm Restore (`Button` destructive):** Shows spinner + "Restoring…" when in progress; disabled during request.

* **Functional Logic:**
  * On confirm: mock `POST` rollback → progress state 0–100% (simulated) → on success close dialog.
  * Success **Toast:** "Version {N} restored"; append new timeline entry at top; refresh preview to current.
  * Failure **Toast:** destructive; re-enable confirm button.
  * Double-submit prevention.

### Screen Name: Version Preview Sheet (optional narrow view — `Sheet` on mobile)

* **Objective:** Full-width preview on small viewports when timeline and preview cannot sit side-by-side.

* **UI Elements & Field-Level Specs:**
  * **Sheet (shadcn `Sheet`):** Slides from right; contains same read-only tree + diff legend as desktop preview.
  * **Close (`Button` ghost):** Returns to timeline-only view.

* **Functional Logic:**
  * Opens when user taps timeline item on viewport &lt; 1024px (responsive breakpoint).
  * Same preview data binding as main preview panel.

## Phase 3: Screen Transition & User Flow Mapping

| # | Trigger Event | Source Screen | Destination Screen | Conditional Paths |
|---|---------------|---------------|--------------------|-------------------|
| 1 | Click **Version History** tab/link | Organization Unit Edit | Version History | Permission denied → 403 |
| 2 | Page load | — | Version History | Invalid `:id` → 404 |
| 3 | Click timeline version item | Version History | Version History (preview updates) | Mobile → also open Version Preview Sheet |
| 4 | Click **Restore this version** | Version History | Rollback Confirmation Dialog | Current version → button disabled |
| 5 | Confirm restore | Rollback Confirmation Dialog | Version History | Success → timeline refreshes; Toast |
| 6 | Cancel rollback | Rollback Confirmation Dialog | Version History | — |
| 7 | Rollback API error | Rollback Confirmation Dialog | Rollback Confirmation Dialog | Toast error; dialog open |
| 8 | Click **Back to edit** | Version History | Organization Unit Edit (`/pms/admin/org/:id/edit`) | — |
| 9 | Breadcrumb **Org Structure** | Version History | Organization Tree (`/pms/admin/org`) | — |
