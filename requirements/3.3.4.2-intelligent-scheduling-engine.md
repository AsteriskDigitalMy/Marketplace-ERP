# 3.3.4.2 Intelligent Scheduling Engine — Frontend Requirements

**Source:** Section 3.3.4.2

---

## Screens

### Scheduling run (`/scm/scheduling/engine`)

#### Mode selection

| Mode | Description |
|------|-------------|
| Forward | Earliest completion |
| Backward | Latest start from delivery |
| Bottleneck | Optimize bottleneck equipment |

#### Constraints panel

**Hard (required):** delivery date, kitting date, equipment hours, process sequence.

**Soft (optional):** min batch, changeover time, same product grouping, same customer grouping, skill match.

`Run schedule` → mock computation progress → results.

---

### Output preview

Workshop / line / process / date breakdown:

| WO | Qty | Start | End | Equipment | Shift |

**Capacity load chart** — utilization % per line.

**Scenario compare** — side-by-side two scheduling runs (mock).

interface SchedulingRunDto {
