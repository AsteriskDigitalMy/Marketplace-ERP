# 3.1.3.1 Project Initiation and Application — Frontend Requirements

**Source:** F-PROJ-001 — Section 3.1.3 Project Management  
**Routes:** `/pms/projects`, `/pms/projects/new`

---

## Functional Summary

Project leaders initiate projects with full basic info, bound KPIs, budget, team, and initial Gantt placeholder. Submit → unique project code, status `Pending Approval`, pushed to auditor queue.

**Prerequisites:** Project initiation permission.

---

## Initiation Form (`/pms/projects/new`)

| Field | Required | Validation |
|-------|----------|------------|
| Project name | Yes | Unique |
| Business type | Yes | Dictionary |
| Planned start / end | Yes | End > start |
| Overall leader | Yes | User picker |
| Execution team | Yes | Multi-select users |
| Budget amount | Yes | > 0 |
| Bound KPI indicators | Yes | Multi from KPI library |
| Partner/client info | Optional | |
| Initial Gantt status | Optional | Milestone sketch (mock) |

Submit → generated project code display + redirect to detail.
