# 3.1.7.1 Global Alert Trigger Rule Custom Configuration — Frontend Requirements

**Source:** F-ALERT-001 — Section 3.1.7 Exception Alert & Management  
**Route:** `/pms/alerts/rules`  
**Related:** [3.7.5 BI alerts](./3.7.5-alerts-trend-forecasting.md) (analytics-oriented thresholds)

---

## Functional Summary

Managers configure exception alert trigger conditions: KPI overruns, project delays, filing overdues. Rules monitor business data 7×24 once enabled.

**Prerequisites:** Alert rule configuration and management permission.

---

## Alert Type Taxonomy

| Type | Examples |
|------|----------|
| KPI indicator | Achievement below threshold, YoY drop |
| Project progress | Milestone overdue, budget overrun |
| Filing overdue | Report not submitted by deadline |

---

## Alert Levels & Channels

| Level | Badge color | Channels (multi-select) |
|-------|-------------|-------------------------|
| General | Default | In-app message |
| Important | Warning | In-app + email |
| Urgent | Destructive | In-app + SMS + email |

Channel delivery is mocked (toast / log panel).

---

## Rule List (`/pms/alerts/rules`)

Columns: Rule name, Type, Monitored object, Level, Channels, Enabled toggle, Last triggered, Actions.

Filters: type, level, enabled status. Bulk enable/disable (mock).

---

## Rule Editor

| Field | Required |
|-------|----------|
| Rule name | Yes |
| Alert type | Yes |
| Monitored business object | Yes (picker: KPI, project, filing entity) |
| Trigger condition | Yes — operator + value (e.g. `achievement_rate < 70`) |
| Alert level | Yes |
| Push methods | At least one channel |
| Enabled on save | Toggle |

**Condition builder:** field dropdown + operator (`>`, `<`, `=`, `>=`, `<=`) + numeric or date value.

Preview card: human-readable rule summary before save.
