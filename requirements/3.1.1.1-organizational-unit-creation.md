# 3.1.1.1 Organizational Unit Creation — Frontend Requirements

**Source:** F-BASE-001 — Section 3.1.1 System Basic Management  
**Route:** `/pms/admin/org`

---

## Functional Summary

System administrators add organizational units to the structure tree: company, plant, department, workshop, production line, process, and shift/team tiers.

**Prerequisites:** Organizational structure management permission.

---

## Org Tree UI (`/pms/admin/org`)

- Expandable tree with tier icons and status badges (active/disabled)
- **Add** on selected parent node → dialog form

### Create form

| Field | Required | Notes |
|-------|----------|-------|
| Parent node | Yes | Pre-filled from tree selection |
| Organization name | Yes | Unique among siblings |
| Tier type | Yes | company / plant / dept / workshop / line / process / shift |
| Process business tag | Yes if type = process | Picker from process dictionary; links KPI library |
| Sort order | Yes | Integer |

**Validation:** duplicate sibling name → inline error; process type without tag → block submit.

On success: new node in tree, mock unique org code displayed in toast, operation log entry (mock).
