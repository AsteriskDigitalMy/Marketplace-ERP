# 3.1.6.1 Role-Specific Cockpit Automatic Loading — Frontend Requirements

**Source:** F-DASH-001 — Section 3.1.6 KPI Visualization Monitoring Dashboard  
**Route:** `/pms/cockpit`  
**Related:** [3.7.2.6 KPI/PMS BI dashboard](./3.7.2.6-kpi-pms-dashboard.md) (read-only analytics layer)

---

## Functional Summary

Automatically matches and displays the exclusive visual data workbench for the logged-in user's role. Chart data loads asynchronously via backend APIs with scheduled auto-refresh.

**Prerequisites:** User is authenticated.  
**Input:** System role of the current user (from session/mock auth context).

---

## Role → Cockpit Mapping

On login or navigation to `/pms/cockpit`, read bound role permissions and render the matching layout:

| Role (mock) | Cockpit variant | Primary widgets |
|-------------|-----------------|-----------------|
| Executive / CFO | Strategic cockpit | Group KPI cards, portfolio charts, alert summary |
| Department manager | Department cockpit | Dept KPI cards, team tables, progress charts |
| Auditor | Audit cockpit | Pending review counts, exception heatmap |
| HR | HR cockpit | Low-performance queue, PDCA pipeline |
| Employee | Personal cockpit | Personal KPI progress, appraisal status |

Unsupported roles → friendly empty state with contact-admin CTA.

---

## Page Layout

### Header

- Role badge + cockpit title (e.g. "Department Manager Cockpit")
- Last refreshed timestamp + manual refresh button
- Auto-refresh interval selector (mock: 30s / 1m / 5m / off)

### Core data cards row

4–6 summary cards (role-specific metrics). Each card shows:

- Metric label, value, unit
- Traffic-light status color (from global rules — see [3.1.6.3](./3.1.6.3-traffic-light-rules.md))
- Sparkline or mini progress bar (optional)

### Statistical charts grid

2–4 charts per role (bar, line, donut — mock with Recharts or placeholder). Charts load independently with per-widget skeleton loaders.

### Business data table

Sortable, paginated table of role-relevant records (e.g. projects, KPIs, alerts). Row click supports drill-down (see [3.1.6.2](./3.1.6.2-cockpit-drill-down.md)).

---

## Async Loading Behavior

1. Page shell renders immediately with skeleton placeholders.
2. Cards, charts, and table fetch in parallel (mock `Promise.all` or staggered).
3. Failed widget shows inline error with retry — other widgets remain visible.
4. Auto-refresh re-fetches without full page reload; preserve scroll and expanded state.

## Acceptance Criteria

- [ ] Cockpit layout auto-selected from mock user role on load
- [ ] Data cards, charts, and table render with independent async states
- [ ] Manual refresh and configurable auto-refresh interval
- [ ] Per-widget error + retry without blocking entire page
- [ ] Traffic-light colors applied from global rule config
- [ ] Unsupported role shows empty state
- [ ] Four async states (loading / empty / error / success) at page and widget level
