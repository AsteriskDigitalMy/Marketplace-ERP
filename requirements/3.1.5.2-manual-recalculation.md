# 3.1.5.2 Manual Re-Calculation Triggered by Administrators — Frontend Requirements

**Source:** F-CALC-002 — Section 3.1.5 KPI Calculation and Statistical Analysis  
**Route:** `/pms/kpi/calculation/recalculate`

---

## Functional Summary

Admins manually trigger single-indicator or full-volume re-calculation after data corrections. Async execution without blocking UI; historical data retained.

**Prerequisites:** KPI calculation management permission.

---

## Re-calculation Panel

| Field | Options |
|-------|---------|
| Scope | All indicators / Designated indicator / Designated cycle |
| Indicator | Picker (if designated) |
| Cycle / period | Picker (if designated) |

**Start re-calculation** → async job with progress bar (mock polling).

Non-blocking: user can navigate away; banner shows job in progress.

---

## Results

On complete:

- Latest results overwrite display values
- Historical backup badge: "Previous run preserved — view history"
- History table: run time, scope, operator, result count
