# 3.1.1.8 Global Dictionary Maintenance — Frontend Requirements

**Source:** F-BASE-008 — Section 3.1.1 System Basic Management  
**Route:** `/pms/admin/dictionaries`

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Goals |
|---------|-------|
| **System Configuration Admin (primary)** | Maintain enumeration dictionaries (leather categories, processes, defects, project types, performance tiers) used app-wide. |
| **Business Analyst** | Add custom dictionary categories and items without developer deployment. |
| **Module Developer (indirect)** | Relies on stable dictionary codes in forms and KPI bindings (`ProcessTag` on `OrgUnit`). |
| **End User (indirect)** | Sees only enabled dictionary items in dropdowns; disabled items hidden immediately (mock). |

### Core Capabilities

1. **FR-001 — Category navigation:** Searchable left panel list of `DictionaryCategory` from `src/models/pms/configuration.ts`.
2. **FR-002 — Sub-item table:** Right panel **Table** for selected category items — Code, Display name, Sort order, Enabled, Actions.
3. **FR-003 — Sub-item CRUD:** Create and edit via **Dialog**; delete allowed for non-builtin categories only.
4. **FR-004 — Uniqueness per category:** Code and DisplayName unique within category per `DictionaryCategorySchema` superRefine.
5. **FR-005 — Builtin protection:** `IsBuiltin = true` categories — items can be disabled but not deleted; delete button hidden with **Tooltip**.
6. **FR-006 — Enable/disable toggle:** Inline or in editor; disabled items struck through in preview sample.
7. **FR-007 — Dropdown preview sample:** Mock combobox showing only `IsEnabled = true` items after save; **Toast** "Forms refreshed (mock)."
8. **FR-008 — Async states:** Loading, empty category, error, success for category list, item table, and save.

### Edge Cases & Alternative Paths

- Category with no items → empty table CTA "Add first item."
- User deletes last item in custom category → allowed; table empty state.
- Builtin item disable → confirm **Dialog** "May affect dependent forms (mock)."
- Duplicate code on save → inline field error.
- User switches category with unsaved editor open → confirm discard.
- Search category with no matches → "No categories match."
- Sort order collision → allowed (sort by SortOrder then Code).
- Double-click Save in editor → disabled during submit.

## Phase 2: Screen-by-Screen Functional Specifications

### Screen Name: Dictionary Maintenance (`/pms/admin/dictionaries`)

* **Objective:** Two-pane layout for category selection and sub-item management.

* **UI Elements & Field-Level Specs:**
  * **Page header:** "Global Dictionaries"; config permission gate.
  * **Left pane — Category list (`ScrollArea`):**
    * **Search (`Input`):** Filters categories by name/code.
    * **Category rows (`Button` ghost variant):** Show Name, Code; builtin **Badge** when `IsBuiltin`.
    * **Selected state:** Highlight active category.
    * **Add category (`Button` outline, footer):** Only for non-builtin custom categories (mock); opens Category Editor Dialog.
  * **Right pane — Sub-items:**
    * **Category title + builtin badge.**
    * **Add item (`Button`):** Opens Sub-item Editor Dialog (create mode).
    * **Items `Table`:** Columns Code, Display Name, Sort Order, Enabled (**Switch** or **Badge**), Actions (Edit, Delete).
    * **Delete action:** Hidden for builtin categories; **Tooltip** "Built-in items cannot be deleted."
    * **Dropdown preview sample (`Combobox` disabled):** Read-only demo of enabled items for current category.
    * **Empty / loading / error states** per pane.

* **Functional Logic:**
  * Load categories from `src/mocks/pms/dictionaries.ts`.
  * Category select → load `Items` into **Table**.
  * Inline **Switch** toggle enabled → immediate mock `PATCH` → **Toast**; update preview.
  * Delete → **Dialog** confirm → remove row (non-builtin only).

### Screen Name: Sub-item Editor Dialog (modal on `/pms/admin/dictionaries`)

* **Objective:** Create or edit a dictionary item within the selected category.

* **UI Elements & Field-Level Specs:**
  * **Dialog:** Title "Add dictionary item" / "Edit dictionary item".
  * **Code (`Input`):** Required; max 50; unique in category; disabled on edit for builtin items (mock: code immutable when builtin).
  * **Display name (`Input`):** Required; max 200; unique in category.
  * **Sort order (`Input` number):** Required; integer ≥ 0; default 0.
  * **Enable status (`Switch`):** Default true on create.
  * **Cancel / Save (`Button`).**

* **Functional Logic:**
  * Validate with `DictionaryItemSchema` + category-level uniqueness check.
  * Save → mock `POST`/`PUT` → close → refresh **Table** → **Toast** "Dictionary updated — forms refreshed (mock)" → refresh preview **Combobox**.
  * Double-submit prevention.

### Screen Name: Delete Sub-item Dialog (modal on `/pms/admin/dictionaries`)

* **Objective:** Confirm removal of custom dictionary item.

* **UI Elements & Field-Level Specs:**
  * **Dialog:** Warning text; item code + display name.
  * **Cancel / Delete (`Button` destructive).**

* **Functional Logic:**
  * Not rendered for builtin categories.
  * Success → remove from **Table** → **Toast**.

### Screen Name: Category Editor Dialog (modal — custom categories only)

* **Objective:** Add a new non-builtin dictionary category (mock extension).

* **UI Elements & Field-Level Specs:**
  * **Category code (`Input`):** Required; unique globally.
  * **Category name (`Input`):** Required.
  * **Cancel / Create (`Button`).**

* **Functional Logic:**
  * On create: append to left list; auto-select new category with empty **Table**.

## Phase 3: Screen Transition & User Flow Mapping

| # | Trigger Event | Source Screen | Destination Screen | Conditional Paths |
|---|---------------|---------------|--------------------|-------------------|
| 1 | Page load | — | Dictionary Maintenance | Permission denied → 403 |
| 2 | Click category row | Dictionary Maintenance | Dictionary Maintenance (right pane updates) | — |
| 3 | Click **Add item** | Dictionary Maintenance | Sub-item Editor Dialog (create) | — |
| 4 | Click **Edit** on row | Dictionary Maintenance | Sub-item Editor Dialog (edit) | — |
| 5 | Save sub-item | Sub-item Editor Dialog | Dictionary Maintenance | Table + preview refresh |
| 6 | Toggle enabled inline | Dictionary Maintenance | Dictionary Maintenance | Toast; no dialog |
| 7 | Click **Delete** (custom) | Dictionary Maintenance | Delete Sub-item Dialog | Builtin → no delete button |
| 8 | Confirm delete | Delete Dialog | Dictionary Maintenance | Row removed |
| 9 | Click **Add category** | Dictionary Maintenance | Category Editor Dialog | Custom categories feature |
| 10 | Switch category with dirty editor | Sub-item Editor Dialog | Confirm discard → Dictionary Maintenance | Stay → remain in dialog |
