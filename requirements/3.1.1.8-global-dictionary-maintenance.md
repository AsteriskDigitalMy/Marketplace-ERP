# 3.1.1.8 Global Dictionary Maintenance — Frontend Requirements

**Source:** F-BASE-008 — Section 3.1.1 System Basic Management  
**Route:** `/pms/admin/dictionaries`

---

## Functional Summary

Unified maintenance of enumeration dictionaries: leather categories, processes, quality defects, project types, performance tiers, etc. Disabled items hidden from all form dropdowns.

**Prerequisites:** System basic configuration permission.

---

## Layout

Left: dictionary category list (searchable).  
Right: sub-items table for selected category.

### Sub-item table

Columns: Code, Display name, Sort order, Enabled, Actions.

Builtin core dictionaries: **Disable only** — no delete button (tooltip).

### Sub-item editor

| Field | Required |
|-------|----------|
| Code | Yes — unique in category |
| Display name | Yes — unique in category |
| Sort order | Yes |
| Enable status | Toggle |

Save → mock "forms refreshed" toast; disabled items struck through in preview dropdown sample.
