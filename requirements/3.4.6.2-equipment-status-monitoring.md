# 3.4.6.2 Real-Time Equipment Status Monitoring — Frontend Requirements

**Source:** Section 3.4.6.2  
**Purpose:** Mock PLC integration — live status, parameters, faults, OEE inputs.

---

## Screens

### Live monitor (`/mes/equipment/monitor`)

**Grid of equipment cards** with live status (mock polling every 5s):

| State | Visual |
|-------|--------|
| Running | green pulse |
| Standby | blue |
| Fault | red alert |
| Shutdown | gray |

---

### Equipment live detail (`/mes/equipment/monitor/:id`)

**Parameter gauges** (mock): speed, temperature, pressure, runtime, output.

**Sparkline** last hour per parameter.

**Fault panel** — active alarms with acknowledge.

---

### Workshop display (`/mes/display/equipment`)

Large-screen grid — same card layout, minimal chrome.

---

### Alert routing

Fault/anomaly → toast + alert inbox to equipment admin + workshop director (mock).

**Runtime log:** running, downtime, fault duration counters for OEE feed.

---

## Mock Data Shape

```ts
interface EquipmentLiveStatusDto {
  EquipmentId: string
  EquipmentCode: string
  State: 'running' | 'standby' | 'fault' | 'shutdown'
  Parameters: { Name: string; Value: number; Unit: string; IsAnomaly: boolean }[]
  ActiveFaults: { Code: string; Message: string; Since: string }[]
  RunningMinutesToday: number
  DowntimeMinutesToday: number
  FaultMinutesToday: number
}
```

Mocks: `src/mocks/mes/equipment-monitor.ts`.
