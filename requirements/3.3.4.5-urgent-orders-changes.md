# 3.3.4.5 Urgent Orders and Change Processing — Frontend Requirements

**Source:** Section 3.3.4.5

---

## Urgent order insertion (`/scm/scheduling/urgent`)

Fast-track form → priority insert into current schedule.

**Impact analysis panel:**

| Existing order | Original delivery | New projected delivery | Delta days |

Requires approval before apply.

---

## Order change response (automation UI)

When sales order qty/date changes (from 3.3.2.4):

- Banner on scheduling board: "Recalculation required"
- `Recalculate schedule` action
- Cancellation → terminate linked WOs with confirm dialog listing released capacity/materials
