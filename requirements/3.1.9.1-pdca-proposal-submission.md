# 3.1.9.1 Online Submission of Improvement Proposals by All Staff — Frontend Requirements

**Source:** F-PDCA-001 — Section 3.1.9 PDCA Improvement Closed-Loop Management  
**Routes:** `/pms/pdca/proposals`, `/pms/pdca/proposals/new`

---

## Functional Summary

All active employees submit online improvement proposals for production optimization, cost reduction, efficiency enhancement, and management optimization. Initial status is "Pending Evaluation"; auto-routed to Auditor review.

**Prerequisites:** Employee logged in.

---

## My Proposals (`/pms/pdca/proposals`)

Table: Proposal ID, Title, Category, Submitted at, Status, Actions (View).

Status badges: Pending evaluation / Approved / Rejected / In execution ([3.1.9.2](./3.1.9.2-pdca-execution-tracking.md)).

**New proposal** button → form.

---

## Proposal Form (`/pms/pdca/proposals/new`)

| Field | Required | Validation |
|-------|----------|------------|
| Title | Yes | Max 200 chars |
| Category | Yes | Production / cost / efficiency / management |
| Current problem status | Yes | Textarea, min 50 chars |
| Improvement scheme | Yes | Textarea |
| Expected results | Yes | Textarea |
| Attachments | No | Mock file upload (names only) |

Optional pre-fill when opened from HR rectification ([3.1.8.5](./3.1.8.5-hr-rectification.md)): query param `?fromHr=employeeId`.

### Submit

- Success → toast + redirect to detail
- Status `Pending evaluation`
- Mock push to Auditor review inbox (banner)

---

## Proposal Detail (submitter view)

Read-only form fields + status timeline + Auditor comments (when evaluated).
