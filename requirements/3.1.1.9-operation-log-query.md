# 3.1.1.9 Operation Log Query and Traceability — Frontend Requirements

**Source:** F-BASE-009 — Section 3.1.1 System Basic Management  
**Route:** `/pms/admin/logs`

---

## Functional Summary

Query critical operational behaviors with multi-condition filters. View before/after data comparison. Export and archive (mock). Logs are immutable — no delete UI.

**Prerequisites:** System audit or supreme management permission.

---

## Filter Bar

| Filter | Type |
|--------|------|
| Operator account | Autocomplete |
| Operation time range | Date range |
| Business type | Multi-select (org, user, role, KPI, project, etc.) |
| Login IP | Text |

**Search** → paginated table.

---

## Log Table

Columns: Time, Operator, Business type, Target, Action, IP, Detail (link).

### Detail drawer

- Operation metadata
- **Before / After** JSON diff viewer (mock highlighted fields)
- No edit or delete controls

**Export** button → mock CSV download. Archive action → mock confirmation.
