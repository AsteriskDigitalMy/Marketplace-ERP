# 3.4.5.1 In-Process Quality Control (IPQC) — Frontend Requirements

**Source:** Section 3.4.5.1  
**Purpose:** Routing-based inspection points, Pad inspection, NCR handling, alerts, analytics.

---

## Screens

### Inspection point config (`/mes/quality/ipqc/config`)

Per routing step:

| Inspection item | Standard | Sampling % | Mandatory |
|-----------------|----------|------------|-----------|

CRUD table linked to PDM routing (mock).

---

### Pad inspection (`/mes/pad/quality/ipqc`)

1. Scan WO or product RFID
2. Load inspection items + standards
3. Enter results, NC qty, reasons, defect images (upload mock)
4. Submit → pass or fail path

**Fail handling:** rework / scrap / concession (let-go) — creates NCR order.

---

### NCR orders (`/mes/quality/ncr`)

Track disposal progress — status pipeline.

---

### Quality alerts

Rules: same defect连续 at process, defect rate > threshold.

Alert inbox integration with MES dashboard.

---

### IPQC analytics (`/mes/quality/ipqc/reports`)

- Process pass rate table
- Defect Pareto chart (shadcn Chart)
- Defect trend line chart

Filters: workshop, process, date range.

---
