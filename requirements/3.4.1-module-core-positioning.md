# 3.4.1 Core Module Positioning — Frontend Requirements

**Source:** Section 3.4.1 — Manufacturing Execution System (MES)  
**Module role:** Production execution carrier — work orders, scheduling, shop-floor reporting, IPQC, equipment, piece-rate wages. Real-time data via Pad and RFID (mocked).

---

## Overview

MES sits between **SCM** (plans in) and **WMS** (materials / finished goods out). The frontend must support:

- **Admin portal** (PC, 1024px+) — planning, monitoring, approvals, analytics
- **Pad shop-floor UI** (360px+) — scan, Bao-Gong reporting, inspections
- **Workshop display** (large screen) — daily schedule, equipment status, alerts

## Module Dashboard (`/mes`)

**KPI cards:** open work orders, in production, overdue schedules, open IPQC failures, equipment faults, WIP overtime alerts.

**Charts:** work order status distribution, today's plan completion %, defect rate trend (mock).

**Alert feed:** anomalies, material shortages, quality thresholds, equipment faults.

---

## Shared Types (`src/types/mes/`)

`WorkOrder`, `ProcessStep`, `BaoGongReport`, `IpqcInspection`, `Equipment`, `Team`, `WageLine`, `WorkOrderCost`.

---

## Global Patterns

1. **Work order status chip** — consistent across module (see 3.4.2.5).
2. **Process stepper** — routing progress on WO detail and Pad.
3. **ScanField** — work order, RFID, material barcodes (mock).
4. **SOP panel** — linked from PDM/GST process library (mock).
5. **Sync badges** — SCM, WMS, SAP (mock).
