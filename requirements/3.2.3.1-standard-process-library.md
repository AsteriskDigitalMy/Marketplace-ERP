# 3.2.3.1 Standard Process Library Management — Frontend Requirements

**Source:** Section 3.2.3.1  
**Purpose:** Unified standard process library for standardized, reusable process management across the enterprise.

---

## Screens

### 1. Process library list (`/pdm/processes`)

**Data table** with filters and search.

| Column | Sortable |
|--------|----------|
| Process code | Yes — monospace |
| Process name | Yes |
| Process type | Yes — Badge (cutting, sewing, finishing, inspection, …) |
| Workshop | Yes |
| Applicable categories | Yes — multi-badge |
| Has SOP | Icon indicator |
| Has video | Icon indicator |
| Status | Active / Inactive |
| Last updated | Yes |

**Filters:** process type, workshop, product category, has SOP, status.

**Actions:** `New process`, import Excel (stub), export (mock).

---

### 2. Process create / edit (`/pdm/processes/new`, `/pdm/processes/:id/edit`)

#### Basic information form

| Field | Type | Required |
|-------|------|----------|
| Process code | Text | Yes, unique |
| Process name | Text | Yes |
| Process type | Select | Yes |
| Assigned workshop | Select | Yes |
| Applicable product categories | Multi-select | Yes, min 1 |
| Description | Textarea | No |
| Status | Toggle | Active default |

#### SOP & media section

| Asset type | UI |
|------------|-----|
| SOP document | PDF upload + preview |
| Operational video | Video upload or URL + embedded player |
| Process parameters | Key-value table editor (param name, value, unit) |
| Quality inspection standards | Rich text or structured checklist builder |

**Parameter table:** add/remove rows; validate numeric fields.

**Checklist builder:** ordered list of inspection items with pass/fail criteria text.

#### Pad access preview

Toggle `Preview pad view` — renders read-only mobile layout:

- Large process name header
- SOP PDF link / video player
- Parameters as definition list
- Inspection checklist

Route mock: `/pdm/processes/:id/pad`.

---

### 3. Process detail (`/pdm/processes/:id`)

**Tabs:** Overview, SOP & media, Parameters, Inspection standards, Usage (where used in routings).

**Usage tab:** table of product routings referencing this process (mocked cross-links).

**Actions:** Edit, Duplicate, Deactivate (with confirm if used in active routings).

---

## Process Reuse UX

When this library is consumed by routing configuration (3.2.3.3):

- Provide **process picker** `Command` combobox: search by code, name, type.
- Selecting a process copies reference (not duplicate data) — show linked badge on routing step.

Export reusable `ProcessPicker` component from `src/components/shared/`.

## Mock Data Shape

```ts
interface StandardProcessDto {
  Id: string
  ProcessCode: string
  ProcessName: string
  ProcessType: 'cutting' | 'sewing' | 'finishing' | 'inspection' | string
  Workshop: string
  ApplicableCategories: string[]
  Description: string | null
  Status: 'active' | 'inactive'
  SopDocument: AttachmentDto | null
  OperationalVideo: { Url: string; FileName: string } | null
  ProcessParameters: { Name: string; Value: string; Unit: string }[]
  InspectionStandards: { Item: string; Criteria: string; Order: number }[]
  UsedInRoutingCount: number
  CreatedDatetime: string
  ModifiedDatetime: string | null
}
```

Mocks: `src/mocks/pdm/processes.ts`. List uses `PagedList<StandardProcessDto>`.
