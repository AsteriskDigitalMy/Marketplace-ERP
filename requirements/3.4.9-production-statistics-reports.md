# 3.4.9 Production Data Statistics and Report Management — Frontend Requirements

**Source:** Section 3.4.9  
**Purpose:** Fixed production reports, custom builder, export/print, scheduled dissemination.

---

## Screens

### Reports hub (`/mes/reports`)

#### Fixed templates

| Report | Route |
|--------|-------|
| Production daily/weekly/monthly | `/mes/reports/production-summary` |
| Work order progress | `/mes/reports/wo-progress` |
| Capacity load | `/mes/reports/capacity` |
| Equipment OEE | links 3.4.6.4 |
| Quality pass rate | links 3.4.5.4 |
| Personnel efficiency | `/mes/reports/personnel-efficiency` |
| Material loss | `/mes/reports/material-loss` |
| Cost accounting | links 3.4.8 |

Each card: run, last generated, export.

---

### Standard report viewer

Shared layout: filters, KPI summary, table, optional chart, Export Excel/PDF, Print.

---

### Custom report builder (`/mes/reports/custom`)

Configure dimensions: workshop, line, WO, product, process, operator, date.

Metrics multi-select: output, hours, defect rate, OEE, cost, etc.

Save template; run ad-hoc.

---

### Subscriptions (`/mes/reports/subscriptions`)

Schedule push to management emails (mock) — same pattern as WMS 3.5.8.

---

### Workshop print view

Print-optimized CSS for floor posting — daily production summary.

---

### Display viewport (`/mes/display/reports`)

Rotate core KPI reports on large screen (mock carousel).
