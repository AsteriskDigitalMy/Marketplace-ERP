# 3.3.2.4 Order Change Management — Frontend Requirements

**Source:** Section 3.3.2.4

---

## Screens

### Change request list (`/scm/orders/changes`)

| Change # | Order # | Type | Status | Impact tags |

**Change types:** qty +/-, delivery date, configuration, process requirements, cancellation.

---

### New change request (`/scm/orders/:id/change`)

| Field | |
|-------|--|
| Change type | Select |
| Reason | Textarea |
| Content | Textarea |
| Impact scope | Multi: Production, Procurement, Cost, Warehouse |

Diff preview: before/after order lines and dates.

Re-triggers multi-department review (link 3.3.2.3).

---

### Post-approval

Notification panel: departments notified (mock). Schedule/MRP refresh banner.

**Archive:** permanent change history on order detail tab.

---
