# 3.5.7 Full-Process Material Traceability — Frontend Requirements

**Source:** Section 3.5.7  
**Purpose:** Forward trace (batch → downstream) and backward trace (finished goods → components & supplier).

---

## Screens

### 1. Traceability search hub (`/wms/traceability`)

**Dual-mode search** (tabs or toggle):

| Mode | Input | Placeholder |
|------|-------|-------------|
| Forward | Material batch number | `BATCH-2026-0042` |
| Backward | Finished goods batch or serial | `SN-FG-88901` |

Large search field + `Trace` primary button.

**Recent traces** list below (mock history).

---

### 2. Forward trace results (`/wms/traceability/forward/:batchNumber`)

**Graph or timeline visualization** (MVP: vertical timeline).

#### Nodes (expandable cards)

1. **Inbound receipt** — supplier, arrival date, inbound voucher #
2. **Supplier identity** — name, code, contact
3. **Quality metrics** — IQC result, report #
4. **Production consumption** — work order #, line, date
5. **Operations (工序)** — step list with operators, machines (mock MES data)
6. **Finished goods batch** — output batch #
7. **Sales order** — order #, customer
8. **Customer profile** — name, region
9. **Shipping** — carrier, tracking, dispatch date

**Edge links:** click node → detail `Sheet`.

---

### 3. Backward trace results (`/wms/traceability/backward/:id`)

Reverse timeline from finished goods batch or serial:

1. Manufacturing work order
2. Operation steps + line workers + machinery
3. Component material batches (table with forward trace links)
4. Original supplier info per component batch
5. Inspection histories
6. Inbound vouchers

**Cross-link:** click component batch → open forward trace in new tab.

---

### 4. Export

**Actions:** Export PDF, Export Excel (mock download).

**Export contents:** full trace chain summary for compliance / root-cause / supplier claims.

Print-friendly CSS for audit packages.

## Mock Data Shape

```ts
interface ForwardTraceResultDto {
  BatchNumber: string
  MaterialCode: string
  MaterialName: string
  Chain: {
    Stage: 'inbound' | 'supplier' | 'quality' | 'production' | 'operations' | 'finished_goods' | 'sales' | 'customer' | 'shipping'
    Title: string
    Timestamp: string | null
    Details: Record<string, string>
    RelatedId: string | null
  }[]
}

interface BackwardTraceResultDto {
  Identifier: string
  IdentifierType: 'finished_batch' | 'serial'
  ProductCode: string
  Chain: {
    Stage: string
    Title: string
    Details: Record<string, string>
    ComponentBatches: { BatchNumber: string; MaterialCode: string }[] | null
  }[]
}
```
