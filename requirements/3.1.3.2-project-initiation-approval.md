# 3.1.3.2 Project Initiation Workflow Approval — Frontend Requirements

**Source:** F-PROJ-002 — Section 3.1.3 Project Management  
**Route:** `/pms/projects/approvals`

---

## Functional Summary

Auditors approve or reject project initiation applications. Pass → `In Progress` + Gantt sync; reject → return to initiator with required reason.

**Prerequisites:** Project approval permission; status `Pending Approval`.

---

## Approval Inbox

Table: Project code, Name, Leader, Budget, Submitted at, Actions (Review).

---

## Review Detail

Read-only initiation materials + bound KPI list + budget summary.

| Field | Required |
|-------|----------|
| Decision | Approve / Reject |
| Opinion | Required on reject |

**Approve:** status → `In Progress`; Gantt initialized (mock); notify leader toast.  
**Reject:** status → `Returned`; leader edit CTA.
