# 3.4.7.3 Piece-Rate Wage Automatic Calculation — Frontend Requirements

**Source:** Section 3.4.7.3  
**Purpose:** Daily wage calc from piece rates / hourly rates; Pad/mobile detail; monthly approval → SAP.

---

## Screens

### Rate management (`/mes/wages/rates`)

| Process | Product | Skill level | Piece rate | Hourly rate alt |
|---------|---------|-------------|------------|-----------------|

CRUD — used in calculation engine (mock).

---

### Daily wage run (`/mes/wages/daily`)

Date picker → `Calculate` (mock).

Table per operator:

| Operator | Qualified wages | Rework wages | OT wages | Bonus | Deductions | Net |
|----------|-----------------|--------------|----------|-------|------------|-----|

Drill-down to output and labor lines.

---

### Pad wage detail (`/mes/pad/wages`)

Operator views own daily breakdown — output lines, hour lines, net pay.

---

### Monthly approval (`/mes/wages/monthly`)

Aggregate month → approval workflow → **SAP payroll sync** badge on approve.

---

## Mock Data Shape

```ts
interface WageCalculationDto {
  OperatorId: string
  OperatorName: string
  Date: string
  QualifiedOutputWages: number
  ReworkWages: number
  OvertimeWages: number
  AttendanceBonus: number
  Deductions: number
  NetWage: number
  Details: { Type: string; Description: string; Amount: number }[]
}

interface PieceRateDto {
  ProcessId: string
  ProductCategory: string
  SkillLevel: number
  PieceRate: number
  HourlyRate: number
}
```
