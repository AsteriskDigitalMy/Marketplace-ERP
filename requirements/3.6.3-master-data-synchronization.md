# 3.6.3 Master Data Synchronization Management — Frontend Requirements

**Source:** Section 3.6.3  
**Principle:** SAP is source of truth for most master data; org/plant/warehouse flows operational → SAP after approval.

---

## Screens

### Master data sync hub (`/sap/master-data`)

**Card per type:**

| Type | Direction | Mode |
|------|-----------|------|
| Material | SAP → Ops | Real-time |
| Supplier | SAP → Ops | Real-time |
| Customer | SAP → Ops | Real-time |
| G/L accounts | SAP → Ops | Daily |
| Cost / profit centers | SAP → Ops | Daily |
| Org / plant / warehouse | Ops → SAP | On approval |

Each card: last sync, record count, failed count, `View log` link.

---

### Sync detail (`/sap/master-data/:type`)

**Inbound from SAP (table):**

| Code | Name | Last synced | Status | Validation |

Failed rows: missing mandatory fields — error message column.

**Outbound to SAP (org structure):**

Approval queue before sync — link to approval workflow.

---

### Change notifications

Inbox panel: "Customer C-100 credit limit updated in SAP" → affected open documents listed (mock).

---

### Sync history (`/sap/master-data/history`)

Permanent log filterable by type, time, status, code.

| Timestamp | Type | Direction | Records | Status | Message |

`Re-sync` action on failed batch.

---

## Scope table UI

Render the spec table as read-only reference panel on hub page.

## Mock Data Shape

```ts
interface MasterDataSyncRecordDto {
  Id: string
  DataType: 'material' | 'supplier' | 'customer' | 'gl_account' | 'cost_center' | 'org_structure'
  Direction: 'sap_to_ops' | 'ops_to_sap'
  SyncMode: 'realtime' | 'scheduled' | 'on_approval'
  EntityCode: string
  EntityName: string
  SyncedAt: string
  Status: 'success' | 'failed' | 'pending'
  ErrorMessage: string | null
  PayloadSummary: Record<string, string>
}
```

## Acceptance Criteria

- [ ] Hub shows all master data types with direction and mode
- [ ] Failed syncs show validation errors for missing mandatory fields
- [ ] Org structure outbound requires approval before sync
- [ ] Change notification panel for SAP updates
- [ ] Permanent sync history with re-sync action
- [ ] Four async states throughout
