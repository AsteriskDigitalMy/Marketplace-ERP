# 3.4.2.1 Work Order Reception and Creation — Frontend Requirements

**Source:** Section 3.4.2.1  
**Purpose:** Auto-receive SCM plans as work orders; manual creation for stocking, rework, trial production.

---

## Screens

### Work order list (`/mes/work-orders`)

| Column | |
|--------|--|
| Work order # | Link |
| Source | SCM auto / Manual badge |
| Type | Standard, Stocking, Rework, Trial |
| Product / style | |
| Quantity | Tabular nums |
| Workshop / line | |
| Planned start–end | |
| Status | |
| Approval | For manual types |

**Filters:** status, type, workshop, date range, source.

**Actions:** `New work order`, `Import from SCM` (mock refresh).

---

### Auto-generated WO (from SCM)

**Inbound queue tab** `SCM dispatches`:

- List of SCM scheduling results not yet converted
- `Generate work order` per row — pre-fills:
  - Style number, qty, BOM version, routing, planned dates, linked orders, customer
- Bulk generate (mock)

---

### Manual work order form (`/mes/work-orders/new`)

| Field | Required |
|-------|----------|
| Work order number | Auto or manual per config |
| WO type | Stocking / Rework / Trial |
| Product information | Style picker |
| Production quantity | Yes |
| Routing | Version select |
| BOM version | Select |
| Planned start / end | Yes |
| Workshop / production line | Yes |
| Linked order (if rework/trial) | Conditional |

**Approval workflow** for manual types before dispatch.

---

### Work order detail (`/mes/work-orders/:id`)

Tabs: Overview, BOM, Routing, Materials, Progress, Approval, Related SCM order.

Read-only pulled fields from PDM/SCM with sync timestamp.

---
