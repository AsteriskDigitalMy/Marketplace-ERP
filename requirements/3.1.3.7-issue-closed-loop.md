## Functional Summary

Project leaders assign handlers, set deadlines, track rectification, and close verified issues. Full disposal workflow log retained.

**Prerequisites:** Logged-in user is project leader.

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Role | Goals | Pain Points |
|---------|------|-------|-------------|
| **Project Leader** | Owns issue resolution | Assign and verify fixes quickly | Bulk issues after milestone slip |
| **Issue Handler** | Executes remediation | Clear deadline and measures | Ambiguous resolution expectations |
| **Auditor (read-only)** | Reviews closure evidence | Complete workflow log | Missing timestamps |

### Core Capabilities (FR-xxx)

| ID | Capability | Priority |
|----|------------|----------|
| FR-001 | Issue aggregator table at `/pms/projects/:id/issues` | Must |
| FR-002 | Bulk assign: handler + deadline + measures template | Must |
| FR-003 | Handler workspace for disposal results + evidence upload (mock) | Must |
| FR-004 | Leader verification: Close or Return with comments | Must |
| FR-005 | Timeline panel: reported тЖТ assigned тЖТ rectified тЖТ closed | Must |
| FR-006 | Permission gate: project leader only for assign/verify | Must |
| FR-007 | Clear task risk badge on close ([3.1.3.6](./3.1.3.6-issue-reporting.md)) | Should |

### Edge Cases & Alternative Paths

- **Non-leader access:** 403 empty state with explanation.
- **Assign without handler:** Validation block.
- **Return to handler:** Status `assigned`; handler notified toast.
- **Close without evidence:** Warning confirm if handler skipped upload.
- **Bulk assign mixed statuses:** Only `open` issues selectable.
- **All issues closed:** Success empty state; risk banner clears on project detail.

## Phase 2: Screen-by-Screen Functional Specifications

### Screen: Issue Aggregator тАФ `/pms/projects/:id/issues`

* **Objective:** Leader views and manages all project issues.

* **UI Elements & Field-Level Specs:**
  - `Card` header: "Project Issues" + status filter `Tabs` (Open | Assigned | Resolved | Closed | All).
  - `DataTable` with row `Checkbox`: Issue ID, Task (link to [3.1.3.3](./3.1.3.3-sub-task-creation.md)), Description (truncate), Resource Type `Badge`, Status `Badge`, Handler, Deadline, Actions.
  - Bulk action bar: `Button` "Bulk Assign" тЖТ `Dialog`.
  - `Button` per row: View timeline, Assign, Verify (contextual by status).

* **Functional Logic:**
  - Load `IssueDisposalDto` from mock; leader permission check on mount.
  - Filters update table client-side or mock API.

---

### Modal: Bulk Assign тАФ `Dialog`

* **Objective:** Assign multiple open issues at once.

* **UI Elements & Field-Level Specs:**
  - Selected count label.
  - **Handler** тАФ `Combobox` user picker, required.
  - **Deadline** тАФ `Calendar`, required.
  - **Resolution measures template** тАФ `Textarea`, optional preset `Select`.
  - Confirm `Button`.

* **Functional Logic:**
  - Updates status `assigned`; appends disposal log entries; mock notify handlers.

---

### Screen: Handler Workspace тАФ `/pms/projects/:id/issues/:issueId` or `Sheet`

* **Objective:** Assigned handler submits disposal results.

* **UI Elements & Field-Level Specs:**
  - Read-only issue context `Card`.
  - `Form`:
    - **Disposal result** тАФ `Textarea`, required.
    - **Evidence** тАФ file `Input` mock multi-upload.
  - Submit тЖТ status `resolved`; notify leader.

* **Functional Logic:**
  - Gate: current user === `HandlerId`.
  - Leader sees "Pending verification" after submit.

---

### Modal: Leader Verification тАФ `Dialog`

* **Objective:** Leader closes loop or returns to handler.

* **UI Elements & Field-Level Specs:**
  - Disposal result and evidence links read-only.
  - **Comments** тАФ `Textarea`, required on Return.
  - `RadioGroup`: Close | Return.
  - Submit decision.

* **Functional Logic:**
  - Close тЖТ status `closed`; append log; clear task risk [3.1.3.6](./3.1.3.6-issue-reporting.md).
  - Return тЖТ status `assigned`; handler toast.

---

### Panel: Disposal Timeline тАФ `Sheet` or side `Card`

* **Objective:** Visual workflow audit trail.

* **UI Elements & Field-Level Specs:**
  - Vertical `ScrollArea` timeline with `Separator` nodes: At, Actor, Action, Detail.
  - States: reported тЖТ assigned тЖТ rectified тЖТ closed.

* **Functional Logic:**
  - Read-only from `DisposalLog` array.
