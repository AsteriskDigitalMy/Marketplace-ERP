# 3.1.1.6 System Role Permission Configuration — Frontend Requirements

**Source:** F-BASE-006 — Section 3.1.1 System Basic Management  
**Route:** `/pms/admin/roles`

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Goals |
|---------|-------|
| **Supreme System Administrator (primary)** | Define roles with precise menu, button, and data-scope permissions; ensure changes propagate to bound users. |
| **Security Architect** | Enforce least privilege — minimum necessary menus and data visibility per role. |
| **Department Admin (read-only)** | View role definitions to understand what subordinates can access (mock read-only mode). |
| **Bound End User (indirect)** | Experiences permission refresh immediately after admin saves role (mock Toast to admin). |

### Core Capabilities

1. **FR-001 — Role list:** Paginated **Table** of roles with name, description, bound user count, updated at, actions.
2. **FR-002 — Role create/edit:** Forms at `/pms/admin/roles/new` and `/pms/admin/roles/:id` aligned with `SystemRoleSchema` in `src/models/pms/identity.ts`.
3. **FR-003 — Unique role name:** Block save when name duplicates existing role (exclude self on edit).
4. **FR-004 — Menu permission tree:** Checkbox tree for PMS, SCM, MES modules; parent-child cascade with indeterminate state.
5. **FR-005 — Button permissions:** Per checked menu, sub-checkboxes for view/add/edit/delete/approve/export per `ButtonPermissionSchema`.
6. **FR-006 — Data visibility scope:** Radio group — company, department, individual, custom; custom requires filter builder text per schema superRefine.
7. **FR-007 — Minimum permission rule:** At least one menu permission required (`MenuPermissions.min(1)`).
8. **FR-008 — Role delete:** Allowed only when `BoundUserCount = 0`; else **Toast** block.
9. **FR-009 — Real-time effect mock:** On save, **Toast** "{N} bound users will see updated permissions."
10. **FR-010 — Async states:** Loading, empty, error, success on list and editor.

### Edge Cases & Alternative Paths

- User checks parent menu then unchecks all children manually → parent becomes indeterminate or unchecked per tree rules.
- Custom data scope selected but empty filter → inline validation error.
- Delete role with users → delete button disabled + tooltip.
- User saves with duplicate name → inline error on name field.
- Navigate away with unsaved permission changes → discard confirm **Dialog**.
- New role save success → redirect to `/pms/admin/roles/:id` or list with **Toast**.
- Double-click Save → disabled during submit.
- Empty role list → CTA "Create first role."

## Phase 2: Screen-by-Screen Functional Specifications

### Screen Name: Role List (`/pms/admin/roles`)

* **Objective:** Browse and manage system roles; entry to create and edit.

* **UI Elements & Field-Level Specs:**
  * **Page header:** "System Roles"; supreme-admin permission gate.
  * **New Role (`Button`):** → `/pms/admin/roles/new`.
  * **Search (`Input`):** Filter by name; debounced.
  * **Roles `Table`:** Columns Name, Remarks (truncated), Bound User Count, Updated At, Actions.
  * **Actions column:** Edit (`Button` ghost) → `/pms/admin/roles/:id`; Delete (`Button` ghost destructive) — disabled when `BoundUserCount > 0` with **Tooltip**.
  * **Delete confirmation `Dialog`:** "Delete role {name}?" — only when count = 0.
  * **Empty / loading / error states:** Standard patterns.

* **Functional Logic:**
  * Data from `src/mocks/pms/system-roles.ts`.
  * Delete → mock `DELETE` → remove row → **Toast** success.

### Screen Name: Role Editor (`/pms/admin/roles/new`, `/pms/admin/roles/:id`)

* **Objective:** Configure role metadata, menu/button permissions, and data scope.

* **UI Elements & Field-Level Specs:**
  * **Basic info card:**
    * **Role name (`Input`):** Required; max 100; unique validation async on blur.
    * **Remarks (`Textarea`):** Optional; max 500.
  * **Menu permission tree (`ScrollArea` + checkboxes):** Modules PMS / SCM / MES with nested menus; indeterminate parents.
  * **Button permissions panel:** Visible for each checked menu; six checkboxes per menu row in sub-**Table**.
  * **Data visibility (`RadioGroup`):** Options: Entire company | Current department only | Individual business data only | Custom filter.
  * **Custom filter builder (`Textarea` or rule rows `Table`):** Required when scope = custom; max 2000 chars; mock rule rows UI acceptable.
  * **Sticky footer:** Cancel → list; Save (`Button`) — disabled when invalid or pristine or submitting.
  * **Bound users callout (`Alert`):** On edit when `BoundUserCount > 0`: "Changes affect {N} users immediately (mock)."

* **Functional Logic:**
  * Load role on edit; empty defaults on new.
  * Validate full `SystemRoleSchema` including custom filter superRefine.
  * Save mock `POST`/`PUT` → **Toast** with bound user count message.
  * Menu tree: checking parent checks all children; unchecking parent unchecks children.

### Screen Name: Delete Role Dialog (modal on `/pms/admin/roles`)

* **Objective:** Confirm deletion of unused role.

* **UI Elements & Field-Level Specs:**
  * **Dialog:** Title + role name; irreversible warning.
  * **Cancel / Delete (`Button` destructive).**

* **Functional Logic:**
  * Only reachable when `BoundUserCount === 0`.
  * On success: close dialog; refresh list.

## Phase 3: Screen Transition & User Flow Mapping

| # | Trigger Event | Source Screen | Destination Screen | Conditional Paths |
|---|---------------|---------------|--------------------|-------------------|
| 1 | Page load | — | Role List | Non-supreme → 403 |
| 2 | Click **New Role** | Role List | Role Editor (`/pms/admin/roles/new`) | — |
| 3 | Click **Edit** | Role List | Role Editor (`/pms/admin/roles/:id`) | — |
| 4 | Click **Save** (valid, new) | Role Editor (new) | Role Editor (`:id`) or Role List | Toast with user count |
| 5 | Click **Save** (valid, edit) | Role Editor (`:id`) | Role Editor (same) | Toast; stay on page |
| 6 | Click **Cancel** | Role Editor | Role List | Dirty → confirm |
| 7 | Click **Delete** (0 users) | Role List | Delete Role Dialog | — |
| 8 | Confirm delete | Delete Dialog | Role List | Row removed |
| 9 | Delete with bound users | Role List | Role List | Button disabled; no dialog |
