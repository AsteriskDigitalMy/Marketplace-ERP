# 3.1.1.6 System Role Permission Configuration — Frontend Requirements

**Source:** F-BASE-006 — Section 3.1.1 System Basic Management  
**Route:** `/pms/admin/roles`

---

## Functional Summary

Customize roles with menu, button, and data-scope permissions. Changes take effect for bound users in real time.

**Prerequisites:** Supreme system administrator permission.

---

## Role List

Columns: Role name, Description, User count, Updated at, Actions (Edit, Delete if no users).

---

## Role Editor (`/pms/admin/roles/new`, `/pms/admin/roles/:id`)

### Basic info

| Field | Required |
|-------|----------|
| Role name | Yes — unique |
| Remarks | Optional |

### Menu permission tree

Checkboxes for module menus (PMS, SCM, MES, etc.). Parent checks children (indeterminate state).

### Button permissions (per checked menu)

Sub-checkboxes: View, Add, Edit, Delete, Approve, Export.

### Data visibility scope

Radio + optional filter builder:

- Entire company
- Current department only
- Individual business data only
- Custom filter conditions (mock rule rows)

**Validation:** at least one business permission checked; duplicate role name blocked.

---

interface SystemRoleDto {
