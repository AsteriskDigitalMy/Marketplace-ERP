# 3.3.2.6 Order Shipment and Closure Management — Frontend Requirements

**Source:** Section 3.3.2.6

---

## Screens

### Shipment plans (`/scm/orders/shipments`)

Auto-generated from delivery dates → WMS prep notice (mock push).

| Order # | Plan date | Qty | WMS status |

---

### Shipment execution

On ship: update shipped/unshipped qty, generate dispatch bill, **SAP AR sync** badge.

---

### Reconciliation (`/scm/orders/:id/reconciliation`)

Match: shipped qty/amount vs invoiced vs payments received.

Variance rows highlighted.

---

### Closure (`/scm/orders/:id/close`)

Preconditions: all shipped, payments complete (checklist).

**Achievement report:** planned vs actual delivery, cost, qty.

Post-close: read-only archive, SAP synced.

---

## Mock Data Shape

```ts
interface OrderClosureDto {
  OrderId: string
  ShippedQty: number
  UnshippedQty: number
  Reconciliation: { ShippedAmount: number; InvoicedAmount: number; CollectedAmount: number }
  AchievementReport: {
    PlannedDelivery: string
    ActualDelivery: string
    PlannedCost: number
    ActualCost: number
    PlannedQty: number
    ActualQty: number
  }
  SapArSyncStatus: string
  IsClosed: boolean
}
```
