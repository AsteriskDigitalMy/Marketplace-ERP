# 3.2.2.2 Product Design Management — Frontend Requirements

**Source:** Section 3.2.2.2  
**Purpose:** Online management and version control for product drawings, design documents, and style information.

---

## Screens

### 1. Style library list (`/pdm/designs`)

**Layout:** Grid or table toggle.

| Field | Display |
|-------|---------|
| Style number | Monospace, link to detail |
| Style name | Primary label |
| Category | Badge |
| Target customer | Text |
| Current version | `v{n}` badge |
| Status | Draft / In review / Approved |
| Last modified | Relative date |
| Thumbnail | Image preview if available |

**Filters:** category, status, target customer, style number search.

**Actions:** `New style`, import (disabled stub), bulk export (mock).

---

### 2. Style create / edit (`/pdm/designs/new`, `/pdm/designs/:id/edit`)

#### Core fields

| Field | Type | Required |
|-------|------|----------|
| Style name | Text | Yes |
| Style number | Text | Yes, unique (async mock validation) |
| Category | Select | Yes |
| Design description | Rich text or textarea | Yes |
| Target customers | Multi-select combobox | No |
| Linked project | Combobox | No — filter active projects |

**Save actions:** `Save draft`, `Submit design review`.

---

### 3. Style detail (`/pdm/designs/:id`)

**Tabs:**

#### Tab: Overview

Read-only summary of style metadata + linked project.

#### Tab: Documents & drawings

**File manager UI:**

| Feature | Detail |
|---------|--------|
| Upload | Drag-and-drop; types: CAD (DWG, DXF), images (PNG, JPG, TIFF), PDF |
| Preview | Inline image preview; PDF in embedded viewer; CAD shows icon + metadata only |
| Download | Per-file download button (mocked SAS URL) |
| Categorize | Tag per file: Drawing, Design doc, Technical process requirement |

**Table columns:** file name, type, category tag, version, uploaded by, date, actions (preview, download, delete).

#### Tab: Versions

**Version history table:**

| Version | Date | Modified by | Change summary | Actions |
|---------|------|-------------|----------------|---------|
| v3 | 2026-06-10 | Jane Doe | Updated sleeve pattern | Compare, Rollback |
| v2 | 2026-05-28 | John Smith | Initial revision | Compare, Rollback |

**Compare view (`Dialog` or dedicated route):**

- Side-by-side metadata diff (highlight changed fields).
- Document list diff: added / removed / changed files.
- Rollback requires `ConfirmDialog` with typed style number confirmation.

**Version creation rule (UI messaging):** every save after approval creates a new version — show info `Alert` when user edits an approved style.

#### Tab: Design review

Same `ApprovalTimeline` pattern as projects.

On approval success:

- Toast: "Design approved — ready for sampling"
- Status badge → `Approved`
- Mock banner: "Will sync to sampling management" (no real sync)

---

## Version Management UX

### Version selector (header component)

Dropdown in style detail header: `v3 (current)` with list of historical versions. Selecting a version loads read-only view with banner: "Viewing historical version v2".

### Rollback flow

1. User clicks Rollback on version row.
2. `ConfirmDialog`: "Rollback style {number} to v2? Current version will be preserved in history."
3. Mock success → new version v4 created from v2 content (explain in toast).

## Mock Data Shape

```ts
interface ProductStyleDto {
  Id: string
  StyleNumber: string
  StyleName: string
  Category: string
  DesignDescription: string
  TargetCustomers: string[]
  ProjectId: string | null
  CurrentVersion: number
  Status: 'draft' | 'in_review' | 'approved'
  Documents: StyleDocumentDto[]
  Versions: StyleVersionDto[]
  ApprovalSteps: ApprovalStepDto[]
  CreatedDatetime: string
  ModifiedDatetime: string | null
}

interface StyleDocumentDto {
  Id: string
  FileName: string
  FileType: string
  Category: 'drawing' | 'design_doc' | 'technical_requirement'
  Version: number
  UploadedBy: string
  UploadedDatetime: string
  PreviewUrl: string | null
  DownloadUrl: string
}

interface StyleVersionDto {
  Version: number
  ModifiedBy: string
  ModifiedDatetime: string
  ChangeSummary: string
  Snapshot: Partial<ProductStyleDto>
}
```
