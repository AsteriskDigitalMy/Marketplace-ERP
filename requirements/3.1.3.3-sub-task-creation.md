# 3.1.3.3 Sub-task Creation and Personnel Assignment — Frontend Requirements

**Source:** F-PROJ-003 — Section 3.1.3 Project Management  
**Route:** `/pms/projects/:id/tasks`

---

## Functional Summary

Decompose projects into sub-tasks with owners, duration, acceptance criteria, update cycle, and prerequisite dependencies. Created tasks push to executor to-do lists.

**Prerequisites:** Project `In Progress`; task assignment permission.

---

## Task List on Project Detail

Table: Task name, Owner, Planned duration, Dependencies, Status, Actions.

**New sub-task** dialog:

| Field | Required |
|-------|----------|
| Task name | Yes |
| Executor | Yes |
| Planned duration | Yes — ≤ project span |
| Workload | Optional |
| Acceptance criteria | Yes |
| Update cycle | Yes (daily/weekly) |
| Prerequisites | Optional multi-select tasks |

**Validation:** no circular dependencies (mock graph check); duration within project dates.

On create → executor to-do badge +1 (mock).
