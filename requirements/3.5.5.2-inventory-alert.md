# 3.5.5.2 Inventory Alert Management — Frontend Requirements

**Source:** Section 3.5.5.2  
**Purpose:** Multi-scenario automated alerts — thresholds, expiry, stagnant stock, location exceptions.

---

## Screens

### 1. Alert inbox (`/wms/alerts`)

**Unified alert feed** with severity and category filters.

| Column | |
|--------|--|
| Severity | Critical / Warning / Info |
| Category | Threshold / Expiry / Stagnant / Location |
| Subject | |
| Material or location | |
| Triggered at | |
| Status | Open / Acknowledged / Resolved |
| Assigned to | |

**Bulk actions:** acknowledge, assign, resolve (with note).

---

### 2. Alert detail (`/wms/alerts/:id`)

- Full message + threshold that triggered
- Related material/location link
- Recommended actions (mock suggestions per type)
- Resolution form: action taken, resolved by, timestamp
- History thread

---

### 3. Alert category views (tabs or sub-routes)

#### Inventory threshold

Cards: below safety, below minimum, above maximum.

Table: material, current qty, threshold, gap, notify targets (procurement, planning, warehouse).

#### Shelf-life / expiration

Table: material, batch, expiry date, days remaining, status (approaching / expired).

Color: warning within N days; destructive if expired.

#### Stagnant stock

Table: material, batch, location, last movement date, days idle.

Link to stagnant inventory report (3.5.8).

#### Location exceptions

| Exception | Display |
|-----------|---------|
| Over-capacity | Location, capacity %, materials |
| Unexpected vacant | Location expected occupied |
| Lock timeout | Locked since, timeout exceeded |

---

### 4. Alert configuration shortcut

Link to `/wms/strategies/alerts` for threshold editing.

## Mock Data Shape

```ts
type AlertCategory = 'threshold' | 'expiry' | 'stagnant' | 'location'
type AlertSeverity = 'critical' | 'warning' | 'info'

interface WarehouseAlertDto {
  Id: string
  Category: AlertCategory
  Severity: AlertSeverity
  Subject: string
  Message: string
  MaterialCode: string | null
  LocationCode: string | null
  TriggeredAt: string
  Status: 'open' | 'acknowledged' | 'resolved'
  AssignedTo: string | null
  ResolutionNote: string | null
  Metadata: Record<string, unknown>  // threshold values, days idle, etc.
}
```
