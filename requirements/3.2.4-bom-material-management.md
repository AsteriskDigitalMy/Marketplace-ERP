# 3.2.4 BOM Material Management — Frontend Requirements

**Source:** Section 3.2.4  
**Purpose:** Multi-level BOM creation, audit, version changes, and lifecycle management — foundation for procurement, production, and cost accounting.

---

## Screens

### 1. BOM list (`/pdm/bom`)

| Column | |
|--------|--|
| Style / product | Link |
| BOM version | Badge |
| Levels | FG / SF / RM / AM counts |
| Status | Draft / In review / Approved / Superseded |
| Effective date | |
| Last modified | |
| Actions | View, Edit, Copy, Analyze |

**Filters:** product, status, version.

**Actions:** `New BOM`, `Import Excel` (upload dialog stub), `Copy from style` (picker dialog).

---

### 2. Multi-level BOM editor (`/pdm/bom/:id/edit`)

**Tree table** (expandable rows) with four hierarchy levels:

| Level | Label | Color cue |
|-------|-------|-----------|
| L0 | Finished goods | Primary |
| L1 | Semi-finished goods | Secondary |
| L2 | Raw materials | Default |
| L3 | Auxiliary materials | Muted |

#### Row columns

| Column | Editable |
|--------|----------|
| Material code | Yes — combobox from material master |
| Material name | Auto |
| Specification | Auto |
| Unit | Auto |
| Unit consumption | Number |
| Wastage rate (%) | Number |
| Effective consumption | Calculated |
| Source type | Select: In-house / Procurement / Subcontracting |
| Level | Select (or inferred by tree depth) |

**Tree actions:** add child, add sibling, delete node (confirm if children exist), expand/collapse all.

**Toolbar:**

- `Duplicate from style` — modal to pick source style BOM
- `Import Excel` — file upload → mock parsed rows preview table before apply
- `Validate` — runs client validation rules

---

### 3. BOM version management

**Version panel** on detail page:

| Version | Date | Author | Change summary | Status |
|---------|------|--------|----------------|--------|
| v4 | 2026-06-12 | Jane | Updated lining material | In review |
| v3 | 2026-05-01 | John | Initial mass prod | Approved |

**New version on edit:** require change reason + change description (textarea).

**Compare versions (`Dialog`):**

- Tree diff: added / removed / modified nodes highlighted.
- Field-level diff for consumption and wastage changes.

---

### 4. BOM change approval workflow

**Multi-department approval UI:**

| Department | Status |
|------------|--------|
| Technical | Pending / Approved / Rejected |
| Production | |
| Procurement | |
| Finance | |

Submit disabled until validation passes.

On approval (mock):

- Toast with sync targets: SCM, MES, WMS, SAP Financial (badges animate to Synced)
- Previous version → `Superseded`

---

### 5. BOM completeness analysis (`/pdm/bom/:id/analysis`)

**Dashboard panel** for a selected product BOM:

#### Summary cards

- Total unique materials
- Materials in stock / partial / shortage
- Estimated total cost (mock from latest prices)

#### Analysis table

| Material | Required qty | On hand | In transit | Shortage | Source type |
|----------|--------------|---------|------------|----------|-------------|
| LIN-001 | 120 m | 80 | 20 | 20 | Procurement |

**Row styling:** shortage rows with warning background.

**Actions:**

- `Export analysis` (mock CSV)
- `Use for order review` — navigates to stub with pre-filled data

**Filters:** show shortages only, source type.

---

### 6. BOM detail read-only (`/pdm/bom/:id`)

For approved BOMs:

- Tree read-only
- Version badge + approval trail
- Links to related cost pricing and routing BOM links
- CTA `Request change` if archived product

## Mock Data Shape

```ts
type BomLevel = 'finished_goods' | 'semi_finished' | 'raw_material' | 'auxiliary'
type SourceType = 'in_house' | 'procurement' | 'subcontracting'

interface BomHeaderDto {
  Id: string
  StyleNumber: string
  ProductName: string
  Version: number
  Status: 'draft' | 'in_review' | 'approved' | 'superseded'
  EffectiveDate: string | null
  ChangeReason: string | null
  Lines: BomLineDto[]
  Versions: BomVersionSummaryDto[]
  ApprovalSteps: ApprovalStepDto[]
  SyncStatus: { system: string; status: string }[]
}

interface BomLineDto {
  Id: string
  ParentId: string | null
  Level: BomLevel
  MaterialCode: string
  MaterialName: string
  Specification: string
  Unit: string
  UnitConsumption: number
  WastageRate: number
  EffectiveConsumption: number
  SourceType: SourceType
  Children: BomLineDto[]
}

interface BomAnalysisDto {
  BomId: string
  TotalMaterials: number
  InStockCount: number
  ShortageCount: number
  EstimatedTotalCost: number
  Lines: {
    MaterialCode: string
    MaterialName: string
    RequiredQty: number
    OnHand: number
    InTransit: number
    Shortage: number
    SourceType: SourceType
  }[]
}
```

Mocks: `src/mocks/pdm/bom.ts`.
