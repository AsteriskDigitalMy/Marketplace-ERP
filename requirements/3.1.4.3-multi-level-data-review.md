# 3.1.4.3 Multi-Level Review of Filled Data — Frontend Requirements

**Source:** F-DATA-003 — Section 3.1.4 Data Collection and Filling  
**Route:** `/pms/data-collection/reviews`

---

## Functional Summary

Team leaders and department heads review submitted data level-by-level. Pass → locked into database for KPI calc; reject → return to filler with reason. Supports in-workflow messaging (mock).

**Prerequisites:** Data review permission; status `Pending Review`.

---

## Review Queue

Filter: level (team / department), period, submitter, status.

Table: Record ID, Indicator, Submitter, Period, Level, Submitted at, Actions.

---

## Review Detail

Read-only filled values + validation summary.

| Field | Required |
|-------|----------|
| Result | Pass / Reject |
| Opinion | Required on reject |

**Pass:** status `Approved`; data locked badge; eligible for [calculation](./3.1.5.1-scheduled-batch-calculation.md).  
**Reject:** notify filler; status `Rejected`; edit and resubmit link.

### Messaging panel (mock)

Thread between reviewer, filler, and other parties for verification Q&A.
