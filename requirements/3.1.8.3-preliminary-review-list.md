# 3.1.8.3 All-Staff Preliminary Performance Review List View — Frontend Requirements

**Source:** F-PERF-003 — Section 3.1.8 Performance Appraisal & Scoring  
**Route:** `/pms/appraisal/preliminary`

---

## Functional Summary

Auditors view system-generated performance results for all employees in the current cycle. Click an employee to see full indicator breakdown in a pop-up.

**Input:** Current performance appraisal cycle.  
**Prerequisites:** Preliminary performance review approval permission.

---

## List Page

### Header

- Cycle selector (dropdown of open cycles)
- Summary chips: total staff, by grade (A/B/C/D counts), pending review count

### Employee table

| Column | Sortable |
|--------|----------|
| Employee ID | Yes |
| Name | Yes — click opens detail |
| Department | Yes |
| Total score | Yes |
| Auto grade | Yes — color badge |
| Confirmed grade | Editable in review flow ([3.1.8.4](./3.1.8.4-preliminary-review-shunting.md)) |
| Status | Pending preliminary review |

Filters: department, grade, name search.

Export button (mock CSV download).

---

## Employee Detail Popover / Dialog

Triggered by clicking employee name.

### Content

- Employee header: name, ID, department, role
- Score summary: total score, auto grade, traffic-light badge
- **Indicator breakdown table:** KPI name, raw score, weight %, weighted contribution, period target vs actual (mock)
- Mini chart: radar or bar of indicator scores (optional)

Read-only in this view; confirmation actions on [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md).
