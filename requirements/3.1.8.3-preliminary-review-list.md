# 3.1.8.3 All-Staff Preliminary Performance Review List View — Frontend Requirements

**Source:** F-PERF-003 — Section 3.1.8 Performance Appraisal & Scoring  
**Route:** `/pms/appraisal/preliminary`  
**Workflow:** Receives drafts from [3.1.8.2](./3.1.8.2-automatic-scoring-grading.md) using schemes from [3.1.8.1](./3.1.8.1-appraisal-scheme-configuration.md) → confirmation actions on [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md) → [3.1.8.5](./3.1.8.5-hr-rectification.md) / [3.1.8.7](./3.1.8.7-executive-final-review.md)

---

## Functional Summary

Auditors view system-generated performance results for all employees in the current cycle. Click an employee to see full indicator breakdown in a pop-up.

**Input:** Current performance appraisal cycle.  
**Prerequisites:** Preliminary performance review approval permission.

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Goals | Pain Points |
|---------|-------|-------------|
| **Auditor** | Review all staff scores in one cycle-scoped view; drill into indicators | Switching between spreadsheets and source systems |
| **HR Administrator** | Monitor grade distribution before shunting | No aggregate grade summary |
| **Executive / CFO** (read-only peek) | Spot-check outliers before final review | Limited visibility until final queue |

### Core Capabilities (FR-001…)

| ID | Capability | Priority |
|----|------------|----------|
| FR-001 | Cycle selector scoped to open/generated cycles | Must |
| FR-002 | Summary chips: total staff, grade counts (A/B/C/D), pending review count | Must |
| FR-003 | Sortable, filterable employee table with scores and grades | Must |
| FR-004 | Employee name opens read-only indicator detail dialog | Must |
| FR-005 | Grade badges use traffic-light colors from global rules | Must |
| FR-006 | Export table to CSV (mock download) | Should |
| FR-007 | Surface confirmed grade column (editable in [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md)) | Must |
| FR-008 | Auditor permission gate | Must |

### Edge Cases & Alternative Paths

- **No cycles with generated data:** Empty state with link to [3.1.8.2](./3.1.8.2-automatic-scoring-grading.md).
- **Cycle switch mid-filter:** Reset filters; refetch table.
- **All employees already shunted:** Pending count = 0; table shows historical statuses.
- **Export with zero rows:** Toast "Nothing to export."
- **Detail dialog for employee with missing indicator:** Show available rows; placeholder for gaps.
- **Deep link `?cycleId=`:** Pre-select cycle on load.
- **Unauthorized user:** 403 full page.

## Phase 2: Screen-by-Screen Functional Specifications

### Screen: Preliminary Review List (`/pms/appraisal/preliminary`)

* **Objective:** Present cycle-scoped all-staff appraisal results for auditor review.
* **UI Elements & Field-Level Specs:**
  * Page title: "Preliminary Performance Review"
  * **Header toolbar:**
    * Cycle selector dropdown (open cycles with generated data)
    * Summary chips: Total staff, Grade A/B/C/D counts (color badges), Pending review count
    * **Export CSV** secondary button
  * **Filters row:** Department (multi), Grade (multi), Name/ID search (debounced 300ms)
  * **Employee table** columns: Employee ID (sortable), Name (sortable, link style), Department (sortable), Total score (sortable), Auto grade (sortable, traffic-light badge), Confirmed grade (shows "—" until set via [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md)), Status (Pending preliminary review / Shunted)
  * Row click on name opens detail dialog (not row selection — selection handled in [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md))
  * Pagination: 25/50/100 per page
  * Four async states: skeleton → empty → error → data
* **Functional Logic:**
  * Default to latest open cycle or `cycleId` query param
  * Sort client-side or server mock; preserve on cycle change optional
  * Export generates mock CSV blob from filtered rows
  * Confirmed grade column read-only on this screen

### Modal: Employee Detail Dialog

* **Objective:** Read-only deep dive into one employee's appraisal breakdown.
* **UI Elements & Field-Level Specs:**
  * Header: Employee name, ID, department, role
  * Score summary row: Total score (large), Auto grade badge, traffic-light indicator
  * **Indicator breakdown table:** KPI name, Raw score, Weight %, Weighted contribution, Target vs Actual (mock columns)
  * Optional mini chart: radar or bar of indicator scores
  * Footer: **Close**; link **Confirm grade** → scroll/highlight row on list for [3.1.8.4](./3.1.8.4-preliminary-review-shunting.md) actions (same page)
* **Functional Logic:**
  * Load `IndicatorDetails` from row data or lazy mock fetch
  * No edit controls; ESC and overlay click close dialog
  * Focus trap for accessibility

## Phase 3: Screen Transition & User Flow Mapping

| # | Trigger | Source | Destination | Conditional paths |

| 8 | Empty state CTA | No data | [3.1.8.2](./3.1.8.2-automatic-scoring-grading.md) | — |
| 9 | Filter/sort change | List | Same page | URL query optional |
