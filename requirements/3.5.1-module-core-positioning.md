# 3.5.1 Core Module Positioning — Frontend Requirements

**Source:** Section 3.5.1 — Warehouse Management System (WMS)  
**Module role:** End-to-end warehouse operations for raw materials, auxiliary materials, semi-finished and finished products — inbound, outbound, intra-warehouse, counting, and traceability via PDA/RFID.

---

## Overview

WMS is the **inventory execution layer** of the ERP frontend. It must support:

- **Admin portal** (PC, 1024px+) — configuration, monitoring, approvals, reports
- **PDA operational UI** (mobile-first, 360px) — scan-centric floor workflows
- **Dashboard viewport** (optional large-screen) — warehouse visualization

Physical stock ↔ system record consistency is a core UX theme: always show book qty, location status, and sync state clearly.

## Global WMS UI Patterns

1. **Barcode scan input** — reusable `ScanField` with mock scan button, success/error haptic-style feedback (visual).
2. **Task card** — status, priority, due date, assigned operator.
3. **Quantity stepper** — large touch targets on PDA; tabular nums.
4. **Location recommendation chip** — "Suggested: A-01-03" with accept/override.
5. **Ledger line status** — `Available` | `Reserved` | `Frozen` | `Locked` badges.
6. **Sync status pill** — per document: Pending / Synced / Failed.

| Requirement | Detail |
