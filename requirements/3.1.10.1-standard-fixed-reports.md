# 3.1.10.1 Automatic Generation of Industry Standardized Fixed Reports — Frontend Requirements

**Source:** F-RPT-001 — Section 3.1.10 Report Center Module  
**Route:** `/pms/reports`  
**Related workflow:** Performance reports consume published data from [3.1.8.7](./3.1.8.7-executive-final-review.md) cycle; KPI reports tie to [3.1.8.1](./3.1.8.1-appraisal-scheme-configuration.md) / [3.1.8.2](./3.1.8.2-automatic-scoring-grading.md)

---

## Functional Summary

Built-in standard business reports for the automotive leather processing industry. Generate daily, weekly, monthly, and quarterly reports with one click. Online preview plus PDF and Excel export (mock).

**Input:** Selected report type, statistical time cycle.  
**Prerequisites:** Business data for the cycle archived in database (mock completeness check).

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Goals | Pain Points |
|---------|-------|-------------|
| **Department Manager** | Quick standard reports without BI setup | Manual Excel assembly |
| **Executive / CFO** | Quarterly appraisal and KPI summaries post-publish [3.1.8.7](./3.1.8.7-executive-final-review.md) | Delayed consolidated views |
| **Quality / Production Lead** | OEE, defect Pareto, yield reports | Multiple disconnected systems |

### Core Capabilities (FR-001…)

| ID | Capability | Priority |
|----|------------|----------|
| FR-001 | Report catalog grouped by category (Production, Quality, Supply chain, Performance, Cost) | Must |
| FR-002 | Per-report supported cycles (D/W/M/Q) and last generated timestamp | Must |
| FR-003 | Generator panel: report type, cycle, period, output format | Must |
| FR-004 | Data readiness gate before generate (mock completeness bar) | Must |
| FR-005 | Online preview with tabular sections and summary KPIs | Must |
| FR-006 | Mock PDF, Excel export, and print-friendly preview | Must |
| FR-007 | Loading/progress state during generation | Must |
| FR-008 | Performance category links to appraisal published cycles [3.1.8.7](./3.1.8.7-executive-final-review.md) | Should |

### Edge Cases & Alternative Paths

- **Data not archived for period:** Generate disabled; readiness bar red with explanation.
- **Report type + cycle mismatch:** Period selector adapts; invalid combos disabled.
- **Generate while another in flight:** Disable button; show in-progress state.
- **Empty result set (mock):** Preview shows empty state section "No data for period."
- **Export without preview:** Allow direct export from generator (mock download toast).
- **Print from preview:** `@media print` hides chrome.
- **Future period selected:** Validation error on generate.
- **Catalog empty (mock failure):** Error state with retry.

## Phase 2: Screen-by-Screen Functional Specifications

### Screen: Report Catalog (`/pms/reports`)

* **Objective:** Browse standard reports and launch generation with preview/export.
* **UI Elements & Field-Level Specs:**
  * Page title: "Report Center"
  * Layout: category tabs or grouped cards — Production, Quality, Supply chain, Performance, Cost
  * Per card/row: Report name, Description, Supported cycles chips (D/W/M/Q), Last generated timestamp (relative), **Select** action
  * Performance category examples: Monthly KPI achievement, Quarterly appraisal summary (data from [3.1.8.7](./3.1.8.7-executive-final-review.md))
  * **Report Generator** panel (right drawer or bottom panel, opens on Select):
    * Report type (read-only when selected from card)
    * Time cycle (Daily / Weekly / Monthly / Quarterly — filtered by report support)
    * Period (date picker / week picker / month picker / quarter picker — dynamic by cycle)
    * Output format (Preview / PDF / Excel)
    * Data readiness indicator (progress bar + "Data archived ✓" or failure message)
    * **Generate** primary (disabled until readiness pass and period valid)
  * Four async states on catalog
* **Functional Logic:**
  * Selecting card pre-fills generator report type
  * Readiness mock checks archived flag for period
  * Generate with Preview → opens/focuses preview pane; PDF/Excel → mock blob toast

### Panel: Preview Pane (inline or split view on `/pms/reports`)

* **Objective:** Display generated report content for review and export.
* **UI Elements & Field-Level Specs:**
  * Report header: title, period label, generated at timestamp
  * Tabular sections per `GeneratedReportDto.Sections` — title, data table, optional summary KPI row
  * Toolbar: **Export PDF**, **Export Excel**, **Print** (all mock client-side)
  * Print-friendly styles: hide nav, white background (`@media print`)
  * Loading skeleton during generation
  * Close preview (returns to catalog-only layout)
* **Functional Logic:**
  * Render sections from mock `GeneratedReportDto`
  * Print invokes `window.print()` on preview container
  * Export buttons trigger mock download with filename `{ReportName}_{Period}.pdf/xlsx`

### Modal: Generation Progress

* **Objective:** Feedback during mock report build.
* **UI Elements:** Indeterminate or stepped progress bar, status text ("Aggregating KPI data…"), cancel optional (mock)
* **Functional Logic:** On complete, close modal and show preview or export per format choice

### Modal: Data Not Ready

* **Objective:** Explain blocked generation when prerequisites fail.
* **UI Elements:** Missing data sources list (mock), **Choose different period** / **Close**
* **Functional Logic:** Shown when readiness check fails on Generate click
