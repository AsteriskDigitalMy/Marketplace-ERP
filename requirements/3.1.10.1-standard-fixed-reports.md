# 3.1.10.1 Automatic Generation of Industry Standardized Fixed Reports — Frontend Requirements

**Source:** F-RPT-001 — Section 3.1.10 Report Center Module  
**Route:** `/pms/reports`

---

## Functional Summary

Built-in standard business reports for the automotive leather processing industry. Generate daily, weekly, monthly, and quarterly reports with one click. Online preview plus PDF and Excel export (mock).

**Input:** Selected report type, statistical time cycle.  
**Prerequisites:** Business data for the cycle archived in database (mock completeness check).

---

## Report Catalog (`/pms/reports`)

Card or table layout grouped by category:

| Category | Example reports (mock) |
|----------|------------------------|
| Production | Daily output summary, weekly OEE, monthly yield analysis |
| Quality | Weekly defect Pareto, monthly IPQC/OQC summary |
| Supply chain | Monthly order fulfillment, quarterly supplier scorecard |
| Performance | Monthly KPI achievement, quarterly appraisal summary |
| Cost | Monthly production cost variance |

Each card: report name, description, supported cycles (D/W/M/Q), last generated timestamp.

---

## Report Generator Panel

| Field | Required |
|-------|----------|
| Report type | Yes — from catalog |
| Time cycle | Daily / Weekly / Monthly / Quarterly |
| Period | Date picker or week/month/quarter selector |
| Output format | Preview / PDF / Excel |

**Generate** button:

1. Data readiness check (mock bar — "Data archived ✓")
2. Loading state with progress
3. On success → preview pane or download toast (mock blob)

---

## Preview Pane

- Report header: title, period, generated at
- Tabular sections + summary KPIs (mock HTML layout)
- Print-friendly styles (`@media print`)
- Actions: Export PDF, Export Excel, Print (all mock client-side)
