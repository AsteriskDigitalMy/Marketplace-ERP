# 3.2.5 Product Cost Pricing Management — Frontend Requirements

**Source:** Section 3.2.5  
**Purpose:** Automatic standard cost calculation from BOM, labor hours, and overheads; quotation generation and variance analysis.

---

## Screens

### 1. Cost pricing list (`/pdm/cost-pricing`)

| Column | |
|--------|--|
| Style / product | Link |
| Cost type | Badge: Sampling / Mass production |
| Standard cost | Currency, tabular nums |
| Quotation status | Draft / Approved / Sent / Confirmed |
| BOM version used | |
| Last calculated | |
| Actions | View, Recalculate, Quotation |

**Filters:** cost type, quotation status, product.

**Actions:** `New cost sheet`, `Recalculate all` (bulk mock).

---

### 2. Cost calculation detail (`/pdm/cost-pricing/:id`)

#### Header

- Product / style reference
- Cost type toggle: **Sampling** | **Mass production** (segregated views)
- Linked BOM version + routing version selectors
- `Recalculate` button — refreshes breakdown from mocked sources

#### Cost breakdown card (auto-calculated)

| Dimension | Formula (display) | Amount |
|-----------|-------------------|--------|
| Material costs | BOM consumption × latest procurement price | $X |
| Labor costs | Standard hours × hourly wage rate | $X |
| Manufacturing overhead | Allocated % | $X |
| Administrative overhead | Allocated % | $X |
| Profit markup | Configurable % | $X |
| **Total standard cost** | Sum | **$X** |

Each line expandable to show line-item detail table (mock).

**Editable parameters panel:**

| Parameter | Input |
|-----------|-------|
| Hourly wage rate | Currency |
| Mfg overhead rate | % |
| Admin overhead rate | % |
| Profit markup | % |

Changes trigger live recalculation (mock formula in frontend).

---

### 3. Quotation sheet (`/pdm/cost-pricing/:id/quotation`)

**Printable quotation layout** (A4 preview).

| Section | Content |
|---------|---------|
| Header | Company logo placeholder, quotation #, date, valid until |
| Customer | From linked project/customer |
| Line items | Product, qty, unit price, total |
| Terms | Textarea (editable) |
| Totals | Subtotal, tax (optional), grand total |

**Actions:**

| Action | Behavior |
|--------|----------|
| Edit | Toggle edit mode on line items and terms |
| Submit for approval | Approval workflow (Sales → Management) |
| Print | `window.print()` styled print CSS |
| Send to customer | Mock email dialog → status `Sent` |
| Confirm | Status → `Confirmed`; show SCM sync banner |

**On confirm (mock):**

- Toast: "Quotation confirmed — available in SCM sales orders"
- Link: `Create sales order` (stub with pre-filled quotation data)

---

## Mock Data Shape

```ts
type CostType = 'sampling' | 'mass_production'

interface CostPricingDto {
  Id: string
  StyleNumber: string
  ProductName: string
  CostType: CostType
  BomVersionId: string
  RoutingVersionId: string
  Parameters: {
    HourlyWageRate: number
    ManufacturingOverheadRate: number
    AdministrativeOverheadRate: number
    ProfitMarkupRate: number
  }
  Breakdown: {
    MaterialCosts: number
    LaborCosts: number
    ManufacturingOverhead: number
    AdministrativeOverhead: number
    ProfitMarkup: number
    TotalStandardCost: number
  }
  BreakdownDetails: CostLineDetailDto[]
  Quotation: QuotationDto | null
  CalculatedDatetime: string
}

interface QuotationDto {
  Id: string
  QuotationNumber: string
  Status: 'draft' | 'in_review' | 'approved' | 'sent' | 'confirmed'
  CustomerName: string
  ValidUntil: string
  LineItems: { Description: string; Quantity: number; UnitPrice: number; Total: number }[]
  Terms: string
  GrandTotal: number
  ApprovalSteps: ApprovalStepDto[]
}

interface CostVarianceDto {
  ProductId: string
  StyleNumber: string
  StandardCost: number
  ActualCost: number
  Variance: number
  VariancePercent: number
  PrimaryDriver: string
  DimensionBreakdown: { dimension: string; standard: number; actual: number; variance: number }[]
}
```

Mocks: `src/mocks/pdm/cost-pricing.ts`.
