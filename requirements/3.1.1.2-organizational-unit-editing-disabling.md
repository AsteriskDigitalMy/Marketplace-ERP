# 3.1.1.2 Organizational Unit Editing and Disabling — Frontend Requirements

**Source:** F-BASE-002 — Section 3.1.1 System Basic Management  
**Route:** `/pms/admin/org/:id/edit`

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Goals |
|---------|-------|
| **System Administrator (primary)** | Correct org metadata (name, tier, sort order) and retire idle units without deleting historical data. Needs clear impact warnings before disable. |
| **HR / Org Architect** | Align org records with physical restructuring; re-enable units when parent becomes active again. |
| **Auditor** | Review change history and verify disabled units are excluded from business selectors app-wide. |
| **Operations Manager (indirect)** | Expects disabled org branches to disappear from KPI and project pickers immediately (mock propagation). |

### Core Capabilities

1. **FR-001 — Edit form auto-fill:** Load `OrgUnitEditDto` for `:id` and populate the same field set as creation ([3.1.1.1](./3.1.1.1-organizational-unit-creation.md)), aligned with `OrgUnitSchema` in `src/models/pms/organization.ts`.
2. **FR-002 — Sibling name uniqueness on save:** Re-validate `Name` uniqueness among siblings excluding current unit; block save with inline error.
3. **FR-003 — Logical disable:** Set `IsDisabled = true` with double-confirmation **Dialog**; cascade visual disable to all descendants in tree mock.
4. **FR-004 — Re-enable unit:** Restore `IsDisabled = false` only when direct parent is active; otherwise block with **Toast** error.
5. **FR-005 — Active workflow gate:** Block disable when `HasActiveWorkflows = true`; show **Alert** with explanation.
6. **FR-006 — Version history on save:** Each successful save appends entry to `ChangeHistory` and increments architecture version (mock).
7. **FR-007 — Selector exclusion mock:** Disabled units and descendants removed from org pickers app-wide (mock flag/event).
8. **FR-008 — Async states:** Loading, error, empty (not found), and success for fetch, save, disable, and enable actions.

### Edge Cases & Alternative Paths

- Unit has many children (`ChildCount > 0`) → disable confirmation shows strong **Alert**: "All sub-organizations will be hidden synchronously."
- User edits name to duplicate sibling → inline error on blur and on save.
- User changes tier to `process` without tag → same validation as creation.
- Parent becomes disabled while editing child → **Enable** on child remains blocked until parent re-enabled.
- User navigates away with dirty form → browser/route **Dialog** "Discard changes?"
- Save succeeds but version write fails (mock) → **Toast** warning; form shows saved values.
- Unit not found (`:id` invalid) → 404 empty state with link back to `/pms/admin/org`.
- Double-click **Save** / **Disable** → buttons disabled until request completes.
- From tree context menu **Edit** → navigate to `/pms/admin/org/:id/edit`.
- Breadcrumb **Org Structure** → `/pms/admin/org`.

## Phase 2: Screen-by-Screen Functional Specifications

### Screen Name: Organization Unit Edit (`/pms/admin/org/:id/edit`)

* **Objective:** Modify org unit attributes and manage active/disabled lifecycle with audit trail.

* **UI Elements & Field-Level Specs:**
  * **Page header:** Unit `Name` + `Code`; status **Badge** (Active / Disabled); breadcrumb: Admin → Org Structure → Edit.
  * **Metadata strip:** `ChildCount`, last changed by/at from `ChangeHistory[0]` (mock).
  * **Parent node (read-only `Input`):** Shows parent name; disabled.
  * **Organization name (`Input`):** Required; max 200; sibling uniqueness validation on blur.
  * **Tier type (`Select`):** Required; disabled if unit has children that constrain tier (mock: lock tier when children exist).
  * **Process business tag (`Combobox`):** Required when tier = process; hidden otherwise.
  * **Sort order (`Input` number):** Required; integer ≥ 0.
  * **Active workflow warning (`Alert` destructive):** Visible when `HasActiveWorkflows`; blocks disable action.
  * **Action bar — Save (`Button`):** Primary; disabled when pristine or invalid or submitting.
  * **Action bar — Disable (`Button` `variant="destructive"`):** Visible when `IsDisabled = false` and not blocked by workflows.
  * **Action bar — Enable (`Button`):** Visible when `IsDisabled = true`; disabled if parent inactive.
  * **Change history section (`Table`):** Columns Version, ChangedAt, ChangedBy, Summary; read-only; paginated if > 10 rows.
  * **Loading / error / not-found states:** Skeleton form; **Alert** + retry; empty 404 card with back link.

* **Functional Logic:**
  * Fetch `OrgUnitEditDto` on mount; populate react-hook-form aligned with `OrgUnitSchema`.
  * **Save:** Zod validate → `PUT` mock → **Toast** success → append `ChangeHistory` row → optional navigate back or stay.
  * **Disable:** Open **Disable Confirmation Dialog** (see below); on confirm → `PATCH` mock → update local `IsDisabled` → **Toast** "Unit disabled; excluded from statistics" → gray styling in linked tree mock.
  * **Enable:** Single **Dialog** confirm → validate parent active → `PATCH` mock → **Toast** success.
  * Dirty form navigation guard via `beforeunload` and route blocker.

### Screen Name: Disable Confirmation Dialog (modal on `/pms/admin/org/:id/edit`)

* **Objective:** Obtain explicit consent before logically disabling a unit and hiding its subtree.

* **UI Elements & Field-Level Specs:**
  * **Dialog:** Title "Disable organizational unit?"; `variant` attention styling.
  * **Body text:** "This unit will be excluded from statistics and business selectors."
  * **Child warning (`Alert` destructive):** Shown when `ChildCount > 0`: "All {ChildCount} sub-organizations will be hidden synchronously."
  * **Unit summary:** Read-only name + code.
  * **Cancel (`Button` outline):** Closes without action.
  * **Confirm Disable (`Button` destructive):** Disabled while request in flight.

* **Functional Logic:**
  * Block open when `HasActiveWorkflows` — show **Toast** instead.
  * On confirm: async disable; double-click prevention; on success close dialog and update parent page state.
  * On error: **Toast** destructive; dialog stays open.

### Screen Name: Enable Confirmation Dialog (modal on `/pms/admin/org/:id/edit`)

* **Objective:** Confirm re-activation when parent chain is valid.

* **UI Elements & Field-Level Specs:**
  * **Dialog:** Title "Re-enable organizational unit?"
  * **Parent status line:** Shows parent name + active badge.
  * **Confirm (`Button`):** Disabled if parent `IsDisabled`.

* **Functional Logic:**
  * Pre-check parent active; if not → **Toast** "Cannot enable: parent unit is disabled" — do not open dialog.
  * On success: **Toast** "Unit re-enabled"; refresh form state.

## Phase 3: Screen Transition & User Flow Mapping

| # | Trigger Event | Source Screen | Destination Screen | Conditional Paths |
|---|---------------|---------------|--------------------|-------------------|
| 1 | Click **Edit** on tree node | Organization Tree (`/pms/admin/org`) | Organization Unit Edit (`/pms/admin/org/:id/edit`) | — |
| 2 | Page load with valid `:id` | — | Organization Unit Edit | Invalid id → 404 state |
| 3 | Click **Save** (valid) | Organization Unit Edit | Organization Unit Edit | Toast success; history table updates |
| 4 | Click **Disable** | Organization Unit Edit | Disable Confirmation Dialog | Active workflows → Toast block, no dialog |
| 5 | Confirm disable | Disable Confirmation Dialog | Organization Unit Edit | Success → status badge Disabled |
| 6 | Click **Enable** | Organization Unit Edit | Enable Confirmation Dialog | Parent disabled → Toast only |
| 7 | Confirm enable | Enable Confirmation Dialog | Organization Unit Edit | Success → status Active |
| 8 | Breadcrumb **Org Structure** | Organization Unit Edit | Organization Tree | Dirty form → discard confirm |
| 9 | Click **Version History** link (mock) | Organization Unit Edit | Version History (`/pms/admin/org/:id/history`) | See [3.1.1.3](./3.1.1.3-org-structure-version-rollback.md) |
