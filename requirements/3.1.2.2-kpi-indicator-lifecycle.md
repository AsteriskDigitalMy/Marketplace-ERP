# 3.1.2.2 Indicator Editing / Disabling / Enabling / Deletion — Frontend Requirements

**Source:** F-KPI-002 — Section 3.1.2 KPI Indicator Library Management  
**Route:** `/pms/kpi/indicators/:id`

---

## Functional Summary

Modify indicators (new version on save), disable/enable, or delete only when no calculation history. Core industry KPIs have locked key fields.

**Prerequisites:** Indicator management permission.

---

## Indicator Detail

Tabs: Attributes, Version history, Usage stats (mock).

### Actions

| Action | Rule |
|--------|------|
| Edit | Creates new version; history retained |
| Disable | No new cycle calculations; historical data preserved |
| Enable | Restores participation |
| Delete | Only if zero calculation records — else disabled button + tooltip |

**Core locked indicators:** code, formula, category read-only with lock icon.

### Version history table

Version, changed at, changed by, summary, diff link (mock).
