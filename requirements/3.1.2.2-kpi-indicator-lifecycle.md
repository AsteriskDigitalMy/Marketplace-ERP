# 3.1.2.2 Indicator Editing / Disabling / Enabling / Deletion — Frontend Requirements

**Source:** F-KPI-002 — Section 3.1.2 KPI Indicator Library Management  
**Route:** `/pms/kpi/indicators/:id`  
**Related:** [3.1.2.1](./3.1.2.1-kpi-indicator-addition.md) (creation), [3.1.2.3](./3.1.2.3-kpi-formula-editor.md) (formula), [3.1.5.1](./3.1.5.1-scheduled-batch-calculation.md) (calculation history), [3.1.3.1](./3.1.3.1-project-initiation.md) (KPI binding)

---

## Functional Summary

Modify indicators (new version on save), disable/enable, or delete only when no calculation history. Core industry KPIs have locked key fields.

**Prerequisites:** Indicator management permission.

---

## Phase 1: Deep Requirement Analysis

### User Personas

| Persona | Role | Goals | Pain Points |
|---------|------|-------|-------------|
| **KPI Administrator** | Maintains indicator library | Evolve formulas without losing audit trail | Accidental deletion of indicators in active use |
| **Compliance Auditor** | Read-only reviewer | Trace version history and who changed what | No diff visibility between versions |
| **Project Manager** | Binds KPIs at initiation | Know if an indicator is disabled before binding | Stale enabled status in project forms |

### Core Capabilities (FR-xxx)

| ID | Capability | Priority |
|----|------------|----------|
| FR-001 | Display indicator detail with tabs: Attributes, Version History, Usage Stats | Must |
| FR-002 | Edit saves new semantic version; prior versions retained in history table | Must |
| FR-003 | Disable/enable toggle with status `Badge` and confirmation `AlertDialog` | Must |
| FR-004 | Delete only when `HasCalculationHistory === false`; else disabled with `Tooltip` | Must |
| FR-005 | Core locked indicators: code, formula, category read-only with `Lock` icon | Must |
| FR-006 | Version history table with diff link (mock) and changed-by metadata | Must |
| FR-007 | Usage stats tab shows mock project/scheme references from [3.1.3.1](./3.1.3.1-project-initiation.md) | Should |

### Edge Cases & Alternative Paths

- **Delete with calculation history:** `Button` disabled; `Tooltip` "Cannot delete — used in calculations ([3.1.5.1](./3.1.5.1-scheduled-batch-calculation.md))".
- **Disable while bound to in-progress projects:** Warning `Alert` listing affected projects (mock); allow disable with confirm.
- **Edit core locked indicator:** Only non-locked fields editable; locked fields use `Input` disabled + lock icon.
- **Concurrent edit conflict:** Mock 409 → toast "Indicator updated elsewhere; refresh".
- **Enable disabled indicator with broken formula references:** Block enable; link to formula editor [3.1.2.3](./3.1.2.3-kpi-formula-editor.md).
- **Indicator not found:** Empty/error state with link back to `/pms/kpi/indicators`.

## Phase 2: Screen-by-Screen Functional Specifications

### Screen: KPI Indicator Detail — `/pms/kpi/indicators/:id`

* **Objective:** View and manage a single indicator's lifecycle, versions, and usage.

* **UI Elements & Field-Level Specs:**
  - `Breadcrumb`: KPI Library → `{Code}`.
  - Header `Card`: indicator name, code `Badge`, status `Badge` (enabled/disabled), version `Badge`, `IsCoreLocked` lock icon.
  - Action bar: `Button` "Edit" (primary), `DropdownMenu` with Disable/Enable, Delete (destructive, conditional).
  - `Tabs`: **Attributes** | **Version History** | **Usage Stats**.

* **Functional Logic:**
  - Load indicator + version list from mock API by `:id`.
  - Permission gate on destructive actions.

---

### Tab: Attributes — `/pms/kpi/indicators/:id` (default tab)

* **Objective:** Read-only view of all fields; switches to edit mode on "Edit".

* **UI Elements & Field-Level Specs:**
  - Read-only `DescriptionList` or disabled `Form` fields mirroring [3.1.2.1](./3.1.2.1-kpi-indicator-addition.md) field set.
  - Formula field: monospace `Textarea` read-only + link "Edit formula" → `/pms/kpi/indicators/:id/formula` ([3.1.2.3](./3.1.2.3-kpi-formula-editor.md)).
  - Locked fields: `Lock` lucide icon + `Tooltip` "Core industry KPI — field locked".

* **Functional Logic:**
  - Edit mode toggles same form as creation with pre-filled values; Save creates new version string (e.g. V1.1).

---

### Tab: Version History

* **Objective:** Audit trail of indicator changes.

* **UI Elements & Field-Level Specs:**
  - `DataTable`: Version, Changed At, Changed By, Summary, Actions (Diff link mock `Button` variant link).
  - `ScrollArea` for long histories; empty state "No prior versions".

* **Functional Logic:**
  - Diff link opens `Dialog` with side-by-side JSON/text diff (mock).
  - Current version row highlighted.

---

### Tab: Usage Stats (mock)

* **Objective:** Surface where indicator is referenced.

* **UI Elements & Field-Level Specs:**
  - `Card` metrics: Active projects count, Appraisal schemes count, Last calculated at.
  - `DataTable` of referencing entities with links to `/pms/projects/:id` where applicable.

* **Functional Logic:**
  - Mock data from extended `kpi-indicators.ts`; read-only.

---

### Modal: Disable / Enable Confirmation — `AlertDialog`

* **Objective:** Confirm status change impact.

* **UI Elements & Field-Level Specs:**
  - Title varies: "Disable indicator?" / "Enable indicator?"
  - Body explains: disable stops new cycle calculations per [3.1.5.1](./3.1.5.1-scheduled-batch-calculation.md); historical data preserved.
  - Actions: Cancel | Confirm.

* **Functional Logic:**
  - On confirm: update status `Badge`; toast success; refresh detail.

---

### Modal: Delete Confirmation — `AlertDialog`

* **Objective:** Irreversible delete when allowed.

* **UI Elements & Field-Level Specs:**
  - Destructive confirm `Button`; only rendered when `HasCalculationHistory === false`.

* **Functional Logic:**
  - Success → navigate to `/pms/kpi/indicators` with toast.

## Phase 3: Screen Transition & User Flow Mapping

| # | Trigger | Source | Destination | Conditional Paths |
|---|---------|--------|-------------|-------------------|
| 1 | Row click / View action | `/pms/kpi/indicators` | `/pms/kpi/indicators/:id` | Not found → error state |
| 2 | Click "Edit" | Detail Attributes tab | Same (edit mode) | Core locks apply |
| 3 | Save edit | Edit mode | Same (read mode) | New version appended to history |
| 4 | "Edit formula" link | Attributes tab | `/pms/kpi/indicators/:id/formula` | [3.1.2.3](./3.1.2.3-kpi-formula-editor.md) |
| 5 | Disable/Enable confirm | `AlertDialog` | Same detail page | Toast feedback |
| 6 | Delete confirm | `AlertDialog` | `/pms/kpi/indicators` | Blocked if calculation history |
| 7 | Tab switch | Detail | Same URL hash/tab state | Attributes / History / Usage |
| 8 | Project link in Usage | Usage tab | `/pms/projects/:id` | [3.1.3.1](./3.1.3.1-project-initiation.md) |
